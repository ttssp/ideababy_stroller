## Implementation plan

### scripts/install.sh(bash 3.2 兼容,≤ 60 行)
1. `set -euo pipefail`
2. 计算 `REPO_ROOT=$(git rev-parse --show-toplevel)`;若 fail → exit 1 提示"必须在 git 仓内"
3. 创建 `~/.local/bin/`(若不存在)+ `ln -sf <REPO_ROOT>/projects/007-007a-pA/bin/friction ~/.local/bin/friction`
4. 检查 `$PATH` 含 `~/.local/bin`,否则 stderr 提示 `add ~/.local/bin to PATH in ~/.zshrc / ~/.bashrc`
5. 写 `.claude/hooks/post_tool_use_failure.sh`(若不存在 — 不覆盖,允许 operator 已存在他类 hook):
   ```bash
   #!/usr/bin/env bash
   # friction-tap PostToolUseFailure hook · architecture Tradeoff-A
   PYTHONPATH="$(git rev-parse --show-toplevel)/projects/007-007a-pA/tools" \
     python3 -m friction_tap.hook
   exit 0  # always 0 · TECH-4
   ```
   `chmod +x .claude/hooks/post_tool_use_failure.sh`
6. 跑 `python3 -m friction_tap.state` 触发首次 default state.json 写入(确认 install 完成 → state.json 存在)
7. **R1 H3 fix · post-install real-payload smoke test**:
   ```bash
   # 临时启用以测试 hook 写入(测试完恢复 disabled)
   friction --on >/dev/null 2>&1 || true
   echo '{"tool_name":"Bash","exit_code":1,"stderr":"Error: smoke test"}' \
     | python3 -m friction_tap.hook
   # 验证 friction-log 末尾确实多了一行
   if tail -1 "$REPO_ROOT/docs/dogfood/v4-friction-log.md" 2>/dev/null \
        | grep -q "smoke test"; then
     echo "✓ post-install hook smoke test passed"
   else
     echo "⚠ post-install smoke test FAILED — hook 写入异常,检查 ~/.config/friction-tap/last_run.log" >&2
   fi
   # 验证 schema_error_count 仍为 0
   if python3 -c "from friction_tap.state import load; import sys; sys.exit(0 if load().schema_error_count == 0 else 1)"; then
     echo "✓ schema_error_count = 0 (no payload drift detected)"
   else
     echo "⚠ schema_error_count > 0 — 见 ~/.config/friction-tap/last_run.log" >&2
   fi
   # 删除 smoke 测试 entry(避免污染真实 dogfood 计数)
   sed -i '' '$ d' "$REPO_ROOT/docs/dogfood/v4-friction-log.md" 2>/dev/null || true
   # 恢复 disabled(operator 须显式 --on 进 dogfood)
   friction --off >/dev/null 2>&1 || true
   ```
8. 输出 next-step menu(architecture / project CLAUDE.md "every command outputs a next-step menu"):
   ```
   ✓ friction-tap v0.1 installed.

   Next steps:
     1. friction --on              # 启动 dogfood mode(D15 / ADR-5)
     2. echo 'docs/dogfood/v4-friction-log.md' >> .gitignore   # SEC-1 推荐
     3. echo 'docs/dogfood/v4-trust-w2.md' >> .gitignore
     4. bash scripts/install_launchd.sh    # day-14 trigger
     5. 看 README.md 了解审阅流程(含 review-mode 必跑教学 · R1 H2)

   To stop:  friction --off
   For review session:  friction --review-mode  # 关 hook 防 stale-save · R1 H2
   To remove:  bash scripts/uninstall.sh
   ```

## Known gotchas

- `~/.local/bin` 不在所有 shell 默认 PATH(zsh 较新版才默认含)→ install.sh 必须显式提示 add to rc
- `.claude/hooks/` 目录可能不存在(operator 仓首次接 Claude Code) → install.sh 先 mkdir -p
- macOS 默认 bash 是 3.2 — 不能用 `[[ ... ]]` 复杂 regex / 关联数组;只用 `[ ... ]` + 简单赋值
- `ln -sf` 如果目标已存在但是普通文件(非 symlink)会被覆盖 — 用户若已手放 friction 脚本会丢失;install.sh 应 detect + stderr 提示
- SEC-4 防御不在此 task 加固(hook entry 是 shell 脚本,operator 自己 git diff 可见)— 与原 design 一致
- 不在此 task commit dogfood log/trust report — operator dogfood 起点 = `friction --on` 后 14 天的 archive,不是本 ship 阶段产物
- next-step menu 必须用 ASCII (`✓` 单字符无副作用 · macOS terminal 默认 utf-8)— 不用复杂 emoji
