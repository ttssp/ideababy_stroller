## Goal(R1 H3 + R2 H-R2-3 + R2 B-R2-3 修订)

实现 Claude Code `PostToolUseFailure` hook 的 Python 入口:从 stdin 读 JSON → 调 judge → 调 format.append → 任何异常 silently swallow(TECH-4 关键约束 — 不能让 hook 异常污染 Claude Code session)。fire-and-forget 语义,exit 始终为 0。**R1 H3 fix · 加 schema-error 计数器**:每次 schema drift / parse 异常调 `state.bump_schema_error_count()`,trust-summary 在 day-14 显示该计数(D18 / TECH-9 mitigation)。**R2 H-R2-3 fix · 显式空 stdin pre-read 检查**:Python `json.load(sys.stdin)` 在空输入下抛 `JSONDecodeError`,但**空 stdin 是合法 noop**(Claude Code 早期版本 / hook 注册路径异常等场景),不应增 schema_error_count;故必须先 `raw = sys.stdin.read()` 检查 empty,**再** json.loads — 显式区分 empty stdin(legal noop)vs malformed JSON(real schema drift)。**R2 B-R2-3 fix · hook gate 加 review_mode 检查**:hook fire only if `state.enabled == true AND state.review_mode == false`(D17 / D20 — review session 期间静默暂停,但 enabled 仍 true → O3 不受影响)。

## Implementation plan

1. `main()` 顶层结构(R1 H3 + R2 H-R2-3 + R2 B-R2-3 修订):
   ```python
   try:
       # === R2 H-R2-3 fix: 显式 pre-read 区分 empty stdin (legal noop) vs malformed JSON (real schema drift) ===
       raw = sys.stdin.read()
       if not raw.strip():
           # Empty stdin = legal noop. 不增 schema_error_count, 不写 log.
           return  # exit 0
       try:
           payload = json.loads(raw)
       except json.JSONDecodeError:
           # Real schema drift (Claude Code 给了非空但不是 valid JSON 的 payload)
           try:
               state_module.bump_schema_error_count(reason="json_decode_error")
           except Exception:
               pass
           _log_exception()
           return  # exit 0
       # === Hook gate (R2 B-R2-3 fix): enabled AND NOT review_mode ===
       state = state_module.load()
       if not state.enabled:
           return  # exit 0 — D15 / ADR-5 默认 off
       if state.review_mode:
           return  # exit 0 — D17 / D20 review session 期间静默暂停 (enabled 仍 true → O3 PASS)
       try:
           disposition, confidence = judge.decide(payload, state)
       except judge.JudgeError as e:
           # D18 / R1 H3:schema drift → 计数 + log,silently exit 0
           state_module.bump_schema_error_count(reason=str(e))
           _log_exception()
           return  # exit 0
       if disposition == "skip":
           return  # exit 0
       line = format.render_agent_emit(
           confidence=confidence,
           tool=payload["tool_name"],
           exit_code=payload["exit_code"],
           stderr=payload.get("stderr", ""),
           task_desc=payload.get("task_description", ""),
       )
       format.append(line)
   except BaseException:  # 兜 KeyboardInterrupt / SystemExit etc · TECH-4
       _log_exception()
   sys.exit(0)
   ```
2. `_log_exception()`:`with open(last_run_log_path(), "a") as f: f.write(f"{datetime.now(timezone.utc).isoformat()} hook error\n{traceback.format_exc()}\n")` — 不抛、不让 exception 逃逸
3. **超时保护**:在最外层加 `signal.alarm(2)` 兜底(SLA hook p99 ≤ 500ms,2s alarm 是兜底);触发 SIGALRM 直接 `sys.exit(0)`
4. **stdin 读取语义(R2 H-R2-3 fix)**:**显式 `raw = sys.stdin.read()` 而非 `json.load(sys.stdin)`** — 前者让 empty case 显式可检测(`raw.strip() == ""`),后者会让空输入触发 `JSONDecodeError` 与 malformed JSON 不可区分。Python 标准 stdin EOF 模型在 hook 场景下天然产生空 string(Claude Code hook fire 时 stdin pipe 关闭但无内容),**这不是 schema drift**
5. **判定区分**(R2 H-R2-3 显式 invariant):
   - **empty stdin**(`not raw.strip()`)→ legal noop,**不增 schema_error_count**,无 log(避免 noise pollution)
   - **non-empty stdin + malformed JSON**(`raw.strip()` 后 `json.loads` 抛 JSONDecodeError)→ real schema drift,**bump_schema_error_count + log traceback**
   - **valid JSON 但 hook gate 关闭**(enabled=false 或 review_mode=true)→ silent skip,**不增 schema_error_count**
   - **valid JSON + judge.JudgeError**(payload 缺 tool_name 等)→ real schema drift,**bump_schema_error_count + log**
6. 顶部 docstring 引用 ADR-5 / TECH-4 / TECH-9 / D17 / D18 / D20 / SLA hook latency / D2 / R1 H3 / R2 H-R2-3 / R2 B-R2-3

## Known gotchas

- TECH-4 是高风险:**任何** Exception 必须捕获;包括 `KeyboardInterrupt` / `SystemExit` 也要兜(用 `except BaseException:` 而非 `except Exception:`)
- 不要 import 重模块(`asyncio` / `multiprocessing`)— hook 启动延迟会 leak 到 SLA p95
- 不在此 task 调 last_24h 检查(那是 trust_summary 的事 / T013)
- **R2 H-R2-3 关键修订**:stdin **必须用 `sys.stdin.read()` + `json.loads(raw)`,而不是 `json.load(sys.stdin)`** — 后者会让 empty input 触发 `JSONDecodeError`,与真正的 malformed JSON 不可区分,导致 empty stdin 被错误地计入 schema_error_count(违反"empty 是 legal noop"语义)。本 task R1 之前的 docstring 写的是反向建议(`json.load(sys.stdin)` 更宽容空输入),这是错的 — Python 标准 `json.load` 对空字符串行为是抛 JSONDecodeError 不是宽容
- 若 friction-log 路径不可写(disk full)→ format.append 抛 FormatError → silently swallow,但 last_run.log 留痕
- **schema_error_count 累加路径(R1 H3)** 必须自身有 try/except 兜底 — 若 state.json 也读不到,log 留痕但 hook 仍 exit 0,**不能让 schema-error-counter 自身的 IO 错破坏 TECH-4 不变量**
- **空 stdin 不计 schema 异常(R2 H-R2-3 显式)**:hook 在 Claude Code 早期版本可能 fire 时 stdin 是空,这是合法 noop 不是 schema drift — 用 `if not raw.strip(): return` 在 json.loads 之前 short-circuit
- **review_mode 的 hook gate 顺序(R2 B-R2-3)**:必须先看 `state.enabled`,再看 `state.review_mode`(两个独立 if · 见 architecture §5.1 Stage A 决策表);**绝不要**把 review_mode 与 enabled AND/OR 在同一行 — 那会让 set_review_mode 与 enabled 决策不可读,引起未来 worker 误改
