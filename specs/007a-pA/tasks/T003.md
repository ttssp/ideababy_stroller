## Goal(R1 medium SEC-2 修订 / R1 PIPE_BUF 措辞修订)

把"业务字段"渲染成 IN-4 单行格式,并以 POSIX `O_APPEND` 原子方式写入 friction-log(本地 APFS / ext4 实测原子;**不依赖 PIPE_BUF 语义** — architecture §5.3 措辞修订)。这是 hook + CLI 共享的"唯一写日志通道"(ADR-4),并发安全 + stderr 截断 + **secret-pattern redact(D19 / R1 SEC-2 fix)** + 转义换行,失败时 hook 路径 silently swallow / CLI 路径 fail loud(由 caller 决定;本模块只抛 `FormatError`)。

## Inputs

- `paths.friction_log_path()`(T001)
- 业务字段:`kind / confidence / tool / exit_code / stderr / task_desc / free_text`
- stdlib `os` / `datetime` / `pathlib`

## Outputs

- `tools/friction_tap/format.py` 导出:
  - `def render_agent_emit(confidence: str, tool: str, exit_code: int, stderr: str, task_desc: str) -> str` — 渲染 `[agent-emit]` 行(内部对 stderr 调 `_redact_secrets()` · D19)
  - `def render_operator_cli(free_text: str) -> str` — 渲染 `[operator-cli]` 行(内部对 free_text 也调 `_redact_secrets()` · 防 operator 在 CLI 误粘 secret)
  - `def _redact_secrets(text: str) -> str`(D19 / R1 SEC-2 fix)— private,内部用 regex 替换常见 token shape 为 `[redacted]`
  - `def append(line: str, path: Path | None = None) -> None` — 原子 append(默认走 `friction_log_path()`,可被 CLI `-f` 覆盖;v0.1 OQ-D 决议 fail loud,本参数仅给测试 fixture 使用)
  - `class FormatError(Exception)`

## Implementation plan

1. 时间戳:`datetime.now(tz=timezone.utc).isoformat(timespec="seconds").replace("+00:00", "Z")` → 例 `2026-05-08T13:30:00Z`(tech-stack §1)
2. stderr 处理顺序(R1 SEC-2 fix):
   1. `stderr[:200]` 截断(IN-4 + SEC-2 documented 默认 200 chars)
   2. `_redact_secrets(text)` 替换 token shape(D19) — **截断后 redact**(避免半截 token regex 命中后留前缀)
   3. `text.replace("\n", "↵").replace("\r", "↵")`(IN-4 转义换行)
3. **`_redact_secrets(text)` 实装(D19 · R1 SEC-2)**:
   ```python
   import re
   _TOKEN_PATTERNS = [
       re.compile(r"sk-[A-Za-z0-9_-]{16,}"),       # OpenAI / Anthropic
       re.compile(r"pk-[A-Za-z0-9_-]{16,}"),       # Stripe public key
       re.compile(r"ghp_[A-Za-z0-9]{16,}"),        # GitHub personal token
       re.compile(r"github_pat_[A-Za-z0-9_]{20,}"),# GitHub fine-grained
       re.compile(r"AKIA[A-Z0-9]{16}"),            # AWS access key
       re.compile(r"aws_[A-Za-z0-9]{16,}"),        # AWS secondary
       re.compile(r"Bearer [A-Za-z0-9_-]{20,}"),   # 通用 OAuth bearer
   ]
   def _redact_secrets(text: str) -> str:
       try:
           for pattern in _TOKEN_PATTERNS:
               text = pattern.sub("[redacted]", text)
           return text
       except Exception:
           return text  # 兜底:redact fail 不阻塞 entry 写入(SEC-2 是 best-effort,not 100%)
   ```
4. `[agent-emit]` 模板(单行,所有字段空格 · 分隔):
   ```
   <ISO> [agent-emit] [confidence:<H|M|L>] [private-to-operator] · <tool> exit <code> · "<stderr-200chars-redacted>" · task: <desc>
   ```
5. `[operator-cli]` 模板:
   ```
   <ISO> [operator-cli] [private-to-operator] · "<free_text-redacted>"
   ```
6. `append(line, path)`:
   - assert `len(line) < 4096`(**conservative engineering bound**,architecture §5.3 措辞修订:不再引用 PIPE_BUF 语义,因为 PIPE_BUF 仅 pipe/FIFO,本场景是 regular file)
   - 路径 = `path or friction_log_path()`;`path.parent.mkdir(parents=True, exist_ok=True)` 兜底
   - `fd = os.open(path, os.O_WRONLY | os.O_APPEND | os.O_CREAT, 0o644)`;`os.write(fd, (line + "\n").encode("utf-8"))`;`os.close(fd)`
   - 任何 OSError → 包装为 `FormatError(orig=e)` 抛出
7. 顶部 docstring 引用 ADR-4 / IN-4 / TECH-5(NFS 不保证)/ TECH-8(并发)/ D19 / R1 SEC-2 / R1 PIPE_BUF measure 修订

## Known gotchas

- `O_APPEND` + 单 write 原子保证在本地 APFS / ext4 / btrfs(**R1 medium PIPE_BUF 措辞修订** — 之前文档错引 PIPE_BUF,实际是 POSIX `pwrite` semantics 在 regular file 上的并发原子保证 + macOS APFS / Linux ext4 实测验证;PIPE_BUF 只适用于 pipe / FIFO);NFS / iCloud Drive 不保证(TECH-5 → README 警告)
- 不要在 hook 路径用 Python `open(... 'a')` + `print()`(可能多次 write);必须用 `os.open` + 单次 `os.write`
- `↵` 字符是 U+21B5(stdlib unicode),终端可能显示为方框;但 grep / cat 不影响
- 不在 format 层做 `[debugging-friction-detection]` 黑名单(那是 judge.py 的事,T010)
- **redact 顺序(R1 SEC-2)**:**先截断后 redact** — 截断后再扫 token,避免半截 token regex 命中后留下"半 token + [redacted]"的诡异输出
- **redact 是 best-effort**(SEC-2 mitigation 中已明示):non-standard token shape(如自定义 prefix `myco-token-XXX` 或纯 entropy 串)v0.1 不覆盖;v0.2 候选 entropy 检测
- **redact 不应抛**:try/except 兜底返回原文 — 让 secret 进 log 远比让 hook 抛错破坏 TECH-4 不变量好(operator 还能 manual edit 删 entry;hook 抛错会污染 Claude Code session)
