## Goal(R1 B1 / H4 修订)

把 hook payload 解析成 `(disposition, confidence)` 对 — 这是 v0.1 是否能在 14 天内捕到 ≥ 8 条信号的关键算法。**R1 B1 fix · 重写为两阶段**:
1. **Classification**(基于 event features,独立于 threshold):tool kind / Claude Code block flag / stderr 强信号 pattern → 决定 confidence H/M/L
2. **Threshold filter**(真过滤):`--threshold high` 仅 emit confidence ≥ H,`medium` ≥ M,`low` 全 emit

完全按 architecture §5.2 重写决策表实现:**白名单 = `{Bash, Edit, Write}`(D8 修订 R1 H4 移除 Read)** + `block: true`;黑名单 = `state.enabled=false` 或 `task_description` 含 `[debugging-friction-detection]`。**TECH-2 误报洪水的主开关由此 mitigated**(`--threshold high` 真过滤 M/L event,而非 relabel)。

## Implementation plan(完全按 architecture §5.2 R1 B1 重写)

### Stage A · Skip checks(短路)
1. `if not state.enabled` → return `("skip", None)` — D15 / ADR-5
2. `if BLACKLIST_TASK_MARKER in (payload.get("task_description") or "")` → return `("skip", None)` — D8 黑名单
3. `if payload["tool_name"] not in WHITELIST_TOOLS and not payload.get("block")` → return `("skip", None)` — D8 修订 + R1 H4(白名单外 + 非 block)

### Stage B · Confidence classification(基于 event features,顺序敏感)
内部 helper `classify(payload) -> "H" | "M" | "L" | None`:
1. `if payload.get("block") is True` → `"H"`(Claude Code lifecycle block,strong signal)
2. `tool = payload["tool_name"]; exit_code = int(payload.get("exit_code", 0)); stderr = (payload.get("stderr") or "")`
3. `if exit_code == 0` → `None`(走 fallback skip)
4. `if tool == "Bash"`:
   - 若 stderr 命中任一 `STRONG_STDERR_SIGNALS` → `"H"`(Bash 失败 + 强信号 = strong)
   - 若 `len(stderr) >= 50` → `"M"`(中等 stderr,typical signal)
   - 若 `len(stderr) < 50` → `"L"`(短 stderr,可能 `set -e` 静默退出,noisy/weak)
5. `if tool in {"Edit", "Write"}` AND `exit_code != 0` → `"H"`(写操作失败通常伴随明确路径错误)
6. fallback `"L"`(理论上 Stage A 已 skip 大部分非 expected;此处兜底)

### Stage C · Threshold filter(R1 B1 · 真过滤)
```python
confidence = classify(payload)
if confidence is None:
    return ("skip", None)
threshold_min = {"low": "L", "medium": "M", "high": "H"}[state.threshold]
order = {"L": 0, "M": 1, "H": 2}
if order[confidence] >= order[threshold_min]:
    return ("emit", confidence)
return ("skip", None)
```

## Known gotchas

- 决策表顺序敏感 — `state.enabled` 检查必须最先(避免 disabled 状态下还跑黑名单匹配)
- `block` 字段在 payload 中可能不存在(早期 Claude Code 版本)— 用 `.get("block")` 而非 `payload["block"]`,缺失等价 `False`
- threshold 值若被外部污染为非合法值 → `state.from_dict` 已校验(T002),judge 这里不再二次防御
- 不要把"exit_code == 0 但 stderr 非空"也判 emit — D3 静态规则 80% 信号的设计前提是 exit code 主驱动
- **`STRONG_STDERR_SIGNALS` 是大小写敏感的 substring match**(`"Error:"` 与 `"error:"` 不同 — 与 Python tracebacks 标准 case 一致;不要小写化以避免误命中 `"error_log"` 之类的子串)
- **threshold ordering 用整数 enum**(L=0, M=1, H=2),`>=` 比较;不用字典 key string lex 比较(`"H" > "M"` lex 错)
