## Goal(R1 H3 + R2 B-R2-3 修订)

封装唯一 config 文件 `~/.config/friction-tap/state.json` 的 read / write / default 行为。state.py 是 v0.1 三入口(hook / CLI / trust_summary)共享的"开关与阈值真理源",必须做到:首次读自动写默认值(D15 默认 `enabled: false`)、写入 atomic、字段最小且 human-readable(D12 / SEC-3)。**R1 H3 fix · 加 `schema_error_count` 字段 + `bump_schema_error_count()` helper**(D18 / TECH-9 mitigation)。**R2 B-R2-3 fix · 加 `review_mode: bool` 字段(默认 false,与 `enabled` 正交)+ `set_review_mode(value: bool)` helper**(D17 / D20 — 修复 R1 H2 review-mode-as-off-alias 与 O3 严格 binary 的设计冲突)。

## Outputs

- `tools/friction_tap/state.py` 导出:
  - `@dataclass class State: enabled: bool, review_mode: bool, threshold: Literal["low","medium","high"], installed_at: str, version: str, schema_error_count: int = 0`(R1 H3 + R2 B-R2-3 字段 · `review_mode` 与 `schema_error_count` 都默认 0/False 兼容旧 state.json)
  - `def load() -> State` 文件不存在 → 写入默认值并返回(默认 `enabled=false / review_mode=false / threshold=medium / installed_at=<now ISO UTC> / version="0.1" / schema_error_count=0`)
  - `def save(state: State) -> None` 用 tempfile + `os.replace` 原子写;写入后 `os.chmod(path, 0o600)`(SEC-3)
  - `def update(**kwargs) -> State` 读 → patch → save 的便利组合(供 CLI 控制)
  - **`def bump_schema_error_count(reason: str = "") -> int`(R1 H3 fix)**:read-modify-write,累加 +1,内部 try/except 兜所有 IO 错(若失败返回 -1,**不抛**),返回新计数;hook 调此函数累计 schema drift
  - **`def set_review_mode(value: bool) -> State`(R2 B-R2-3 fix)**:`update(review_mode=value)` 的 thin wrapper(供 CLI `--review-mode` / `--review-mode-off` 调用);**重要 invariant:不动 `state.enabled`**;hook gate 由 `state.enabled AND NOT state.review_mode` 决定(D17 / D20)
  - `class StateError(Exception)` 用于 schema 异常

## Implementation plan

1. 定义 `Threshold = Literal["low","medium","high"]`(stdlib `typing`)
2. `@dataclass` State,字段顺序 = enabled / **review_mode** / threshold / installed_at / version / **schema_error_count**(R2 B-R2-3 加 review_mode · 默认 False;R1 H3 schema_error_count 默认 0;backward-compat:旧 state.json 无 review_mode / schema_error_count 字段时 from_dict 自动填 False / 0);提供 `to_dict()` / `from_dict(d)`,后者做 isinstance 校验(架构 §5.4 + non-goals 4.2 手写校验,无 pydantic)
3. `load()`:state_path 不存在 → 调 `_default()` 构造 State 并 `save()` 后返回;存在 → `json.loads` → `from_dict`;字段缺失或类型错 → 抛 `StateError(message, hint="re-run install or delete state.json")`;**`schema_error_count` / `review_mode` 缺失视为 0 / False → 不抛**(backward-compat 关键 R1 H3 + R2 B-R2-3)
4. `save()`:写入流程 = `NamedTemporaryFile(dir=path.parent, delete=False)` → `json.dump(indent=2)` → `os.replace(tmp, path)` → `os.chmod(path, 0o600)`(原子 + 权限收紧)
5. `update()`:`s = load(); s = replace(s, **kwargs); save(s); return s`
6. **`set_review_mode(value: bool)`(R2 B-R2-3)**:`return update(review_mode=value)` thin wrapper;顶部 docstring 必须含 invariant:"**永远不动 `state.enabled`**;hook gate 行为见 architecture §5.1"
7. **`bump_schema_error_count(reason)`(R1 H3)**:
   ```python
   def bump_schema_error_count(reason: str = "") -> int:
       try:
           s = load()
           new_count = s.schema_error_count + 1
           save(replace(s, schema_error_count=new_count))
           # 同时写一行到 last_run.log 记录 reason(便于排错 TECH-9)
           with open(last_run_log_path(), "a") as f:
               f.write(f"{datetime.now(timezone.utc).isoformat()} schema_error +1 reason={reason}\n")
           return new_count
       except Exception:
           return -1  # 不抛,保 TECH-4 不变量
   ```
8. 顶部 docstring 引用 D12 / D15 / D17 / D18 / D20 / SEC-3 / ADR-4(原子写)/ R1 H3 / R2 B-R2-3

## Known gotchas

- 不要在 `load()` 里调 `update()`(死循环);默认值由 `_default()` 直接返回 + `save()` 一次性写入
- `os.chmod` 必须放在 `os.replace` 之后(否则 chmod 改的是 tempfile,replace 可能保留新文件原权限)
- `installed_at` 只在首次默认写入时设置;后续 `update()` 不应覆盖它(用 `dataclasses.replace` 时显式排除)
