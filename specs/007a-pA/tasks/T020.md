## Implementation plan(R1 修订完整)

### A · 模板(`templates/trust_w2_template.md`)

```markdown
# Trust mini-summary · day 14 · {iso_date}

## Metrics
- Total agent-emit entries: {total_agent}
- acked: {acked_count} ({acked_pct})
- disputed: {disputed_count} ({disputed_pct})
- needs-context: {needs_context_count}
- Operator CLI entries: {total_cli}

## Hook adoption(O3 — 严格 binary,R1 B3)
- state.json `enabled`: {enabled}
- O3 verdict: {o3_verdict}(仅 enabled flag 决定,不依赖 activity)

## Schema health(R1 H3 · D18)

- **解读**:K > 0 → hook 内部有 K 次 payload schema drift;若 K 显著 → Claude Code 升级可能改了 hook payload schema,看 `~/.config/friction-tap/last_run.log` traceback

## Hook Operational Status(R3/R4 H-R3-2 · D17 / D20 / D21)
- `state.review_mode = {review_mode}`(R4-cleanup H-R4-1:渲染后形如 `state.review_mode = true` / `state.review_mode = false`,与 spec §6 + T031 assert 完全一致 — 等号空格形式是 contract)
- **解读**:review_mode=true 时 hook 在 review session 期间静默暂停(architecture §5.1 Stage A1b);O3 verdict 与 review_mode 正交(D20),review_mode=true 不破 O3
{review_mode_warning}

## Outcome check
- O1 ≥ 8: {o1_verdict}(本次 N={total_agent})
- O2 acked ≥ 50%: {o2_acked_verdict}(本次 acked={acked_pct})
- O2 disputed ≤ 20%: {o2_disp_verdict}(本次 disputed={disputed_pct})

## Self-interview(R1 H1 · 结构化必填)
```

### B · `main()` 步骤(默认无 flag 路径)

1. argparse:`--check-enabled` 与 `--check-self-interview` 互斥;无 flag → full report
2. `state = state_module.load()`
3. `total_agent = grep.count_since(timedelta(days=14), "agent-emit")`
4. `total_cli = grep.count_since(timedelta(days=14), "operator-cli")`
5. `ratios = grep.tag_ratios()`
6. `recent = grep.has_recent_agent_emit()` — 仅 informational(R1 B3)
7. `o3_verdict = "pass" if state.enabled else "fail"`(**R1 B3 严格 binary** — 不再 `state.enabled and recent`)
8. `o1 = total_agent >= 8;o2_acked = ratios.get("acked", 0) >= 0.50;o2_disp = ratios.get("disputed", 0) <= 0.20`
9. **R3/R4 H-R3-2 · review_mode 状态与 stuck-true WARNING 计算**:
   - `review_mode_str = "true" if state.review_mode else "false"`
   - 若 state.review_mode == True AND state.review_mode_set_at(architecture §5.4 加 field;backward-compat 缺失 → None)not None:
     - `delta = now - parse_iso(state.review_mode_set_at)`
     - 若 `delta > timedelta(hours=24)` → `review_mode_warning = "- **WARNING**: review_mode has been on for >24h ({delta_h}h); hook is paused. Run \`friction --review-mode-off\` if review session is over."`
     - 否则 → `review_mode_warning = ""`(空字符串,模板渲染该行隐藏)
   - 若 state.review_mode == False → `review_mode_warning = ""`
10. 渲染 `template.format(...)` → 完整 markdown 字符串(`{schema_error_count}` / `{review_mode}` / `{review_mode_warning}` 字段从 state 取)
11. **结构化 prompt 防 drift**:assert 末尾含 `[label:` 与 `Reason:` 行;缺 → RuntimeError
12. **`## Hook Operational Status` 段防 drift**:assert 渲染后 markdown 含字面串 `state.review_mode = ` (注意:**等号空格形式**,R4-cleanup H-R4-1 — 与 spec §6 + T031 assert 完全一致);缺 → RuntimeError
13. 写入路径(防覆盖,见 step C)
14. exit code 按 outcome 设定

### D · `--check-self-interview` flag(R1 B2 + R1 H1 / O6 verification)
1. 读 `paths.trust_w2_path()`;若不存在 → exit 3 + stderr 提示
2. 找 `[label:` 行;提取 label 值(strip 空格);若不在 `{"relieved","watched","both"}` → exit 1
3. 找 `Reason:` 行(及之后到下一标题的所有自由文本);算 CJK 字符数(`r"[一-鿿]"`)与 English word 数(`re.findall(r"[A-Za-z]+", text)`)
4. 任一阈值满足(CJK ≥ 20 OR English words ≥ 8)→ exit 0;否则 → exit 2 + stderr `reason too short: <X CJK> / <Y English words>`

## Known gotchas

- 模板 `{}` 与 markdown 表格的 `|` 字符可能冲突;用 `str.format()` 时注意 escape `{{` / `}}`
- `tag_ratios` 返回 `total_agent_emit` 可能是 0(分母);模板渲染时除以 0 → 用 `format` 之前先把 `acked_pct` 等算成字符串(`"N/A"` 或 `"0.0%"`)
- `trust_w2.md` 路径若 docs/dogfood/ 不存在 → paths.py 已 mkdir(T001);本 task 直接写
- launchd 触发的 process cwd 不一定是 repo — `paths.repo_root()` 仍依赖 `git rev-parse`,launchd plist 必须设置 `WorkingDirectory`(T021 负责)
- **R1 H1 self-interview 校验**:`--check-self-interview` 是离线校验已存在的文件,不依赖 state;operator 在 day-14 后任何时间都可重跑该 flag 验证答案
- **R1 B3 严格 binary**:O3 现仅 state.enabled,**绝对不要**让脚本回到旧 D7 双指标 — 那会让 BUS-1 操作员暂离场景产生 false fail
- CJK 字符 regex 使用 `[一-鿿]`(基本 CJK 块),不含扩展 A/B(后续 v0.2 可扩);English word 用空格 + 字母分词,不计标点
