## Goal(R1 H2 + R2 B-R2-3 修订)

实现 operator 唯一主动入口:`friction "<msg>"`(append subjective entry · IN-6)+ `friction --on / --off / --threshold {low,medium,high}`(state 控制 · IN-7)+ **`friction --review-mode` / `--review-mode-on`(D17 + D20 / R2 B-R2-3:设 `state.review_mode=true`,**与 enabled 正交,O3 不受影响**)+ `friction --review-mode-off`(退出 review_mode)**。CLI 必须 fail loud(D6:cwd 不在仓 → 显式报错让 operator `friction -f <path>`),不 fallback 到隐藏路径,与 hook 的 silently-swallow 形成对称(privacy 与 friction-tap 的 dual-voice 写法)。

**R2 B-R2-3 关键变更**:R1 原方案让 `--review-mode` 等价 `--off`(都设 enabled=false)— 但这会让 review session 期间触发 trust_summary `--check-enabled` 的 false fail。新方案:`--review-mode` 调 `state.set_review_mode(True)`(**不动 enabled**),`--review-mode-off` 调 `set_review_mode(False)`。hook gate 由 `enabled AND NOT review_mode` 决定(architecture §5.1 / D17 / D20)。

## Implementation plan

1. argparse:主 parser 含 flag `--on / --off / --review-mode / --review-mode-on / --review-mode-off / --threshold / --status / --version / -f` + 默认位置参数 `msg`(用 `nargs="?"` + 后置判定让 `friction "msg"` 直接写);`--review-mode` / `--review-mode-on` 是同义 alias(都设 review_mode=true),`--review-mode-off` 是退出
2. **`friction --on`(R3/R4 H-R3-1 / D21 修订 · 完整恢复语义)**:
   - 读 pre-state(`prev = state.load()`)
   - `update(enabled=True)`
   - 若 `prev.review_mode == True` → `set_review_mode(False)` + 打印 `friction-tap enabled · review_mode cleared · log: <path>`(显式告知 operator review_mode 也被清,防 `--on` 后 hook 仍卡死静默暂停 + O3 PASS 的 false adoption 状态)
   - 若 `prev.review_mode == False` → 仅打印 `friction-tap enabled · log: <path>`(无重复噪音)
   - 若 `repo_root()` 抛错 → 提示 `--on doesn't write log; OK to enable from anywhere` 仍容忍仓外
   - **invariant**:`--on` 完成后 state.json 必有 `enabled=true AND review_mode=false`(双字段一致)— 这是 D21 防卡死的核心
3. `friction --off`:`update(enabled=False)` + 打印 `friction-tap disabled`(**注意**:`--off` 不清 review_mode — `--off` 只关 enabled,review_mode 保持 — 与 `--on` 完整恢复非对称,operator 应用 `--review-mode-off` 单独清 review_mode)
4. **`friction --review-mode` / `--review-mode-on`(D17 / D20 · R2 B-R2-3)**:`state.set_review_mode(True)`(thin wrapper to `update(review_mode=True)`)+ 打印 `review mode active · hook suppressed during file edit · enabled={state.enabled} unchanged · run friction --review-mode-off when done`(**显式告诉 operator enabled 状态没变**);**测试断言**:跑 `--review-mode` 后 `state.enabled` 与跑前完全相同(invariant)
5. **`friction --review-mode-off`(R2 B-R2-3)**:`state.set_review_mode(False)` + 打印 `review mode off · hook resumed (enabled={state.enabled})`;同样不动 enabled
6. `friction --threshold high`:`update(threshold="high")` + 打印当前 threshold
7. `friction "msg"`:
   - 调 `friction_log_path()`(`-f` override 优先)— `NotInRepoError` 时 fail loud:stderr 输出 `error: not in a git repo. specify -f <path> or cd into repo.` exit 1
   - 调 `format.render_operator_cli(msg)`(内部走 D19 redact)+ `format.append(line, path=<resolved>)` — `FormatError` 抛 → exit 3
   - stdout 输出 `recorded · <path>`
8. `friction --status`:dump state JSON(human-readable 缩进,**含 `review_mode` R2 B-R2-3 + `schema_error_count` R1 H3**)+ 加一行"log path: <path or 'not in repo'>"
9. 顶部 docstring 引用 IN-6 / IN-7 / D6 / D15 / D17 / D20 / C14 / R2 B-R2-3;README 由 T032 写

## Known gotchas

- argparse 同时含 subparsers 与位置参数有歧义 → 用 `parser.parse_known_args()` + 手工分流;或主 parser 设 subparsers `required=False`,主 parser 含位置参数 msg
- `-f` flag 仅作用于本次 `friction "msg"`,不持久化(避免 OPS-4 多 dogfood 周期混路径)
- `--on` 可在仓外执行(未来 dogfood 起点仪式 — 仓内 `friction --on` 也行,但仓外不阻塞)
- 不要用 `print(..., file=sys.stderr)` 区分错误等级 — argparse 已带 `parser.error()` 直接 exit 2;手工 `sys.exit(1)` 用于 D6 fail-loud
- CLI 返回值要稳定:成功 0,所有 fail 用区分 exit code(便于 README 教学)
