## Goal

实现 D9 决议:用 macOS launchd(非 cron)在 install_at + 14d 那天 09:00 单次触发 trust_summary.py。本 task 包含 plist 模板 + install / uninstall bash 脚本 + 安装后 `launchctl print` 验证(TECH-6 mitigation)。

## Inputs

- 已实现的 `trust_summary.py`(T020)
- `state.load().installed_at`(T002,作为 +14d 锚点)
- macOS launchd `~/Library/LaunchAgents/`

## Outputs

- `templates/launchd.plist.template` — 含 `{python_bin}` / `{repo_root}` / `{trigger_year}` / `{trigger_month}` / `{trigger_day}` 占位符
- `scripts/install_launchd.sh` — 计算 install_at + 14d → 渲染 plist → `launchctl bootstrap` → `launchctl print` 验证
- `scripts/uninstall_launchd.sh` — `launchctl bootout` + 删除 plist 文件
- 副作用:`~/Library/LaunchAgents/dev.idsmaintainer.friction-tap.trust-w2.plist` 创建(install)或删除(uninstall)
- exit codes:install 成功 0 / launchctl bootstrap 失败 1 / 模板渲染失败 2

## Implementation plan

1. `launchd.plist.template` 内容(标准 macOS plist · D9):
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
   <plist version="1.0">
   <dict>
     <key>Label</key><string>dev.idsmaintainer.friction-tap.trust-w2</string>
     <key>ProgramArguments</key>
     <array>
       <string>{python_bin}</string>
       <string>-m</string>
       <string>friction_tap.trust_summary</string>
     </array>
     <key>WorkingDirectory</key><string>{repo_root}</string>
     <key>StartCalendarInterval</key>
     <dict>
       <key>Year</key><integer>{trigger_year}</integer>
       <key>Month</key><integer>{trigger_month}</integer>
       <key>Day</key><integer>{trigger_day}</integer>
       <key>Hour</key><integer>9</integer>
       <key>Minute</key><integer>0</integer>
     </dict>
     <key>RunAtLoad</key><false/>
     <key>StandardOutPath</key><string>{home}/.config/friction-tap/launchd-stdout.log</string>
     <key>StandardErrorPath</key><string>{home}/.config/friction-tap/launchd-stderr.log</string>
     <key>EnvironmentVariables</key>
     <dict>
       <key>PYTHONPATH</key><string>{repo_root}/tools</string>
     </dict>
   </dict>
   </plist>
   ```
2. `install_launchd.sh`(bash 3.2 兼容):
   1. `installed_at = python -c "from friction_tap.state import load; print(load().installed_at)"` → ISO 时间
   2. `trigger = python -c "from datetime import datetime,timedelta; t=datetime.fromisoformat('$installed_at'.replace('Z','+00:00'))+timedelta(days=14); print(t.year,t.month,t.day)"` 拆出 year/month/day
   3. `python_bin = $(command -v python3)`(macOS 默认 `/usr/bin/python3`)
   4. `repo_root = $(git rev-parse --show-toplevel)`
   5. 用 `sed` 替换 template 占位符 → 写到 `~/Library/LaunchAgents/dev.idsmaintainer.friction-tap.trust-w2.plist`
   6. `launchctl bootstrap gui/$(id -u) <plist>`(新 API,macOS 14+)
   7. `launchctl print gui/$(id -u)/dev.idsmaintainer.friction-tap.trust-w2` → 检查输出含 `state = scheduled` 或 `lastexitcode = 0`(TECH-6 验证)
   8. 失败 → exit 1,提示 `manual fallback: python -m friction_tap.trust_summary` 在 day-14 那天手动跑(TECH-6 mitigation)
3. `uninstall_launchd.sh`:`launchctl bootout gui/$(id -u)/dev.idsmaintainer.friction-tap.trust-w2`(若 label 不存在容忍)+ `rm -f ~/Library/LaunchAgents/dev.idsmaintainer.friction-tap.trust-w2.plist`。
4. 顶部注释引用 D9 / TECH-6 / Tradeoff-B / OPS-1。

## Known gotchas

- `launchctl load` 已废弃(macOS 11+)— 必须用 `launchctl bootstrap` / `launchctl bootout` API
- `gui/$(id -u)` 是 user agent domain;`system/` 是 system-level(本场景必须 user agent)
- 新 macOS 上首次安装 user agent 可能弹"Background Items added" 通知 — README 提示这是正常
- `StartCalendarInterval` 单次触发 = 设置具体 Year+Month+Day(若只设 Hour/Minute 会每天触发)
- 若 install_at 已是 14 天前(operator 安装拖延后再装 launchd)→ trigger 已过期,launchd 不会触发;install_launchd.sh 应检测此情况并 stderr 警告 `trigger date already past, run trust_summary manually`
- `WorkingDirectory` 必须设为 repo_root(否则 trust_summary 内 `git rev-parse` 失败)
