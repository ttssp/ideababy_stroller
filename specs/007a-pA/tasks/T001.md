## Goal

提供"探测 repo-root + 计算 friction-log 绝对路径"的 stdlib-only helper。这是 v0.1 privacy promise(D4 hardcoded path · ADR-2)的物理载体:CLI / hook / trust_summary 都通过 paths.py 拿到唯一的 log path,任何"cwd 不在 git 仓"的边界都在此模块内显式 fail(OQ-D 决议 D6)。

## Inputs

- `git rev-parse --show-toplevel` 子进程结果(stdlib `subprocess`)
- 环境变量 `HOME`(stdlib `os.path.expanduser`)
- 无上游 task

## Outputs

- `tools/friction_tap/__init__.py` — 空包占位 + `__version__ = "0.1.0"`
- `tools/friction_tap/paths.py` — 导出三个函数:
  - `repo_root() -> Path` 探测当前 cwd 所属 git 仓库的 toplevel,失败抛 `NotInRepoError`
  - `friction_log_path() -> Path` 返回 `<repo_root>/docs/dogfood/v4-friction-log.md`(D4 硬编码)
  - `trust_w2_path() -> Path` 返回 `<repo_root>/docs/dogfood/v4-trust-w2.md`
  - `state_path() -> Path` 返回 `~/.config/friction-tap/state.json`(D12)
  - `last_run_log_path() -> Path` 返回 `~/.config/friction-tap/last_run.log`(architecture §6)

## Implementation plan

1. 定义 `NotInRepoError(Exception)` — 顶层 sentinel,带 `suggestion: str` 字段(默认 `"specify -f path or run inside a git repo"`)。
2. `repo_root()`:`subprocess.run(["git", "rev-parse", "--show-toplevel"], check=False, capture_output=True, text=True)`;非零 returncode → 抛 `NotInRepoError`;成功 → `Path(output.strip())`。
3. `friction_log_path()` / `trust_w2_path()` 调 `repo_root()` 拼路径;`mkdir(parents=True, exist_ok=True)` 保证 `docs/dogfood/` 存在(IN-3)。
4. `state_path()` / `last_run_log_path()`:`Path.home() / ".config" / "friction-tap" / <name>`;同样 `mkdir(parents=True, exist_ok=True)`。
5. 文件顶部加中文 docstring 描述 D4 / D6 / D12 / ADR-2 立场。

## Known gotchas

- `git rev-parse` 在 git submodule / worktree 场景会返回 worktree-local toplevel(非 superproject)— 这是 git 标准行为,符合 architecture §6 意图
- 不要 cache `repo_root()` 结果(operator 可能 cd 切仓);每次调用都查
- 不在此 task 实现 mkdir 锁(D4 路径首次创建容忍并发,反正 hook 是 fire-and-forget)
