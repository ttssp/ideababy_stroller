## Goal

把 Vitest（unit / integration）与 Playwright（E2E）两套测试 runner 的配置定死。Vitest 支持临时 Postgres schema（供 T003/T006 测试数据库交互），Playwright 支持启动 dev server + 登录辅助 helper（供 T032 E2E 使用）。单独暴露 `/api/healthz` endpoint 供 smoke test 验证 harness 可跑。

## Implementation plan

1. 安装 `vitest@^3`, `@playwright/test@^1.48`（已在 T002 devDeps）。
2. `vitest.config.ts`：支持 `@/` alias；`pool: 'threads'`；`testTimeout: 30000`。
3. `playwright.config.ts`：单 project；`use.trace = 'on-first-retry'`；`reporter: [['list'], ['html', { open: 'never' }]]`。
4. `vitest-setup.ts` 导出 `withTempSchema`：连 Postgres、`CREATE SCHEMA test_<rand>`、run migrations in that schema、call fn、`DROP SCHEMA CASCADE`。
5. `playwright-global-setup.ts` 登入 admin、保存 cookie 到 `.playwright/admin-state.json`；后续 spec 用 `storageState` 加载。
6. `/api/healthz` route handler：简单 return Response JSON。
7. 在 `package.json` 补 `test`, `test:e2e`, `test:coverage` scripts。

## Known gotchas

- Playwright `webServer.reuseExistingServer: true` 在 CI 里要 false；使用 `!process.env.CI` 条件
- Vitest `v8` coverage 与 Next 15 配合偶有 sourcemap 错位；可退化到 `istanbul` 如果影响调试
- `withTempSchema` 的 Postgres CREATE SCHEMA 要求 user 有 `CREATEDB` 权限，dev 环境给 superuser；prod 不会跑这类测试
- Playwright 浏览器下载在 CI 第一次会较慢（~5 min）；`cache: pnpm` + `node_modules` cache 不足以缓存浏览器二进制，需单独缓存 `~/.cache/ms-playwright`
- `playwright-global-setup.ts` 不能 import Next 15 的 server-only API（会跨 runtime 爆）；只能调 HTTP
