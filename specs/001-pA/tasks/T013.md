## Goal

把 T004 adapter 的 `provider.summarize()` 封装为 **"一批 (paper × topic) 对 → SummaryRecord → 持久化到 `paper_summaries`"** pipeline：

```
runSummaryPass 流程（per (paper, topic) 对）：
  1. record = await provider.summarize({ paper, topic })
     // adapter 内部已 INSERT llm_calls 并把 id 填到 record.llmCallId（T004）
     // adapter 内部也做 truncateTo3Sentences（G1 count-terminator 实现）
  2. persistSummary({ paperId, topicId, record })
     // 本 task 负责：INSERT INTO paper_summaries 一行（FK → record.llmCallId）
     // ON CONFLICT (paper_id, topic_id, prompt_version) DO UPDATE
  3. catch LLMProviderError / LLMBudgetExceededError / DB CHECK violation:
     // log to fetch_runs.notes + skip this pair + continue
```

**G2 关键纠正（2026-04-24 R_final B2 fix · 权威真源）**：
- **Adapter 写 `llm_calls`**（T004 内部 `recordLLMCall`），**T013 不写 `llm_calls`**
- **T013 只写 `paper_summaries`**，用 `record.llmCallId` 作为 FK 值
- 列名权威（与 `reference/schema.sql §13` / `§14` 对齐）：
  - `llm_calls`：`purpose` / `called_at` / `cost_cents` / `request_hash`（**无 `kind` · 无 `created_at`**）
  - `paper_summaries`：`summary_text` / `llm_call_id` / `model_name` / `prompt_version`
- OpenAI output pricing = **$15/M**（不是 $10/M）
- SummaryRecord 字段 camelCase：`summaryText` / `promptVersion` / `modelName` / `llmCallId` / `inputTokens` / `outputTokens` / `latencyMs` / `truncated` / `requestHash`

3. **persist.ts**（~60 min · 业务关键）
   ```ts
   export async function persistSummary({ paperId, topicId, record }): Promise<{ paperSummaryId: number }> {
     try {
       const [row] = await dbWorker
         .insert(paperSummaries)
         .values({
           paperId,
           topicId,
           summaryText: record.summaryText,   // camelCase in TS · Drizzle maps to summary_text
           llmCallId: record.llmCallId ?? null, // adapter 已写 llm_calls；fallback 路径 null
           modelName: record.modelName,
           promptVersion: record.promptVersion,
         })
         .onConflictDoUpdate({
           target: [paperSummaries.paperId, paperSummaries.topicId, paperSummaries.promptVersion],
           set: {
             summaryText: record.summaryText,
             llmCallId: record.llmCallId ?? null,
             modelName: record.modelName,
           },
         })
         .returning({ id: paperSummaries.id });
       if (!row) throw new Error('paper_summaries upsert returned empty');
       return { paperSummaryId: row.id };
     } catch (err) {
       // Postgres CHECK violation (summary_sentence_cap) surfaces as error
       // with code 23514; in theory truncateTo3Sentences should prevent this,
       // but if the adapter is bypassed or mocked wrong, log and re-throw so
       // caller's catch in runSummaryPass can skip this pair cleanly.
       console.error('[RED-LINE-2-DB-REJECT]', { paperId, topicId, err });
       throw err;
     }
   }
   ```
   - **本函数不写 `llm_calls`**（adapter 的职责 · ADR-6 + T004 契约）
4. **pass.ts**（~90 min）
   ```ts
   export async function runSummaryPass(
     pairs: Array<{ paperId: number; topicId: number }>,
     provider: LLMProvider,
   ): Promise<SummaryPassResult> {
     const rows: SummaryPersistResult[] = [];
     let capped = false;
     let budgetHitAt: number | null = null;

     for (let i = 0; i < pairs.length; i++) {
       if (i > 0 && i % 10 === 0) {
         const monthly = await currentMonthCostCents();
         if (monthly > MONTHLY_BUDGET_CENTS) {
           capped = true;
           budgetHitAt = pairs[i].paperId;
           break;
         }
         if (monthly > WARN_THRESHOLD_CENTS) {
           console.warn(`[COST-WARN] llm_calls monthly cumulative=${monthly} cents > threshold ${WARN_THRESHOLD_CENTS}`);
         }
       }
       const { paperId, topicId } = pairs[i];
       try {
         const paper = await loadPaper(paperId);
         const topic = await loadTopic(topicId);
         const record = await callWithFallback((p) => p.summarize({ paper, topic }));
         // adapter 内部已写 llm_calls · record.llmCallId 已填
         const { paperSummaryId } = await persistSummary({ paperId, topicId, record });
         rows.push({
           paperId, topicId, paperSummaryId,
           llmCallId: record.llmCallId ?? null,
           modelName: record.modelName,
           promptVersion: record.promptVersion,
           truncated: record.truncated,
         });
       } catch (err) {
         if (err instanceof LLMBudgetExceededError) {
           capped = true;
           budgetHitAt = paperId;
           break;
         }
         // Generic skip: LLMProviderError / ZodError / DB CHECK violation
         console.warn('[SUMMARY-PASS-SKIP]', { paperId, topicId, err });
         await noteFetchRunError({ paperId, topicId, err });
         continue;
       }
     }
     return { rows, capped, budgetHitAt };
   }
   ```
5. **Fallback 路径**（~20 min · TECH-1 mitigation）

## Known gotchas

- **成本是累积而非单次**：查询必须限 `called_at >= date_trunc('month', now())` 避免上月数据算进来
- **Pricing 唯一入口**：`tech-stack.md §2` / `llm-adapter-skeleton.md §3 / §4` 的 `calcCostAnthropicCents` / `calcCostOpenAICents` 已内置 `rate / 1_000_000 * tokens`；本 task **不再**重新计算；`persistSummary` 只消费 `record.llmCallId`（adapter 已写）
- **OpenAI output $15/M**（G2 · 2026-04-24 R_final B2 fix）：之前旧版本（`tech-stack.md §2.2` 初版）写 $10/M；权威值 $15/M 已在 T004 `calcCostOpenAICents` 落实；本 task 任何测试断言都基于 $15/M · 月度估 ≈ $12（仍在 C11 envelope $50 内）
- **`llm_calls` 列名权威**（schema.sql §13）：`purpose`（不是 `kind`）· `called_at`（不是 `created_at`）· `cost_cents` · `request_hash` · `input_tokens`/`output_tokens`；**任何其他列名 = PR 打回**
- **T001 fallback 路径**：spike 结果若 < 70%，operator 签字仅用 summary（judge disabled）；summary 路径仍可用。若 summarize 也全挂（provider 全局故障），T013 降级为 `enableHeuristicFallback=true`，用 heuristic 模板写 `paper_summaries`（`llm_call_id=null` · F5 策略 allow null）
- **adapter 写 `llm_calls` · 本 task 写 `paper_summaries`**：分工严格，避免双写重复 audit（ADR-6 + T004 契约）
- **一篇 paper 被多 topic 匹配**：会调 `summarize` 多次（同 paper × 不同 topic 产出不同 summary） — 这是 ADR-3 per-topic 判定的代价；成本 cap 需要把这考虑进去（worker 预算检查按 pair-count 算）
