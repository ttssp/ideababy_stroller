## Goal

spec §5 Phase 1 T017 "Action traceability page"：展示某 paper 的**全部 action 历史 + why**（per-seat 汇总，admin 可看全 lab；member 仅看自己+匿名统计）。让 PI 回看 "这篇论文过去几个月被我和 operator 怎么处理过"——是 O4 "漏看案例" 自陈的证据基础。刻意简朴：表格一张，时间倒序。

## Inputs

- `specs/001-pA/spec.md` §5 Phase 1 T017 描述、O4 verification（self-report 要截图）
- `specs/001-pA/architecture.md` §5.1（actions + breadcrumbs + resurface_events schema）
- T003 产出：`actions`、`breadcrumbs`、`resurface_events` schema
- T006 产出：`requireAuth`（admin 与 member 分支）
- T015 产出：`/today` 页面上 paper 链接到此页

## Outputs

- `src/lib/papers/history.ts` — 导出 `getPaperHistory({ paper_id, viewer_seat_id, viewer_role })`：
  - admin: 查所有 `actions` + 所有 `breadcrumbs` + 所有 `resurface_events.click/dismiss`，带 seat_email
  - member: 只查自己的 actions + 全 lab 的 匿名聚合（`read_now` 总次数、`skip` 总次数）
- `src/app/(main)/papers/[id]/history/page.tsx` — server component：
  - 顶部 paper 基本信息 `{ title, abstract, authors, arxiv_id, arxiv_url }`
  - 中部当前 paper 的 summary（**R1 变更**：来自 `paper_summaries`，按 topic 聚合展示；若一 paper 对应多 topic，逐 topic 列出 summary）
  - 下部历史表格：时间 / seat / action / why 或 "—"
  - **R1 新增（B2 展示）**：`action='skip'` 的行 `why` 列**必须**使用加粗 + 独立列宽显式展示（≥ 5 字符保证存在），**不**隐藏到 hover/tooltip；这是红线 2 追溯的 UI 兑现
  - member 视图里 seat 列改为 "自己" 或 "其他（匿名）"
- `tests/lib/papers/history.test.ts` — admin 看到全 lab、member 只看自己；排序倒序；resurface 事件混入时间线；**R1 新增**：skip action 的 why 被渲染为 `<strong>` 或等价 visible element（不在 title / hover）

## Implementation plan

1. `history.ts`：三次 SELECT → UNION ALL 后在应用层 sort by `event_time` DESC（events 数量小 ≤ few dozen rows，无需 SQL sort）。
2. 统一 event type：`{ kind: 'action'|'breadcrumb_created'|'resurface_clicked'|'resurface_dismissed', time, seat_id, seat_email?, detail: { action?, why?, trigger_type?, context_text? } }`
3. `/history/page.tsx`：表格 `<table>` + tailwind；每行 highlight kind 颜色（action = 蓝色、breadcrumb = 黄色、resurface = 紫色）。
4. member 的匿名化：仅展示 seat_email 的首字母 + 星号（e.g. `c***@l***`）或固定 "lab 成员"；自己的行显示邮箱原样。
5. URL：`/papers/<numeric-id>/history`；paper 不存在 → 404。

## Known gotchas

- resurface_events 的 clicked_at / dismissed_at 是不同列（同行可能同时非空）；在 event timeline 里当作两条 event 展开
- 一篇 paper 可能没有 summary（未被 briefing 覆盖）；此时展示 "尚无 LLM 摘要"
- member 匿名化必须彻底：不要在 inspect page DOM 里暴露真实 seat_id（审计需求 O3/O4 不需要 member 看到其他 seat_id）
- paper 的 `arxiv_url` = `https://arxiv.org/abs/<arxiv_id>`；拼接即可，无需存库
- 该页面是 **read-only**；不写任何表
- **R1 新增（B1）**：multi-topic 场景下 summary 按 `paper_summaries.topic_id` 分组；同 topic 多 prompt_version 取 latest（max created_at）
- **R1 新增（B2 · 红线 2 可见性）**：skip.why 的展示必须**显式** —— operator 60 天后想回忆"我当时为什么 skip 这篇"时必须一眼可见；若未来有人把 why 藏进 tooltip 作为"UI 优化"，本 task 的 verification 会挂
