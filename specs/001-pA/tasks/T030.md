## Goal

落地单 VPS 部署流水线：`provision.sh` 一键创建 `webapp_user` / `worker_user` / database、`deploy.sh` pull + build + `systemctl restart`、Caddy 反代 + 自动 TLS、`pg_dump` daily timer（RPO ≤ 24h）。本 task 产出 **shell 脚本 + pg-dump systemd unit + Caddyfile**，操作员在 VPS 上按 `deploy/README.md` 一次性 provision 即可。

**G4 H3 职责划分（2026-04-24 R_final）**：
- **T007 owns**：web / worker systemd unit **文件本身**（`deploy/systemd/pi-briefing-web.service` / `pi-briefing-worker.service` / `pi-briefing-worker.timer`）· 由 T007 task 在 `deploy/systemd/` 创建
- **T030 owns**：
  - `deploy/systemd/pg-dump.{service,timer}`（新增 · 备份专用 · 不在 T007 域）
  - **部署时安装 / 启用 / 验证** T007 的 systemd unit 文件（`cp` 到 `/etc/systemd/system/` + `systemctl daemon-reload` + `systemctl enable`），但**不创建** `pi-briefing-*.service` 源文件

## Implementation plan

1. `provision.sh` 为幂等（使用 `set -e` + `apt install -y`；`systemctl enable` 安全可重复）
2. `init.sql` 按 architecture §7 定义两个 Postgres 角色 + 弱化 pg_hba.conf（另外在 README 提醒改 `local all all scram-sha-256`）
3. `grants.sql` 按 architecture §7 原则；写好后 `psql -U postgres -d pi_briefing -f grants.sql`
4. Caddyfile 尽量短：reverse_proxy 3000 + gzip；**不引入** auth 层 at Caddy；auth 在 Next middleware 解决（T006）
5. `pg-dump.timer` 05:00 早于 worker 06:00 确保 snapshot 是昨日的 final state（worker 跑完会改 DB）
6. `deploy.sh` 的 drizzle migrate 要用 `DATABASE_URL` 指向 webapp_user；确保 migration 权限（可能需要临时切 superuser：`SET ROLE postgres; ... RESET ROLE;`）

## Known gotchas

- Caddy auto-TLS 要求 ≤ 80/443 端口开放 + DNS 解析到 VPS；VPS 刚 provision 时域名可能还没 propagate → 用 `tls internal` 跑开发快照
- Postgres 16 在 Ubuntu 24.04 默认 monitoring 关闭；不要启用 pg_stat_statements（NG-B3）
- systemd `LoadCredential` 路径必须 absolute；provision.sh 要 `mkdir -p /etc/pi-briefing/credentials && chmod 700`
- `pg_dump -Fc` 产出 custom format，便于 `pg_restore` 选择性恢复；不要用 `-Fp` plain SQL（更占空间、无压缩）
- `deploy.sh` pull 代码时要 `git stash` 保护本地改动不然会 refuse；本任务默认 operator 不在 VPS 上改代码
- web 与 worker 的 `NODE_ENV=production` 必须设；否则 Next 15 跑 dev mode 严重影响性能（SLA §1.2 p95 保证不了）
- **G4 H3 · DB role 不是 OS user**：`webapp_user` / `worker_user` 是 Postgres 的登录角色，**不是** Linux 用户；任何 `sudo -u webapp_user psql ...` 都会失败（Linux 无此 OS user）；正确方式是 `PGUSER=webapp_user PGPASSWORD=<pw> psql ...`（或 `~/.pgpass`）
- **G4 H3 · systemd unit 源文件分工**：`pi-briefing-web/worker.{service,timer}` 的**文件内容**由 T007 产出（`deploy/systemd/`）；T030 只负责把这些文件**安装**到 `/etc/systemd/system/` 并 `systemctl enable`。`pg-dump.{service,timer}` 文件内容本 task 产出
