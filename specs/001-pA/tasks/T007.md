## Implementation plan

1. `src/env.ts`：zod schema 定义权威 env var 清单（与 `reference/directory-layout.md §3` 逐字对齐；provider enum 是 `anthropic|openai`，不是 `claude|gpt`）：
   - `NODE_ENV` (`development | production | test`)
   - `APP_ORIGIN` (url)
   - `APP_TIMEZONE` (string · default `Asia/Shanghai`)
   - `DATABASE_URL` (url · webapp_user)
   - `DATABASE_URL_WORKER` (url · worker_user)
   - `DATABASE_POOL_WEBAPP` (number · default 5)
   - `DATABASE_POOL_WORKER` (number · default 2)
   - `LLM_PROVIDER` (enum `anthropic | openai` · **不是 `claude | gpt`** · 与 `.env.example` + `reference/skeletons/llm-types.ts` 的 `LLMProviderName` 一致)
   - `LLM_FALLBACK_PROVIDER` (optional enum `anthropic | openai`)
   - `ANTHROPIC_API_KEY` (optional · LLM_PROVIDER/LLM_FALLBACK_PROVIDER 选中 `anthropic` 时必需 · zod `refine`)
   - `ANTHROPIC_MODEL` (string · default `claude-sonnet-4-6-20250701`)
   - `OPENAI_API_KEY` (optional · 同 refine · openai 时必需)
   - `OPENAI_MODEL` (string · default `gpt-5.4-turbo`)
   - `LLM_MONTHLY_COST_USD_CAP` (number · default 50 · C11 envelope)
   - `ARXIV_API_BASE` (url)
   - `ARXIV_RATE_LIMIT_MS` (number · default 3000)
   - `SESSION_SECRET` (string · min 32)
   - `INVITE_TOKEN_TTL_HOURS` (number · default 24)
   - `SMTP_HOST`, `SMTP_PORT` (number), `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM` (all optional · v0.1 invite 不发邮件)
   - `WORKER_DAILY_TIME` (string `HH:MM` · default `06:00`)
   - `WORKER_CATCHUP_WINDOW_HOURS` (number · default 24)
   - `LAB_DEFAULT_NAME` (string)
   - `ADMIN_EMAIL` (string · T003 seed)
2. Export `export const env = schema.parse(process.env)`；模块顶层 throw → fail-fast。
3. `.env.example`：逐 key 注释 + 示例值（假值）；加 header `# 绝不 commit 带真实值的 .env`。
4. systemd unit 文件按 architecture.md §9 + SEC-3；`LoadCredential` 从 `/etc/pi-briefing/credentials/anthropic` 读；service 内通过 `${CREDENTIALS_DIRECTORY}/anthropic` 消费。
5. worker timer：`Persistent=true` 必须（errata for OPS-2）；`RandomizedDelaySec=0` 保证 06:00 准时。
6. `validate-env.ts` 就 `import '@/env'`，错误即 exit；CI 在 install 后跑一次保证 `.env.example` 没缺 key。
7. `tests/env.test.ts` 用 `beforeEach` 清 `process.env` → set mock → import module（用 `vi.resetModules()`）。

## Known gotchas

- zod 的 `z.string().email()` 在 Node 22 有轻微正则差异；建议 `z.string().regex(/^[^@]+@[^@]+$/)` 更可移植
- systemd `LoadCredential=name:/path/to/file` 路径必须绝对路径；若相对则 service 启动时静默跳过导致 API key 为 undefined — zod 校验拦截
- `EnvironmentFile` 与 `LoadCredential` 分工：非敏感 env（`DATABASE_URL`）走 EnvironmentFile，敏感 API key 走 LoadCredential（SEC-3）
- `.env.example` 要包含 `NODE_ENV=development`，CI 用 `NODE_ENV=test`；production 由 systemd EnvironmentFile 注入
- zod 3.23 的 `z.coerce.number()` 处理 string PORT；需要显式使用避免 type 推导失败
- **G2 fix（2026-04-24 · R_final B2）**：provider enum 权威值 = `anthropic | openai`（与 Drizzle schema / skeleton / ops-runbook 保持一致）；**不得**写 `claude | gpt`。`refine(env.ANTHROPIC_API_KEY必需当LLM_PROVIDER=anthropic OR LLM_FALLBACK_PROVIDER=anthropic)` 同理对 OpenAI
