## Goal

**Ship gate**：让新部署 VPS 连续跑 3 天完整 worker + briefing + resurface，operator 通过 `prod-smoke.sh` 每天一次验证：
1. `fetch_runs` 3 天内都有 `status='ok'`
2. `briefings` 每天每 topic 都有行
3. `/today` 响应 200 + < 1s（SLA §1.2）
4. `llm_calls` 累计 < $5（3 天预期）
5. pg_dump 文件每天产生
6. Caddy TLS 证书有效

同时 sanity check `/admin/sentinel`（since_day < 30 时 banner 不触发；day-1 默认不触发）。Phase 3 Exit 条件 = "connectable ≥ 7 天 briefing 无人工干预"（spec Phase 3 Exit），**T034 验证前 3 天**，之后 operator 自然观察后 4 天。

## Outputs

- `scripts/prod-smoke.sh` — 一次性 smoke：
  - `curl -fsS https://<domain>/api/healthz` 200
  - `curl -fsS -H "Cookie: session=..." https://<domain>/today` 200 + `< 1s` (自己用 `time`)
  - `psql -c 'SELECT status, count(*) FROM fetch_runs WHERE started_at > now() - interval 3 day GROUP BY status'` 应 `ok` 占主导
  - `psql -c 'SELECT count(*) FROM briefings WHERE date >= current_date - 2'` ≥ topics * 3
  - `psql -c 'SELECT sum(cost_cents)/100 FROM llm_calls WHERE created_at >= current_date - 2'` < $5
  - `ls -lt /var/backups/pg/ | head -5` 最近 3 天 3 dump
  - `curl -vI https://<domain> 2>&1 | grep 'subject'` 显示 TLS 证书有效
- `scripts/prod-metrics.sql` — 上述 SQL 合集成单个文件；日常可直接 `psql -f`
- `docs/runbook/soak-checklist.md` — operator day-by-day checklist：
  - Day 1: 部署完成 + 首次 worker 跑成功 → prod-smoke.sh
  - Day 2 / 3: 每日 08:30 跑 prod-smoke.sh；记录结果
  - Day 4+: 自然观察 `journalctl` 无 error + self-report 填入
  - 任一 day 失败 → 回 runbook/incident-response.md 查表

## Implementation plan

1. prod-smoke.sh 用 `set -euo pipefail` + 每项 echo + exit code 累计；最终返回非零退出码表示失败
2. `time` 命令在 Linux 输出 stderr；smoke 要 parse `real` 字段对比 1.0s
3. TLS 证书检查：`echo | openssl s_client -connect <domain>:443 2>/dev/null | openssl x509 -noout -dates` 验证 notAfter 距今 ≥ 30 天
4. soak-checklist.md 中文 + 每日 expected row / 数值；operator 抄表式填写
5. 本 task 自身不运行 7 天（那是现实时间），但保证工具就位：operator 启动 soak 第一天 commit；第三天做 intermediate check；第七天 close

## Verification (MUST be runnable)

- [ ] `bash scripts/prod-smoke.sh` 在部署后 24 小时第一次跑退出码 0
- [ ] 连续 3 天 `prod-smoke.sh` 都 pass；失败任一天 → incident-response 查表 → 解决 → 继续
- [ ] `psql -f scripts/prod-metrics.sql` 输出对齐 soak-checklist 预期
- [ ] day-3 末 `cat docs/runbook/soak-checklist.md` 所有 checkbox 都 ticked
- [ ] day-7 后 `journalctl -u pi-briefing-worker --since '7 days ago' | grep -c 'status=failed'` ≤ 1（SLA §1.4 "任一周不超 2 天失败"）

## Known gotchas

- 3 天 soak 是 wall-clock 时间；本 task `estimated_hours=4` 指脚本开发 + day-1 + day-3 verification；不计算 operator "在旁边等"的时间（operator 回去做别的事）
- arXiv 夜间维护可能某晚失败，fetch_runs 有 1 行 `failed`；catchup 第二天补足即可（SLA §1.3）
- C11 envelope 为月 $50；3 天预期 $3–5；高出此范围需查 `llm_calls` 异常
- TLS 证书 Caddy 自动续期；若 notAfter < 30 天多半是 DNS 或 rate limit 问题
- `prod-smoke.sh` 依赖 cookie；建议 operator 用 `curl --cookie-jar / --cookie` 保存 session

## Out of scope

- 不做自动化 soak（需要 CD + 健康钩子；solo operator 手工跑足够）
- 不做 uptime monitoring 外部探测（NG-B3）
- 不做 Load test（v0.1 ≤ 3 seat 无需）
- 不做 canary deployment（v0.1 无 blue/green）
- 不做 synthetic user flow monitoring（T032 E2E 已覆盖核心 flow）
