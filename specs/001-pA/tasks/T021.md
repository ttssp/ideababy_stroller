## Goal

实现 spec §4.2 的完整 resurface 调度：(a) **timed** 三条规则 `{6w, 3mo, 6mo}` 去重 + 去 dismiss 后生成 `resurface_events`；(b) **citation-triggered**（优先级高于 timed、不消耗 timed quota），当新 paper 进入 `papers` 且其 `paper_citations.cited_arxiv_id` 命中某 breadcrumb 的 paper + 同 topic 约束（TECH-7 噪声控制）→ 立即生成 `resurface_events`；(c) 自动填 `context_text` 按 spec §4.2 两种模板。该 pass 被 T011 worker 调用；每日一次即足够（TECH-7 控制频率）。

## Implementation plan

1. `timed.ts`：核心 SQL 一次拉所有 active breadcrumb + 其 resurface_events 聚合（`array_agg(trigger_type)`）；应用层决定下一个应触发哪个 stage。
2. `citation.ts`：
   ```sql
   SELECT b.id AS breadcrumb_id, np.id AS trigger_paper_id
   FROM papers np
   JOIN paper_citations pc ON pc.citing_paper_id = np.id
   JOIN papers bp ON bp.arxiv_id = pc.cited_arxiv_id
   JOIN breadcrumbs b ON b.paper_id = bp.id AND b.dismissed_at IS NULL
   WHERE np.id = ANY($new_paper_ids)
     AND EXISTS (
       SELECT 1 FROM paper_topic_scores pts_b
       JOIN paper_topic_scores pts_n ON pts_b.topic_id = pts_n.topic_id
       WHERE pts_b.paper_id = b.paper_id AND pts_n.paper_id = np.id
     )
   ```
3. `context.ts`：严格按 spec §4.2 模板（`trigger_type` 枚举使用权威 `timed_6wk | timed_3mo | timed_6mo | citation`）：
   - `timed_6wk` / `timed_3mo` / `timed_6mo`: `于 YYYY-MM-DD 被 breadcrumb；距今已 N 周`（N 依据具体 trigger）
   - `citation`: `于 YYYY-MM-DD 被 breadcrumb；今天被新 paper <title>(<arxiv_url>) 引用`
4. `resurface-pass.ts` 用 `dbWorker` 客户端；事务批量 INSERT；每行记日志到 worker stdout（journalctl 查阅）。
5. 调用顺序：citation first → timed（避免重复），但若某 breadcrumb 今日同时满足两路径，**保留 citation event，跳过当日 timed**（TECH-7 噪声：优先语义更丰富的 citation）。

## Known gotchas

- `dayjs` / `date-fns` 算周数：用 `differenceInWeeks(now, created_at)` 取整下取
- dismissed_at 的语义：用户主动 dismiss → 此 breadcrumb 不再 resurface；但 re-breadcrumb（T022）会 reset dismissed_at = null 且更新 created_at
- citation 的去重键是 `(breadcrumb_id, trigger_paper_id)`，不是单 breadcrumb —— 同 breadcrumb 可能被多个新 paper 引用
- TECH-7 的 topic-overlap 过滤非常关键：无此过滤会让 operator 收到一堆不相关 resurface，导致 O2 指标膨胀但信号无意义
- worker 内要用 `dbWorker`；Postgres GRANT 要允许 `INSERT INTO resurface_events`（T003 GRANT SQL 已覆盖）
- **R1 co-ownership（B3）**：`src/workers/daily.ts` 由 T011 owner；本 task 只能在 `// T021-HOOK: resurface` 注释标记处追加 `await runResurfacePass(run_id, newPaperIds);`，不能改动 T011 / T014 既有的 fetch / match / briefing 逻辑；若需改动，打开 PR 给 operator
- **R1 执行顺序**（B3）：daily.ts 内 hook 顺序 = `fetch → match → briefing(T014) → resurface(T021)`；resurface 必须在 briefing 之后以便 `paper_summaries` 已就绪（citation context_text 里如需展示 new paper 的 summary 片段也能读到）
