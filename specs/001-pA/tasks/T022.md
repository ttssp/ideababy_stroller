## Goal

让 T021 产出的 `resurface_events` 在 UI 被实际看见并被用户响应：
1. `/today` 页顶部出现 **Resurface banner**（T015 已预留位置；本任务把组件真正落地）
2. 独立 `/resurface` 页面列出过去 30 天的 resurface events（含已 click / 已 dismiss 状态）
3. 点击 card 的 "打开论文" → 写 `resurface_events.clicked_at` = now（O2 验收信号）
4. 点击 "dismiss" → 写 `resurface_events.dismissed_at` + `breadcrumbs.dismissed_at`（如是最后一次 resurface）
5. 点击 "再 breadcrumb 一次" → `re-breadcrumb`：reset `breadcrumbs.dismissed_at = null` + `breadcrumbs.created_at = now()`（重置时钟起点）

## Implementation plan

1. `actions.ts`：单事务 UPDATE；`markClicked` 只设 clicked_at；`markDismissed` 同时设 dismissed_at + 若该 breadcrumb 已收到 6mo resurface 就 UPDATE breadcrumbs.dismissed_at；`reBreadcrumb` UPDATE breadcrumbs SET dismissed_at=null, created_at=now()
2. banner pre-fetch：在 /today/page.tsx 顶部 `const pending = await getPendingResurfacesForSeat(seat_id)`；传给 client component
3. 整体样式：banner 背景米黄 + emoji `🔖`（user 要求不引入 icon library；仅 emoji）；card 紧凑两行布局
4. `/resurface` 页：表格 or 卡片流；支持筛选 "全部 / 未处理 / 已 dismiss"
5. click / dismiss 后 UI 乐观更新；失败 rollback；成功后 banner 自动消失当前卡片
6. `re-breadcrumb` 按钮仅在已 dismissed 的 resurface 卡片下显示

## Known gotchas

- banner 只展示 "今日 surfaced_at"；过去日的 resurface 不打扰 /today 页（避免堆积噪声）
- /resurface 页的 30 天窗口：`WHERE surfaced_at >= now() - interval '30 days'`
- `re-breadcrumb` 的 created_at 重置意味着：timed 6w/3mo/6mo 从今天重新计时；这和 spec §4.2 "resurface 后 user re-breadcrumb" 语义一致
- click 后 302 到 arxiv_url 时：不要用前端 `window.location.href = ...` 导致 SSR redirect；后端 return redirect URL 字符串，前端 `window.open(url, '_blank')`，这样 clicked_at 能先写后跳转
- dismissed 的 resurface_event 不再出现在 banner，但在 /resurface 历史页可见
- 多 trigger 之间的状态：一个 breadcrumb 可能有多次 resurface，每条独立 click/dismiss；`breadcrumbs.dismissed_at` 仅在**最后一次允许的 resurface 后用户 dismiss** 时置位（语义：永久结束此 breadcrumb）
- **R1 co-ownership（B3）**：`src/app/(main)/today/page.tsx` 由 T015 owner；本 task 只在页面顶部（briefing 之前）追加 `<ResurfaceBanner seat_id={...} />` 插槽；不修改 T015 的 briefing 渲染或 skip UI；已有 `depends_on: [T015]` 确保串行
- **R1 co-ownership**：T024 也会向 `/today/page.tsx` 插 `<SentinelBanner />`（位置**优先于** ResurfaceBanner · 最顶部），T022 留 SentinelBanner 插槽注释 `// T024-HOOK: sentinel-banner`
