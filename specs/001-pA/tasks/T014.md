## Goal

把 T012（state-shift engine）+ T013（summary pass）组合为每日 briefing 预计算 pass：对每个 active topic（per-lab）产出 `briefings` 行 `{lab_id, topic_id, date, state_summary, trigger_paper_ids, anchor_paper_id, llm_provider, token_cost_cents}`。**这是 ADR-2 的核心兑现**——此 pass 完成后，`/today` SSR 只读 Postgres，绝无 LLM 调用。该 task 同时把 `runBriefingPass()` 注入 T011 `daily.ts` 的 hook，成为完整的 worker 步骤链 (fetch → match → state-shift → summary → briefing)。

## Outputs

- `src/lib/briefing/types.ts` — `BriefingRow`, `TopicBriefingInput`, `AssembledBriefing`
- `src/lib/briefing/assemble.ts` — 导出 `assembleTopicBriefing({ topic_id, date, provider })`：
  - `shift = await runStateShiftPass(topic_id, provider)`（可能返回空或 k 个）
  - `trigger_paper_ids = shift[0]?.trigger_papers ?? topCandidatesByScore(topic_id, 3)`
  - `anchor_paper_id = shift[0]?.anchor_paper?.id ?? null`
  - **R1 变更**：`summaries = await runSummaryPass(trigger_paper_ids.map(pid => ({ paper_id: pid, topic_id })), provider)`（签名带 topic_id，写入 `paper_summaries`）
  - **R1 变更**：briefing 不再把 summary 文本直接存到 `briefings.state_summary`；改为 briefing 行只存 `{ trigger_paper_ids, anchor_paper_id, state_summary（= 一句 topic-level 判断语，不含 per-paper summary）}`；T015 在 render 时 JOIN `paper_summaries` 拿 per-paper 文本
  - `state_summary`：若有 shift → `"发生 state shift: anchor《X》被《A》《B》以冲突立场引用"`（模板由 architecture §4.2 context_text 模式借鉴）；若无 shift → **统一文案** `"frontier stable, no shift"`（M3 建议的二者合一择一，今日无新 paper 也归此情况；之前"今日无新论文进入本 topic"已并入同一语义）
  - `token_cost_cents = summaries.rows.reduce(+cost)`
  - `llm_provider = provider.name`
- `src/workers/briefing-pass.ts` — 导出 `runBriefingPass(run_id)`：for each active topic → `assembleTopicBriefing` → UPSERT `briefings`（ON CONFLICT (lab_id, topic_id, date) DO UPDATE）；同时更新 `fetch_runs.papers_matched` 等计数
- **R1 co-ownership 变更**：修改 `src/workers/daily.ts`（T011 owner；T014 填 `// T014-HOOK: briefing` 插槽）：在 match-topics 完成后 `import { runBriefingPass } from './briefing-pass'; await runBriefingPass(run_id);`
- `tests/lib/briefing/assemble.test.ts` — 覆盖 shift / no-shift / capped-cost / empty topic 四种路径；**R1 新增**：`paper_summaries` 被正确写入（每个 trigger_paper 一行）；briefing.state_summary **不**冗余 per-paper 文本
- `tests/workers/briefing-pass.test.ts` — 3 topic 测试，end-to-end 验证 `briefings` 表写入 + UPSERT 幂等

## Implementation plan

1. `assemble.ts`：
   - 先跑 state-shift；若无 shift（空 result），fallback 到 `topCandidatesByScore`：按过去 7 天加入本 topic 的 paper 数排序（`paper_topic_scores` count），取 top 3
   - 对 trigger_paper_ids（最多 3）调 summary pass；rate limit 已在 T004/arxiv 端控制
   - 拼 state_summary：中文固定模板 + 动态插入 paper 标题
2. `briefing-pass.ts`：
   - `const today = dayjs().tz('Asia/Shanghai').format('YYYY-MM-DD')`
   - 遍历 active topic；逐个调 `assembleTopicBriefing`；UPSERT `briefings`
   - 跑 UPSERT 而非 INSERT 保证 worker 重跑幂等（on-boot catchup 场景）
3. UPSERT SQL 借助 Drizzle `.onConflictDoUpdate({ target: [lab_id, topic_id, date], set: { ... } })`
4. 注入 T011 的 `daily.ts`：在 match-topics 完成后调用；worker 出口 status 由 briefing-pass 成功决定。

## Known gotchas

- 时区：`Asia/Shanghai` 取日期做 UNIQUE key；不可用 UTC（否则 06:00 cron 产出 date 仍是"昨天"）
- 空 topic（今天无 paper 入）：仍要写一行 briefings，`state_summary="frontier stable, no shift"`；避免 /today 页 join 漏 topic（M3 统一文案）
- 若 T001 fallback（LLM disabled），provider.summarize 仍启用；T012 的 engine 在无 LLM 时也能产出 shift；只是 label_conflict 为 null
- **R1 重要 gotcha**：T013 `runSummaryPass` 内部失败（provider 全挂）会走 heuristic fallback 产出 `paper_summaries` 行（`model_name='fallback-heuristic-v1'`）；T014 不关心来源，直接 JOIN `paper_summaries` 拿文本即可
- **R1 co-ownership**：`daily.ts` 由 T011 owner；本 task 只能在 `// T014-HOOK: briefing` 注释标记处插入代码，不能改动 T011 既有的 fetch / match / run-log 逻辑；若需修改，打开 PR 给 operator 协调
- UPSERT 的 date 冲突发生在 on-boot catchup：确保 SET 子句更新 state_summary / cost / trigger 但**保留原 created_at**
- 单 topic assemble 失败不要阻断全 lab briefing pass；用 try/catch 记 error 到 fetch_runs.error_text（逗号分隔）
- `token_cost_cents` 要乘以实际 summaries.rows，不是 estimate
