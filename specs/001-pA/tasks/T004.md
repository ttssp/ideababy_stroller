## Goal

把 `reference/llm-adapter-skeleton.md §1–§7` 的 **权威 LLM 合同** 落到 `src/lib/llm/`：

1. **Interface**（`types.ts`）：`LLMProvider` interface + `SummaryRecord`（**camelCase**）+ `StateShiftVerdict`（discriminated union）+ `LLMProviderError`。
2. **Adapters**：`anthropic.ts` + `openai.ts`，实现 `LLMProvider`；均不写 `paper_summaries`；但**内部**调 `recordLLMCall()` 写 `llm_calls`、取回 id、把 id 放进返回的 `SummaryRecord.llmCallId` 后再交给 caller。
3. **Select**（`select.ts`）：env-driven primary/fallback + `callWithFallback()` 跨 provider 热切。
4. **Helpers**：`sanitize.ts` (stripPII) + `utils.ts` (truncateTo3Sentences · G1 count-terminator 实现) + `prompt-version.ts` + `audit.ts` (recordLLMCall + budget gate)。
5. **Tests**：mock SDK（msw / vi.mock），覆盖 happy path、error path、adversarial fixture、truncate、confidence downgrade。

**G2 纠正（2026-04-24 R_final B2 fix · 权威真源）**：

## Known gotchas

- **Adapter 写 `llm_calls` · 不写 `paper_summaries`**：ADR-6 的职责分工；`recordLLMCall` 是 adapter 内部的 "成本审计原子"；caller（T013 `persistSummary`）再单独写 `paper_summaries`（非事务内，但 `llmCallId` 已由 adapter 返回 · 构成 FK 依赖）
- **camelCase 在 TS 合同层 · snake_case 在 DB 层**：Drizzle `.values({ summary_text: record.summaryText, prompt_version: record.promptVersion })` 做映射（由 T013 `persistSummary` 负责；本 task 的 types 保持 camelCase）
- **`judgeRelation` 是 per-pair**：T012 state-shift pass 内部 `for (candidate, anchor) of pairs` 循环调 adapter；**不**要求 adapter 支持 batch（batch 在 spike harness `eval-harness.ts` 里单独实现）
- Anthropic SDK `metadata.user_id` 如果传了会被记入 abuse detection log，**不要传**
- OpenAI `response_format: json_object` 要求 prompt 里含 "return JSON"；`STATE_SHIFT_SYSTEM_PROMPT_V1` 已满足
- 两家 token 计数字段不同：Claude `response.usage.input_tokens/output_tokens`；OpenAI `usage.prompt_tokens/completion_tokens` → adapter 统一转 camelCase `inputTokens/outputTokens`
- **`sanitize` 不是静默替换** —— 静默替换会让 "prompt 里有 email" 的 bug 悄悄通过测试，必须 throw
- **G1 truncate 与 DB CHECK 对齐**：`reference/skeletons/llm-utils.ts` 权威实现 + `paper_summaries.summary_sentence_cap` 权威 CHECK（数终结符）行为一致；**应用层永远 defensive 追加 `。` 给无终结符文本**，避免 DB CHECK 拒 worker persist
- **OpenAI output pricing $15/M**（G2 · 2026-04-24 R_final B2 fix）：`calcCostOpenAICents` outputRate = `1500/1_000_000`；此前 `tech-stack.md §2.2` 初版写 $10/M 是 2026-01 价，R1 refresh 后实际为 $15/M；实际月度估约 $12 仍在 C11 envelope（$50）内
- **SDK 版本 pin**：先 `pnpm add @anthropic-ai/sdk@latest openai@latest`（**0.4.0 修订**：Phase 2 T001-v2 sign-off 后才锁死版本 · Phase 0/1 允许 caret range · T001-v2 签字后一并 pin-to-patch）
- **Contract test 用 msw**：参考 `reference/testing-strategy.md §5.3` mock helper；HTTP 层完全拦截，不跨网络
