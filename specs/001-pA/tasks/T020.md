## Goal

提供 per-seat 的 breadcrumb 历史列表：当前 seat 所有 breadcrumb（含已 dismiss 与未 dismiss），支持按 topic 筛选、时间倒序。这是 PI "3 个月后回查过去 90 天我标过 breadcrumb" 场景（spec §Core user stories #3）的主入口。同时展示每条 breadcrumb 已被 resurface 过的次数（便于 O2 观察）。

## Inputs

- `specs/001-pA/spec.md` IN-4、§6 O2 verification（`resurface_events.clicked_at IS NOT NULL`）、Core user story 3
- `specs/001-pA/architecture.md` §5.1（breadcrumbs + resurface_events schema）
- T003 产出：breadcrumbs + resurface_events schema
- T006 产出：auth
- T010 产出：topics（用于筛选器）
- T015 产出：paper info render pattern 可复用

## Outputs

- `src/lib/breadcrumbs/list.ts` — 导出 `listBreadcrumbs({ seat_id, topic_id? })`：
  - SELECT breadcrumbs LEFT JOIN papers LEFT JOIN topics LEFT JOIN LATERAL (SELECT count(*), max(surfaced_at) FROM resurface_events WHERE breadcrumb_id = b.id)
  - 返回 `BreadcrumbRow[] = Array<{ id, paper: { title, arxiv_url }, topic: { id, name }, created_at, dismissed_at, resurface_count, last_surfaced_at, has_clicked }>`
  - 支持 topic_id 过滤；按 created_at DESC
- `src/app/(main)/breadcrumbs/page.tsx` — server component + topic filter（URL query `?topic=<id>`）+ 表格展示；顶部 summary count "您共有 N 条 breadcrumb，其中 K 被 resurface 过"
- `tests/lib/breadcrumbs/list.test.ts` — 覆盖空、多 breadcrumb、筛选、resurface 计数

## Implementation plan

1. `list.ts`：一次 SQL 取完；`LATERAL` subquery 汇总 resurface 计数与 `has_clicked`。
2. 页面：`<table>`：paper 标题（链 arxiv） · topic · created · "最近一次 resurface / 未触发" · dismiss 按钮（client component 调 PATCH /api/breadcrumbs/:id/dismiss，本任务**不**实现 PATCH——推到 T022）
3. topic filter：`<form method="GET">` + select`<option>` 来自 active topics。
4. 统计顶部 banner：两行文字：`共 N 条 breadcrumb` + `其中 K 条在过去 30 天被 resurface` —— 直接映射 O2 信号，帮助 operator self-report。

## Known gotchas

- 一个 breadcrumb 可能跨 topic（paper 匹配多 topic），但 `breadcrumbs.topic_id` 存的是用户点击时的"主" topic；此 list 不需要 expand 多 topic 视角
- 空状态页：用户新创建 lab 后一条 breadcrumb 都没有，要显示 "尚无 breadcrumb。在 /today 页对你想日后回看的 paper 点'breadcrumb'即可" 而非空白
- 不要 JOIN 其他 seat 的 breadcrumbs（per-seat 隐私）
- topic filter 的值为 "all" 时 URL 里去掉该 query（SEO 不care，但 UX 干净）
