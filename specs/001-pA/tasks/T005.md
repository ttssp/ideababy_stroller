## Goal

把 arXiv 的 Atom/XML feed 封装成一个可复用的、速率自律的、类型安全的 adapter。暴露两个主方法：`fetchRecent({ categories, sinceISO })` 与 `extractCitations(arxivId)`（后者用于 §4.2 citation-triggered resurface）。严格遵守 arXiv ToS：rate limit ≤ 1 req/3s（compliance.md LEG-2 + risks.md TECH-2 的硬约束），三次 exponential backoff（SLA §1.3）。

## Outputs

- `src/lib/arxiv/client.ts` — 导出 `fetchRecent({ categories: string[], sinceISO: string, maxResults: number })`：返回 `ArxivPaper[]`；内部分页 (`start` / `max_results=100`) + rate limit

## Implementation plan

1. `types.ts`：`ArxivPaper = { arxiv_id, title, abstract, authors: string[], published_at: Date, categories: string[], primary_category, pdf_url }`；**不含 pdf 内容**（OUT-6）。
2. `rate-limit.ts`：简单 `let lastCall = 0; await sleep(Math.max(0, 3000 - (Date.now() - lastCall)))`；module-level 全局；导出 `async function enter()`。
3. `parser.ts`：`fast-xml-parser` 配 `ignoreAttributes: false, parseTagValue: true`；展开 Atom `<entry>`；抽 `<id>` / `<title>` / `<summary>` / `<author>/<name>` / `<published>` / `<category>/@term`。
4. `client.ts`：`fetchRecent` 构造 URL `http://export.arxiv.org/api/query?search_query=cat:{cats}&start=0&max_results=100&sortBy=submittedDate&sortOrder=descending`；遍历分页；**User-Agent: "pi-briefing-console/0.1 (+contact)"**（ToS 要求自标识）；3 次 exponential backoff on 5xx；总超时 60s（SLA §1.3）。
5. `citations.ts`：走 OpenAlex 公开接口（无 API key 要求，免费额度足够日均 ≤ 100 论文 × 30 天）；失败时返回空数组（TECH-7 噪声控制：宁可少 resurface 不可乱 resurface）。
6. 所有 outbound HTTP 走 Node 22 原生 `fetch`（undici）；不引入 axios。
7. 测试用 `undici.MockAgent` 拦截 HTTPS。

## Known gotchas

- arXiv API 用 HTTP（不是 HTTPS）历史原因；Node 22 `fetch` 支持；**不要**强制 https 否则 URL 报错
- `fast-xml-parser` 对单条 vs 多条 element 会返回不同类型（object vs array）；需要 normalize helper
- OpenAlex 的 citation endpoint 不保证覆盖每一篇 arXiv paper，失败视为"空引用列表"，不 panic（TECH-7）
- rate limiter 必须是**模块级单例**，多处 import 仍共享；如果 T011 worker 跨进程还需要额外 file lock（worker 是单进程 systemd job，无需）
- arXiv XML 的 `<summary>` 含多行换行，需要 `.trim().replace(/\s+/g, ' ')` 再入库
- compliance.md LEG-2 明确"metadata 不公开再分发"——本 adapter 仅为内部消费，不要暴露作为 REST 接口
