## Goal

落地 architecture.md §7 的完整 auth 骨架：email token invite（24h 有效、单次使用、hash 存库）→ magic-link verify → JWT httpOnly session cookie → middleware 拦截所有非 `/login*` 路由。PI + operator 各自独立 login（spec IN-5 + D3），admin 与 member 角色分离（SEC-1）。**所有路由默认 deny**，白名单 `/login`、`/login/verify` 通过后才可访问其他页面。这是 C7 / OUT-8 红线 3 的机械守门。

## Implementation plan

1. `session.ts`：用 `jose.SignJWT` + `HS256` 签名；payload `{ sub: seat_id, iat, exp }`；`verifySession` 校验 `exp` 与 `sessions.revoked_at IS NULL AND sessions.token_hash = sha256(token)`。
2. `invite.ts`：`crypto.randomBytes(32).toString('base64url')`；`seats.invite_token_hash` 列存 `sha256` + `seats.invite_expires_at`；`consumeInvite` 事务里校验并置 null。
3. **数据库 schema（R1→R2 Q2 resolution · 2026-04-23）**：T003 已前置 ship `seats.invite_token_hash` / `seats.invite_expires_at` / `seats.invited_at` / `sessions.revoked_at` / `sessions.last_active_at` 五列（随 `0000_initial.sql` 一次 ship，见 T003 Implementation 步骤 14）。**T006 不再写任何 migration 文件**，只从 `src/db/schema.ts` import 类型 + 从 `src/db/client.ts` 拿到 `db` / `dbWorker` 执行读写。
4. `middleware.ts`：匹配 `matcher: ['/((?!login|_next/static|_next/image|favicon.ico|api/healthz|api/invite/consume).*)']`；无 cookie → `NextResponse.redirect('/login')`。`/api/invite/consume` 加白名单（无 auth · G4 H4）
5. Invite URL `{APP_ORIGIN}/login#invite=<plain>`（**fragment 形式** · G4 H4）；plain token 仅出现在 admin 看到的 `inviteUrl` 响应里 + 被邀者浏览器的 URL fragment · 服务端**永不** log plain token；日志里只写 hash（SEC-2）。
6. `sessions.login_date` 在 session 创建时写当日本地日期（Asia/Shanghai），供 O1/O3 index 查询。
7. 所有 handler 捕获 throw，返回统一 `{ error: { code, message } }`，**不暴露** "email not found" vs "wrong token" 差异（SEC-2）。`/api/invite/consume` 的 `INVITE_TOKEN_EXPIRED` 覆盖 invalid / consumed / expired 三种情况，message 统一（不区分）。
8. **G4 H4 CSRF 防御三层**（`reference/api-contracts.md §E2 Notes`）：(a) `SameSite=Lax` session cookie（跨站表单无 cookie）· (b) `csrf.ts::assertOrigin` 强制 Origin header 匹配 · (c) v0.2 加 one-time nonce（`X-Invite-Nonce` header + cookie 同步 · 本 task 留 hook 不实现）

## Known gotchas

- Next 15 middleware 运行在 edge runtime；`jose` ok，但 `crypto.randomBytes` 只能在 node runtime——`invite.ts` 用 Web Crypto `crypto.getRandomValues`
- `sessions.login_date` 必须用 `Asia/Shanghai` 时区截日，否则 O1 跨零点统计会错 1 天
- Nodemailer SMTP 失败时不要把 "邮件没发出" 返回给前端（SEC-2：避免暴露 "该 email 是否存在"）；fallback 是 log + 后端告警
- `jose` HMAC secret 要用 `new TextEncoder().encode(SESSION_SECRET)` 且 secret ≥ 32 byte
- **R1→R2 Q2 resolution（2026-04-23）**：T003 已 ship 这 5 列（`seats.{invite_token_hash, invite_expires_at, invited_at}` + `sessions.{revoked_at, last_active_at}`），T006 只写 `src/lib/auth/invite.ts` + session middleware，不再需要写 migration。若本地 merge 顺序异常（T003 尚未 apply），先 rebase 拉 T003 的 schema；**禁止** 在 T006 PR 里新增 `src/db/migrations/0002_auth_columns.sql`
