2. `auth.ts`：测试快捷 login endpoint `/api/test/login?email=<x>`（仅 `NODE_ENV=test` 启用）—— 直接创建 session 跳过 invite 邮件；生产 build 时此 route 404

4. `admin-export.spec.ts`：用 `page.request.get('/api/export/full')` → `await response.json()` → assert `schema_version >= '1.1'`（R1→R2 Q4 resolution 2026-04-23 · T023 已 bump 至 `1.1` 因新增 `paper_summaries` 顶层 key；用 `>= '1.1'` 写法前向兼容未来 bump）+ 各字段；再调 `node scripts/export-import-round-trip.ts` 断言 round-trip

## Known gotchas

- `/api/test/login` 是测试便利门户；CI 部署 workflow 要确保 `NODE_ENV=production` 此路由返回 404（手工 grep 验证）
- `page.clock` API 在 Playwright 1.48+ 但对 server-side 时钟无效；resurface 用 SQL backdate 更可靠
- 20 次 perf 测试在 CI runner 上 p95 更慢（共享 VM）；CI 的阈值放宽到 1500ms，local dev 才 1000ms；`expect(p95).toBeLessThan(process.env.CI ? 1500 : 1000)`
- Playwright 浏览器在 CI 要缓存 `~/.cache/ms-playwright`，否则每次 ~5 min 下载
- `beforeEach` 调 seedMinimalLab 会影响并发 spec 执行；建议 `test.describe.configure({ mode: 'serial' })`
- 不要 E2E 测 LLM 调用（成本 + flakiness）；把 LLM provider 在 test env 换成 `FakeProvider` 返回固定 summary
