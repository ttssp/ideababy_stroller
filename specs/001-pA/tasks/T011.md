## Goal

写每日 06:00 cron 的核心 worker：(1) `arxiv.fetchRecent` 拉过去 24h 所有 lab 关注 category 的 paper；(2) 按每 topic 的 keyword_pool / arxiv_categories / seed_authors 匹配写 `paper_topic_scores`；(3) 记录 `fetch_runs` 审计；(4) 支持 **on-boot catchup**（worker 启动时查 `last_success`，若 > 24h 回补过去 24h）。本 task 仅负责"拉 + 存"，LLM summary / state-shift / resurface scheduler 留给 T012 / T013 / T021（它们作为同一 worker 的后续 step 在 T014 装配）。

- `tests/workers/backoff.test.ts` — 模拟 1、2、3 次失败后成功的 retry 行为

## Implementation plan

1. `backoff.ts`：经典 `for (let i = 0; i < attempts; i++) try { return await fn() } catch (e) { await sleep(base * 2^i) }`；第三次仍失败抛最后一次 error。
2. `run-log.ts`：`startRun` → INSERT 返回 id + `started_at`；`finishRun(id, status, { papers_fetched, papers_matched, error_text? })` → UPDATE row。
3. `fetch-papers.ts`：
   - `const topics = await db.select().from(topics).where(isNull(topics.archived_at))`
   - `const categories = [...new Set(topics.flatMap(t => t.arxiv_categories))]`
   - `const papers = await arxivClient.fetchRecent({ categories, sinceISO, maxResults: 500 })`（TECH-2：单次最多 500 避免长跑）
   - 逐篇 `INSERT INTO papers ... ON CONFLICT (arxiv_id) DO NOTHING RETURNING id`
   - 并发 ≤ 3 调 `extractCitations` 写 `paper_citations`
4. `match-topics.ts`：对每张 topic × paper 跑 3 类匹配：
   - **category**: `topic.arxiv_categories ∩ paper.categories ≠ ∅` → match_signal=`category`
   - **keyword**: 标题+摘要转小写后模糊命中 ≥ 1 keyword → `keyword`
   - **seed_author**: `paper.authors ∩ topic.seed_authors ≠ ∅` → `seed_author`
   - 每个信号写独立 `paper_topic_scores` 行（允许同 (paper,topic) 多行，不同信号）
5. `daily.ts`:
   - `const last = await getLastSuccessfulRun()`
   - `const sinceISO = last ? new Date(last.started_at).toISOString() : dayjs().subtract(24, 'hour').toISOString()`
   - on-boot catchup：`if (Date.now() - last.started_at > 24h) sinceISO = last.started_at`；log 显式提示 "catching up from X"
   - 执行三步；预留 `runBriefingPass` hook（T014 会 import 并注入）
6. 本任务**不调 LLM**（T013 才调）；仅 fetch + match，保证 Phase 1 可单独 verify。
7. 命令：`pnpm worker:daily` = `pnpm tsx src/workers/daily.ts`。

## Known gotchas

- Drizzle `ON CONFLICT DO NOTHING` 语法在 postgres driver 下需要 `.onConflictDoNothing()`
- `paper_topic_scores` 一 paper-topic 对可能有多 match_signal；不要去重，下游 T012 做 OR 聚合
- 并发 > 3 调 arXiv `extractCitations` 会触发 rate limit（LEG-2）；用 `p-limit` 或手写 semaphore
- Worker 结束后**必须** exit（Node 进程默认 stay alive 等 Postgres 连接）；显式 `await db.$client.end()` 然后 `process.exit(code)`
- systemd `Type=oneshot` 对长时间 worker 正确；但如果某次 worker 卡住 30min+，`TimeoutStartSec=30min` 要配（在 T007 service 文件里补）
- 避免 `catchup` 回补多日时一次拉太多：限制 catchup 窗口 ≤ 7 天，超过就 log 警告 + 只回补 24h
- **R1 co-ownership 注意**（B3）：`src/workers/daily.ts` 由 T011 **owner**，下游 T014（注入 `runBriefingPass`）+ T021（注入 `runResurfacePass`）会在此文件追加 import + hook 调用；T011 必须先提前在 `daily.ts` 里保留**两个清晰注释的 hook 插槽**（`// T014-HOOK: briefing` 与 `// T021-HOOK: resurface`），否则下游 task 并行工作时会出现 merge 冲突
