## Goal

把 Next.js 15 App Router + TypeScript strict + Biome + pnpm 单 package 骨架一次性铺好，并配好最小 CI workflow。这个任务是 Phase 0 几乎所有其他任务的 upstream（T003/T004/T007/T008），必须把依赖树、脚本命令、lint 规则、CI 矩阵定死，让后面每个 task 只管自己的 file_domain 即可。

## Inputs

- `specs/001-pA/tech-stack.md` §1（主栈版本）、§5（版本锁 package.json 样板）、§6（本地开发环境）
- 没有 upstream task 依赖

## Outputs

- `package.json` — 按 tech-stack.md §5 样板，精确版本锁定 production dep（不用 `^` / `~`）；scripts 含 `dev`, `build`, `start`, `lint`, `test`, `test:e2e`, `typecheck`, `db:migrate`, `worker:daily`（后两项 stub）
- `tsconfig.json` — `"strict": true`, `"noUncheckedIndexedAccess": true`, `"target": "ES2022"`, `"moduleResolution": "bundler"`, paths alias `@/*` → `src/*`
- `biome.json` — 继承官方推荐；禁用 any，启用 import sort
- `.gitignore` — Node / Next / Biome cache + `.env*`（必须防止 SEC-3）
- `next.config.mjs` — 最小；`reactStrictMode: true`；无 custom webpack
- `src/app/layout.tsx` + `src/app/page.tsx` — 空壳，`<html lang="zh-CN">`，body 单句 "PI Briefing Console · v0.1"
- `.github/workflows/ci.yml` — 单 workflow：`pnpm install --frozen-lockfile` → `pnpm biome ci` → `pnpm typecheck` → `pnpm vitest run`（Playwright 推迟到 T008 启用）
- `README.md` — 50 行内，含 `pnpm install`、本地 Postgres 启动、如何跑 dev；链到 spec 与 architecture

## Implementation plan

1. `pnpm init`，写 `package.json` 按样板；`pnpm install` 验证 lockfile。
2. 写 `tsconfig.json`；`pnpm tsc --noEmit` 通过。
3. 写 `biome.json`；`pnpm biome check .` 通过。
4. Next 15 基础壳：`src/app/layout.tsx` + `src/app/page.tsx`；`pnpm dev` 能启动，`pnpm build` 能产出 `.next/`。
5. `.gitignore` 防止 `.env*` 入库（SEC-3）。
6. GitHub Actions workflow：单 job `ubuntu-latest` + Node 22 + pnpm 9；跑 lint + typecheck + vitest（vitest 此时无 test，空跑即可通过）。
7. README 精简介绍。

## Known gotchas

- Node 22 LTS 的 ESM/CJS 互操作仍有偶发坑；`next.config.mjs` 用 ESM，且 `package.json` **不**标 `"type": "module"`（Next 15 内部会根据文件扩展名处理；强制 ESM 会让 Drizzle migrations 报错）
- Biome 1.9 与 Next 15 的 JSX handling 需要 `javascript.jsxRuntime = "reactJsx"`
- tech-stack.md §4 规定生产 dep ≤ 30 个；此 task 引入 7 个（next/react/react-dom/typescript/@types/node + Biome/Next dev bundle），留 23 个 buffer 给后续 task
- 不要在此任务装任何 LLM SDK —— T001 spike 后由 T004 决定装哪个
