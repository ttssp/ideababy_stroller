## Implementation plan

1. 按 architecture.md §5.1 写 schema：所有表用 snake_case；枚举用 Drizzle `pgEnum`（action / role / match_signal / trigger_type / fetch_run_status / llm_call_kind / **export_type**）。
2. 建 index 按 §5.2：`actions(seat_id, created_at)`、`sessions(seat_id, login_date)`、`resurface_events(breadcrumb_id, surfaced_at)`、`paper_citations(cited_arxiv_id)`、`briefings(lab_id, topic_id, date UNIQUE)`、**`paper_summaries(paper_id)`**、**`paper_summaries(topic_id)`**、**`export_log(lab_id, created_at)`**。
3. `papers.arxiv_id` 加 UNIQUE 约束；`seats.email` 加 UNIQUE；`actions.why` 加 `length(why) <= 280` CHECK 约束（spec IN-3 + §4.1）。
4. `breadcrumbs` 外键 CASCADE DELETE 到 `resurface_events`（compliance.md §5.2 删除流程要求）；`paper_summaries` 外键 CASCADE DELETE 到 `papers` / `topics`。
5. `drizzle.config.ts` 指向 `DATABASE_URL`；`pnpm drizzle-kit generate` 产出 `0000_initial.sql`。
6. `client.ts`：两份连接。**webapp_user** 只 GRANT SELECT/INSERT/UPDATE 在 {sessions, actions, breadcrumbs, resurface_events, topics, seats, export_log}；**worker_user** 额外 GRANT INSERT 到 {papers, paper_citations, paper_topic_scores, briefings, fetch_runs, llm_calls, **paper_summaries**}。权限 GRANT SQL 作为单独 migration `0001_grants.sql`（由 T007 在部署时 apply）。
7. `db-seed.ts` 可执行：读 `process.env.ADMIN_EMAIL` 后创建 admin seat；**INSERT labs 时 `first_day_at=now()`、`allow_continue_until=null`**；幂等（用 `ON CONFLICT DO NOTHING`）。
8. `schema.test.ts` 每张表一条 fixture + 一个 roundtrip 断言。
9. **R1 新增 R1-a：`paper_summaries` 表**（**F5 · 2026-04-24 v0.2.2 加固** · `llm_call_id` 改 nullable + `on delete set null`；**G1 · 2026-04-24 v0.2.3 R_final B1 fix** · `summary_sentence_cap` CHECK 换成"数终结符"语义）：
   ```sql
   create table paper_summaries (
     id             bigserial primary key,
     paper_id       bigint not null references papers(id) on delete cascade,
     topic_id       bigint not null references topics(id) on delete cascade,
     summary_text   text   not null,
     -- F5: nullable + ON DELETE SET NULL supports JSON export round-trip
     -- (option (a) per api-contracts.md §3.11 Import round-trip notes).
     llm_call_id    bigint null references llm_calls(id) on delete set null,
     model_name     text   not null,
     prompt_version text   not null,
     created_at     timestamptz not null default now(),
     -- G1 fix (R_final B1 · 2026-04-24): count terminator occurrences
     -- (English + Chinese). Accept 1..3; reject 0 (unterminated) or >3.
     -- Whitespace-independent, so pure Chinese "中文。第二句。第三句。"
     -- is correctly judged as 3 sentences.
     constraint summary_sentence_cap check (
       coalesce(
         array_length(regexp_matches(summary_text, '[.!?。！？]', 'g'), 1),
         0
       ) between 1 and 3
     ),
     unique (paper_id, topic_id, prompt_version)
   );
   create index paper_summaries_paper_id_idx on paper_summaries (paper_id);
   create index paper_summaries_topic_id_idx on paper_summaries (topic_id);
   ```
   这是 B1 blocker 的 authoritative 落点：per-paper per-topic per-prompt_version 一条 summary。**不**把 summary 文本塞进 `llm_calls`（审计表只记 cost / tokens）。`llm_call_id` 允许 null 是**故意**的（F5 round-trip 策略 option (a)）· 正常 worker INSERT 路径仍写非空值（T013 `persistSummary` 事务要求），null 仅出现在 import-from-export 的 round-trip 场景。**G1 新 CHECK 语义**：纯中文 3 句（终结符 = 3）通过，5 句拒绝；无终结符文本拒绝（应用层 `truncateTo3Sentences` 在 persist 前 defensive 追加 `。`，保证 worker 路径不会把无终结符文本喂进 DB）。
10. **R1 新增 R1-b：`actions` 表新增 `skip_requires_why` CHECK**（红线 2 机械兜底第 1 层 · B2）：

    ```sql
    alter table actions add constraint skip_requires_why check (
      action != 'skip' or (why is not null and char_length(btrim(why)) >= 5)

    ```
    `>= 5` 避免空格 / `a` / `.` 之类的 bypass；API 层（T015）与 UI 层（T015）叠加防御，见对应 task。

## Known gotchas

- Drizzle `pgEnum` 需要在 schema.ts 顶部先定义，再被 `text({ enum: [...] })` 使用时会有类型推导缓慢问题——用 Drizzle 原生 `pgEnum` + `.enum` column helper
- `paper_citations` 的 `cited_arxiv_id` 存 arxiv_id 字符串而非 FK（因为被引 paper 可能还没进 `papers` 表）；这是 architecture.md §5 §5.2 热路径的前置设计
- `briefings(lab_id, topic_id, date)` UNIQUE 约束避免同 topic 同日重复 briefing（worker 重跑时需要 UPSERT）
- 不要给 `labs` 表加 multi-tenant 列——架构 v0.1 单 lab 部署（architecture.md §1 原则）；`lab_id` 冗余只为未来 v1.0 federation 留口子
- `sessions.login_date` 是冗余列（= `issued_at::date`）但 O1/O3 查询用它走 index，不要移除
- **paper_summaries 的 `summary_sentence_cap` CHECK**（G1 · R_final B1 · 2026-04-24 重写）使用 `coalesce(array_length(regexp_matches(summary_text, '[.!?。！？]', 'g'), 1), 0) between 1 and 3`，按"数终结符"语义判断句数。优点：与空格无关，纯中文 `"中文。第二句。第三句。"` 正确判为 3 句；旧实现按 `split on terminator+whitespace` 会把这段视为 1 句（漏判）。副作用：无终结符文本（如 `"No terminator"`）会被拒绝；**应用层 `truncateTo3Sentences` 在 persist 前 defensive 追加 `。`**，正常 worker 路径不会把无终结符文本喂进 DB（风险登记见 risks.md TECH-8）。
- **skip_requires_why CHECK** 的 `btrim + char_length >= 5` 是在 DB 层的"最小机械保证"，语义质量仍靠 UI 提示（T015）与 operator self-report 月度抽查（risks.md TECH-9）
- `paper_summaries.llm_call_id` 外键指回 `llm_calls`；这是审计链（通过 llm_call 可查到当时 provider / 成本 / 时间），但 summary 文本**只**在 `paper_summaries` 里。**F5（2026-04-24 v0.2.2 pre-R_final）**：该列改为 `nullable + on delete set null`，以支持 JSON export round-trip（option (a)：import 时 `llm_call_id = null`，见 `api-contracts.md §3.11 Import round-trip notes`）；常规 worker 写入路径（T013 `persistSummary`）仍写非空值
- `export_log.row_counts_jsonb` 使用 Postgres `jsonb` 类型（非 `json`）便于查询子键；T023 写入时一并记录全部 **9 张 export 表**的行数（labs / seats / topics / papers / actions / breadcrumbs / resurface_events / briefings / paper_summaries）
