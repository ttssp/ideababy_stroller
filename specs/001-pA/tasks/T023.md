## Goal

实现 admin-only JSON export endpoint `/api/export/full` —— 这是 **C8 portability 承诺 + IN-6 + spec §6** 的 ship-critical 项。按 **`reference/api-contracts.md §3.11 ExportEnvelope` 权威单一真源** 输出全部 user-authored + LLM-authored derived 数据；**不包含** API key / session token / DB 密码 / `sessions` / `llmCalls` / `exportLog`。同时写一个 `export-import-round-trip.ts` 脚本验证 round-trip（T032 E2E 会走此路径）。

**SEC-1 守门**：middleware + controller 双重 admin 检查；**每次成功 export 同步写 `export_log` 一行**（`row_counts_jsonb` + `byte_size`），operator 月度自审（compliance §7）。

**G3 关键纠正（2026-04-24 R_final B3 fix · 权威真源）**：
- **权威 envelope shape = `reference/api-contracts.md §3.11`**（camelCase · 12 顶层 key）
- **顶层 12 个**：`labs / seats / topics / papers / paperTopicScores / paperCitations / paperSummaries / briefings / actions / breadcrumbs / resurfaceEvents / fetchRuns`
- **排除 3 个**：`sessions`（runtime state）· `llmCalls`（审计内部）· `exportLog`（self-referential · 本 endpoint 写入但不含）
- **顶层字段**：`schemaVersion: '1.1'`（camelCase · 不是 `schema_version`）· `exportedAt`（ISO）· `lab: { id, name, firstDayAt, exportedAt, exportedBySeatId }`
- **`buildFullExport(labId: number)`** —— labId 是**参数**，从 session 传入（`session.labId`）；**不**从 `env.LAB_ID` 读（`.env.example` 无此变量）
- **F5 round-trip 策略**：default option (a)：`paperSummaries.llmCallId = null` 在 import 端（schema 已 nullable + `on delete set null`）· option (b) 保留为 v0.2 升级路径

## Implementation plan

1. **schema.ts**（~45 min）· 用 zod `.object({...})` 精确按 api-contracts.md §3.11；每张表 row shape 单独 `z.object`，再组 envelope `z.object({ schemaVersion: z.literal('1.1'), exportedAt: z.string(), lab: z.object({...}), seats: z.array(...), ... })`
2. **builder.ts**（~2h）
   - 12 SELECT（Drizzle query builder），顺序 = labs → seats → topics → papers → paperTopicScores → paperCitations → paperSummaries → briefings → actions → breadcrumbs → resurfaceEvents → fetchRuns
   - 对每个 row 显式 pick needed fields（不 `SELECT *` · 避免意外泄 invite_token_hash）
   - `papers` 字段派生 `arxivUrl = 'https://arxiv.org/abs/' + arxivId`
   - `paperSummaries` 逐字输出（含 `llmCallId` 字段 · 可为 null）
   - zod parse 自身作为 sanity check（zod parse 失败 → `throw new Error('export shape drift')` · handler catch 返 500）
3. **audit.ts**（~30 min）· 简单 INSERT，返 `{ id: number }`
4. **route.ts**（~45 min）· 从 `reference/skeletons/api-export-full-route.ts` 拷贝骨架；替换 TODO stub
5. **round-trip.ts**（~60 min）
   - 解析 stdin JSON → zod parse envelope
   - `dbWorker.transaction` 内按 FK 顺序 INSERT 12 张表
   - `paperSummaries` 插入时 **统一设 `llm_call_id: null`**（option (a) · F5 策略）
   - 对比 row counts → 打印 diff
6. **Tests**（~2h）
   - builder.test：12 keys 存在 + schemaVersion='1.1' + 不含 llmCalls/sessions/exportLog + 不含 inviteTokenHash
   - audit.test：成功 export 后 `export_log` +1 · failed build 后 `export_log` 不变
   - route.test：admin 200 + SEC-5（`?filename=../` 被拒）
   - round-trip.test（在 `tests/app/export` 下 · 走真实 Postgres）：export → drop llmCalls → import → `paperSummaries.llmCallId IS NULL` 全 true

## Known gotchas

- 大批量 SELECT 要流式响应？v0.1 规模 ≤ 几万行，单次 JSON.stringify 足够；若未来数据 > 100k 行改 streaming NDJSON
- `seats.invite_token_hash` / `invite_expires_at` / `invited_at` 必须排除；用显式 `.select({ id: seats.id, email: seats.email, role: seats.role, createdAt: seats.createdAt })` 而非 `select().from(seats)` 默认全字段
- SEC-5：**绝不接受** `?filename=` 参数；filename 由服务端生成固定 `pi-briefing-export-YYYYMMDD.json`
- **`buildFullExport(labId)`**（G3）：labId 是参数，不是 env；route handler 从 `session.labId` 传入；这让未来多 lab 部署（v1.0）无需重构
- FK 顺序 · import 时按 labs → seats → topics → papers → paperTopicScores → paperCitations → paperSummaries → briefings → actions → breadcrumbs → resurfaceEvents → fetchRuns
- **F5 round-trip 权威策略（option (a)）**：`paperSummaries.llm_call_id` 指向 `llmCalls`，但 envelope 不含 `llmCalls` → import 时一律 `llm_call_id = null`；schema 已 nullable + `on delete set null`；不建 stub rows（避免审计表污染）。option (b) 留 v0.2 升级
- **schemaVersion bump 规则**（api-contracts.md 权威）：1.x → 1.(x+1) 前向兼容；1.x → 2.0 breaking；round-trip 脚本需支持 1.0（跳过 paperSummaries）与 1.1（完整导入）
- compliance.md §5.3 规定整体删除前须 export；T023 是这条流程的前置

## Out of scope

- 不实现 import UI（脚本形式够用，v0.2 加 UI）
- 不支持 CSV / XML（IN-6 仅 JSON）
- 不做分库 export（v0.1 单 lab）
- 不做 per-seat export（v0.1 admin-level 全量）
- 不做自动定时 export（操作员按需触发）
- 不建 stub `llm_calls` rows at import 时（option (b) · v0.2）
