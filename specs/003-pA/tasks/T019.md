## Context

**O3 / C13 / C22 硬约束**(D-OP3 收窄):resume 成功率 > 95%,**仅限同机重启**(同 OS + 同 GPU + 同 CUDA 版本)。checkpoint 写在 `checkpoints/<id>/`(worktree 外),`state.json` 记录 phase + checkpoint path + budget 已用。`pars sft resume <id>` 基于这两个文件重建 worker env → 从中断点续跑。

**明示不支持(C22)**:跨机器(4090 ↔ H200)resume 不在本 task 范围;RecallKit `pars sft resume` 会**检测机器指纹**(GPU name + CUDA version 记入 state.json),mismatch → 拒绝 + 提示"跨机器迁移走 `docs/h200-rsync-playbook.md` 的远端重跑路径"。

关联 Outcome:O3;关联 Risk:R2(熔断跳过)、R10(手工迁 H200 走重跑不走 resume)。

  - **warning_fields**(mismatch 仅警告、不拒绝,防同机 patch 升级误判):

    - `os_release`(精确小版本变化,例 "25.2.0" → "25.2.1",但 os_major 不变)
  - `write_fingerprint(run_id, fp: dict)` — 原子写 `runs/<id>/machine_fingerprint.json`(不是 state.json 内嵌,独立文件便于 rsync 审计)

  - `diff_fingerprint(saved: dict, current: dict) -> tuple[list[str], list[str]]` — 返回 `(hard_mismatches, warn_mismatches)`;`hard_mismatches` 非空即 cross-machine;`warn_mismatches` 仅打印 warning 不拒绝
- `pars/orch/resume.py`:
  - `can_resume(run_id) -> tuple[bool, str]` — 检查:
    - state.phase != completed/failed
    - checkpoint 目录非空

## Implementation plan

1. `can_resume`:简单逻辑,看 phase + checkpoint 目录非空 + worktree 可重建
2. `prepare_resume`:
   - 读 config + state
   - 计算 remaining_budget = config.budget.usd_cap - state.usd_spent
   - 若 remaining_budget ≤ 0 → refuse(already over)
3. `execute_resume`:
   - 重建 worktree(T013 API;若 `.worktrees/<id>/` 还在,先 remove)
   - 注入 env `RECALLKIT_RESUME=1` + `RECALLKIT_CKPT_DIR=checkpoints/<id>`
   - worker system prompt 加一段"这是 resume,检测 checkpoints/last/ 存在则 `resume_from_checkpoint=True`"
   - 启动 worker 同 T023
4. CLI:
   - typer command 读 id → 调 `can_resume` → 若不能,退出码 2 + 说明
   - 若能,打印 plan 表(current phase / remaining budget)→ 确认 → 调 execute
5. 测试:用 TinyLlama 1.1B + alpaca 极小子集,seed=42 保证确定性

## Known gotchas

- `kill -9` 在训练 GPU 时 checkpoint 保存若未完成可能损坏 → Unsloth/SFTTrainer 是每 epoch 结束后原子写;epoch 中间 kill → fallback 到上一 epoch checkpoint(T016 模板 save_steps=epoch_end 确保这点)
- resume 时 HF cache 已在,不重下模型(HF_HOME 保留)
- state.phase 若是 `eval`(训练已完成,只差评测),resume 直接从 eval 开始,不重跑 LoRA
- **误差 < 1%** 依赖 fixed seed + 确定性 sampler + torch.use_deterministic_algorithms(True);测试 setup 要保证这些
