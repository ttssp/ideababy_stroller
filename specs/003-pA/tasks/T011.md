## Implementation plan

1. `build_worker_env`:
   - `base = os.environ.copy()`
   - 黑名单硬删除 `["ANTHROPIC_API_KEY", "ANTHROPIC_ADMIN_KEY"]`
   - 模式剔除:regex `(KEY|TOKEN|SECRET|PASSWORD|PASSWD|CREDENTIALS)` — 除白名单 `{"HF_TOKEN"}` 外全丢
   - 追加 `ANTHROPIC_BASE_URL`
   - 合并 `extra`(extra 能覆盖 base,但不能含 `ANTHROPIC_API_KEY`,要检)
2. `validate_worker_env_stripped`:
   - 扫字典,若任何 key == "ANTHROPIC_API_KEY" 或任何 value 以 "sk-ant-" 开头 → `SecurityError`
3. 测试用 `monkeypatch.setenv` / `monkeypatch.delenv` 造受控环境

## Known gotchas

- **不要**保留 `ANTHROPIC_*` 任何 env(包括未知的 ANTHROPIC_ORGANIZATION 等);只显式设 `ANTHROPIC_BASE_URL`
- `HF_TOKEN` 是白名单例外,但**默认场景无需**(Qwen3-4B / gsm8k 公开);仅 gated 模型才需,**要求 read-only scope**(architecture §5 + README)
- **无 HF_TOKEN 场景必须能跑通公开 demo**(R1 HIGH #1 验证点,见下方 Verification)
- 不用 `os.environ.pop`(会改变主进程 env),永远用 `os.environ.copy()` 衍生新 dict
- **macOS 无 `/proc/<pid>/environ`**:任何 env 断言测试一律用 `psutil.Process(pid).environ()`(跨平台),不能用 `/proc/<pid>/environ`(R1 Medium #1 修复)
