## Context

把 T010 (proxy) + T011 (env) + T012 (readonly .claude) 拼装,实现一个完整的 worker 生命周期:

1. 创建 git worktree(`.worktrees/<run-id>/`)
2. 挂只读 `.claude/` 到 worktree
3. 用剥离后的 env 启动 `claude -p ... --output-format stream-json --max-turns N`
4. 解析 stream-json stdout → 产出 `tool_use` / `assistant_message` 事件
5. 生命周期管理(SIGINT on demand / 等待 / 清理)

关联 Outcome:O1(跑通 baseline→LoRA→eval)、O3(resume 的基础);关联 Risk:R4+R6(组合实现)。

## Outputs

- `pars/orch/worktree.py`:
  - `create_worktree(run_id: str, base_branch: str = "main") -> Path` — `git worktree add .worktrees/<id> -b run/<id>`

- `pars/orch/worker.py`:
  - `class WorkerHandle`(pid, stdout_queue, stderr_queue, env_snapshot, start_time)
  - `launch_worker(run_id, prompt_path: Path, proxy_port: int, max_turns: int = 50) -> WorkerHandle`
    1. `create_worktree(run_id)`
    2. `ensure_readonly_claude_dir(worktree, "worker_claude_dir")`
    3. `env = build_worker_env(proxy_port, extra={"RECALLKIT_RUN_DIR": ..., "RECALLKIT_CKPT_DIR": ...})`
    4. `validate_worker_env_stripped(env)` — safety gate
    5. `subprocess.Popen(["claude", "-p", "--output-format", "stream-json", "--max-turns", str(max_turns), ...], env=env, cwd=worktree, stdout=PIPE, stderr=PIPE)`
    6. 启两个后台线程读 stdout/stderr 入 queue
  - `wait(handle, timeout=None) -> int`
  - `stop(handle, signal=SIGINT, grace=10s) -> int` — SIGINT → wait grace → SIGKILL
  - `cleanup(handle)` — release readonly handle + remove worktree
- `tests/unit/test_worktree.py` — worktree 创建 / 删除 / list(用 tmp git repo)
- `tests/integration/test_worker_lifecycle.py`(mark integration + slow):

  - 验证 env 不含 key
  - 验证 worktree 创建 + 清理
  - 验证 SIGINT 能优雅停

## Implementation plan

1. `worktree.py`:subprocess 调 git;error handling(已存在 worktree → cleanup → retry)
2. `stream_parser.py`:NDJSON 按行解析;tolerant 坏行(log warn + skip)
3. `worker.py` 是主 orchestration:顺序组合调用,**失败路径必须 cleanup**(readonly mount release + worktree remove),用 try/finally 或 context manager
4. stdout/stderr 读线程:非阻塞 `readline` + queue;worker 退出时线程 join
5. 测试用 mock Claude(一个 bash 脚本在 `$PATH` 前,输出几行 NDJSON 后 exit 0)

## Known gotchas

- `claude -p --output-format stream-json` 需要 Claude Code CLI 2026-04-latest(tech-stack §7);测试用 mock 绕过
- git worktree 同名冲突:先 `git worktree list` 查,存在即 `remove --force` 重建
- SIGINT 在 Windows 行为不同 — spec 只支 macOS/Linux,OK
- subprocess `cwd` 必须是**绝对路径**,相对路径 + 后续 `os.chdir` 会诡异
- `subprocess.Popen` 启动后,父进程必须阻塞(或读 stdout 线程)避免僵尸
