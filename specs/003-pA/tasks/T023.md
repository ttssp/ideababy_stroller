## Implementation plan

1. `env_snapshot.py`:`platform.python_version()` + `platform.platform()` + `subprocess(nvidia-smi --query-gpu=name ...)` + `subprocess(pip freeze | sha256sum)` — 捕获 subprocess 失败(mac 无 GPU → gpu_name=None)
2. `RunOrchestrator.start`:长方法,但每步清晰注释,try/finally 确保 cleanup
3. 错误路径:
   - ANTHROPIC_API_KEY 缺 → fail-fast(退出码 2)
   - **`.claude/` fail-closed 不可用(C21)** → 退出码 2 + 打印 OQ5 引导(refuse to start)
   - proxy 启动失败 → cleanup + exit
   - worker 启动失败 → cleanup
   - worker SIGINT(stuck or budget)→ state.phase=failed + cleanup + 返回 handle(同机可 resume)
   - **proxy 前置拒绝(C20)** → worker 收 402/429 → state.failure_reason=budget_prereject + cleanup
4. CLI:Typer 参数对齐 config schema;参数校验委托 Pydantic(写 config 时自校验)
5. 集成测试:mark `slow + gpu`,CI skip;操作员手跑(接受 30-90min)

## Known gotchas

- 长方法容易写烂;用 helper methods per step(`_validate_env`, `_create_ledger`, `_launch_infrastructure`, `_launch_worker`, `_monitor_until_done`, `_finalize`)
- 两个 monitor thread + 一个 worker subprocess → 退出顺序重要:worker 先 → monitor 线程 join(超时 10s)→ proxy stop
- Ctrl+C 要捕获(signal.SIGINT handler)→ 同步 cleanup,不能直接退出留垃圾
- `--name` 覆盖 ULID 要校验:合法目录名 + 未被占用
- rich.Table live update 在 CI 会乱行;用 `no_color` 或 fallback 到纯文本输出(env var `PARS_NO_COLOR=1`)
