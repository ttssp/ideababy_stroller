## Context

Ledger 的基础契约 = `config.yaml` 的 schema。所有下游模块(orch / report / compare / resume)都读写它,必须先稳定。

用 Pydantic BaseModel(已在 lm-eval / transformers 传递依赖里,无额外成本)确保 type-safe,**拒绝** 手写 dict。

关联:O2 报告 schema / O5 wall-clock + usd / O6 compare 的前置。

## Inputs

- `architecture.md §4.2` runs/<id>/ 布局
- `architecture.md §9` 报告 schema 的元数据字段清单
- `spec.md §4 D14` ULID 命名
- 上游:T004 paths,T005 CLI 占位

## Outputs

- `pars/ledger/config_schema.py` — Pydantic `RunConfig` 模型,含:
  - `run_id: str`(ULID,正则 `^[0-9A-HJKMNP-TV-Z]{26}$`)
  - `research_question: str`(非空,≤500 chars)
  - `base_model: str`(HF repo id,如 `Qwen/Qwen3-4B`)
  - `dataset: DatasetConfig`(hf_id, split, n_samples)
  - `training: TrainingConfig`(backend="unsloth", lora_rank, lora_alpha, lr, epochs, batch_size, max_seq_len)
  - `eval: EvalConfig`(backend="lm-eval", tasks: list[str], n_shot, limit)
  - `budget: BudgetConfig`(usd_cap, wall_clock_hours_cap, gpu_hours_cap)
  - `env_snapshot: EnvSnapshot`(python_version, platform, cuda_version, gpu_name, pip_freeze_sha256)
  - `created_at: datetime`, `started_at: datetime | None`, `completed_at: datetime | None`
  - `wall_clock_seconds: float | None`, `usd_total: float | None`, `gpu_hours: float | None`
- `pars/ledger/config_io.py`:
  - `read_config(run_id) -> RunConfig`
  - `write_config(run_id, cfg)` — atomic write(临时文件 + os.rename)
  - `update_config_field(run_id, key, value)` — 部分更新(用于 Budget tracker 刷 usd_total)
- `tests/unit/test_config_schema.py` — 8+ test 覆盖:合法 config、非法 run_id、缺 required field、atomic write、concurrent update 互斥(fcntl.flock)

## Implementation plan

1. `config_schema.py`:Pydantic v2 模型,`model_config = {"extra": "forbid"}`(拒绝未知字段)
2. `config_io.py`:
   - YAML 用 `yaml.safe_load` / `yaml.safe_dump`(T002 已加 pyyaml 依赖)
   - atomic write:写 `.tmp` + `os.replace`
   - `update_config_field` 用 `fcntl.flock` 排它锁,先读 → 改 → 写 → 放锁
3. 测试:用 `hypothesis` 生成随机 config?不,T003 没加 hypothesis(压依赖)。用手写 fixture 即可。

## Known gotchas

- Pydantic v1 vs v2 API 差异很大,确认 T002 装的是 v2;若是 v1 本 task 升级或用 v1 API
- `datetime` 字段序列化到 YAML 要用 ISO8601 字符串,不是 datetime 对象(YAML 的 datetime 支持跨实现不稳)
- `fcntl.flock` 在 Windows 不通,但 spec 只支持 macOS/Linux(tech-stack §7),OK
