## Outputs

- `pars/compare/engine.py`:
  - `compare_configs(a: RunConfig, b: RunConfig) -> list[DiffRow]` — 用 deepdiff + 扁平化,列出 `(field, value_a, value_b)`
  - `compare_metrics(a_metrics, b_metrics) -> list[DiffRow]` — 聚合 eval_scores 最后一次的 task_name × metric;列出对比表
  - `compare_conclusions(a_report_path, b_report_path) -> list[DiffRow]` — 抓两份 report.md 的 `## 决策` section 内容
  - `format_markdown(config_diffs, metric_diffs, conclusion_diffs) -> str`
- `pars/cli/compare.py`:
  - `pars compare RUN_A RUN_B [--output PATH]`(stdout 默认,--output 写文件)
  - 调 engine → print markdown
- `tests/unit/test_compare_engine.py` — ≥ 8 test,含:config diff 完全相等(空)、diff 1 字段、metric 缺失、report 缺 `## 决策`
- `tests/integration/test_compare_two_runs.py`:
  - 造两个 run(同 dataset,差 lora_rank=8 vs 16)
  - 调 `pars compare A B`
  - 断言 stdout 含 `## Config diff` / `## Metric diff` / `## Conclusion diff` 三个 H2
  - `lora_rank` 出现在 Config diff 表中

## Implementation plan

1. `compare_configs`:扁平化 nested dict(path='.'.join keys)→ deepdiff → map 到 DiffRow
2. `compare_metrics`:从 metrics.jsonl 抽 event==eval_scores 的最后一组;对齐 task_name × metric key
3. `compare_conclusions`:正则 `^## 决策\s*\n(.*?)(?=^##|\Z)` 抓内容
4. `format_markdown`:三个 H2 section,每 section 一 markdown table

## Known gotchas

- deepdiff 输出结构复杂(TypeChanges/ValuesChanged/...);需简化为 `(path, old, new)` 三列,扔掉复杂变更
- metric 对齐:两 run 跑不同 eval task → task_name 不匹配时显示 "(only in A)" / "(only in B)"
- report.md 可能缺 `## 决策`(T021 schema 校验失败的 run)→ 返回 "(no decision section)" + log warning

## Out of scope

- ❌ 不做 visual diff(纯 markdown)
- ❌ 不做多 run N-way compare(只 2-way)
- ❌ 不做 LLM-based summary(non-goal N5-1)
