## Implementation plan

1. 先写 prompt 模板(worker 的系统提示),因为决定模板用法流程
2. `baseline_script.py.j2`:
   - 仅 load model + run lm_eval;不训练
   - 5-6 行 metrics.jsonl append(用相对路径 `from pars.ledger.metrics import append_metric`?worker 可否 import pars?**否**:worker 环境可能不包含 pars package 的开发安装。**改用**:worker 直接 `open(os.environ["RECALLKIT_RUN_DIR"] + "/metrics.jsonl", "a")`,写 JSON 一行;T008 的 `append_metric` 只在 host 侧用
3. `lora_script.py.j2`:
   - Unsloth `FastLanguageModel.from_pretrained(<base>, load_in_4bit=True, max_seq_length={{max_seq_len}})`
   - `FastLanguageModel.get_peft_model(model, r={{lora_rank}}, ...)`
   - SFTTrainer with train dataset + save_steps=<每 epoch>
   - 每 epoch callback 写 metric
   - checkpoint save: `trainer.save_model(os.environ["RECALLKIT_CKPT_DIR"] + "/last")`
4. `eval_script.py.j2`:
   - `lm_eval.simple_evaluate(model_path, tasks=[...], num_fewshot={{n_shot}}, limit={{limit}})`
   - 写 scores.json + metrics.jsonl
5. `render.py`:标准 Jinja2 + StrictUndefined
6. 测试:
   - unit:render 出来的字符串里含预期变量值,missing var → UndefinedError
   - integration smoke:真实跑,断言三类 metrics 事件存在

## Known gotchas

- Unsloth 的 `FastLanguageModel` API 在 2026.3.x 稳定;pin 到 tech-stack §8
- `load_in_4bit=True` 需要 CUDA + bitsandbytes,mac 开发机单测只能 mock
- lm_eval `simple_evaluate` 签名随版本变;pin 0.4.5
- `SFTTrainer` 的 `resume_from_checkpoint=True` 需要 trainer 识别目录格式 — 测试要验真能 resume(T019 integration test 覆盖)
- **关键**:worker 渲染的脚本**绝不**含 API key / HF token 字面量,所有 secret 通过 env 注入
