## Implementation plan

1. `conftest.py`:
   - session-scope fixture `_prepare_hf_cache`:提前下载 TinyLlama 到 HF_HOME(避免每 test 重下)
   - function-scope fixture `tmp_run_env`:设 `RECALLKIT_RUN_DIR=tmp_path/runs` 等
2. 每个 test 文件只包含 1-2 个 test,薄 wrapper 调上游模块
3. Markers:
   - `@pytest.mark.e2e` 全部
   - `@pytest.mark.slow` 真实 GPU 跑
   - `@pytest.mark.gpu` 需要 GPU,无则 skip
4. CI 跑 `pytest -m "e2e and not slow and not gpu"`(合成版)
5. 操作员本机跑 `pytest -m "e2e"` 全套

## Verification

- [ ] `uv run pytest tests/e2e -m "e2e and not slow and not gpu" -v` 在 CI 全绿,< 10min
- [ ] `uv run pytest tests/e2e -m e2e -v` 操作员本机全绿(含真实 GPU ≈ 1-2h)
- [ ] 5 个 Outcome(O1 O2 O3 O6 O7)每个至少 1 个 e2e 测试 pass
- [ ] 覆盖率:整体 `uv run pytest --cov=pars` ≥ 75%(高于 SLA §4 gate 的 70%)

## Known gotchas

- HF cache 预下载耗磁盘;CI 跑合成版,不下模型
- 真实 GPU test 在 GitHub Actions 无法跑(CI 无 GPU);只能操作员本机跑 + README 注明
- e2e test 失败诊断难;每 test 里加足够 log + 保留 tmp run dir(teardown 不 rmtree,只 log 路径)
- pytest collect 时 mark 过滤确认:`--collect-only -m "e2e and not gpu"` 看清 skip 列表
