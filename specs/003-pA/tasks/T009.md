## Context

把 T006 (config) / T007 (state) / T008 (metrics) 三块拼成一个 `Ledger` 外观模块,**隐藏**文件系统细节给上层 orch / CLI / report 用。同时实现 `pars sft status <run-id>` — Outcome 观测入口,SLA §1.3 响应 < 2s。

关联:O3 resume 需要 status 先工作;所有 phase 1+ task 通过 Ledger 读写。

## Inputs

- `architecture.md §4.2` 全 runs/<id>/ 布局
- `SLA.md §1.3` `pars status` < 2s
- 上游:T005 CLI 占位,T006 config,T007 state,T008 metrics

## Outputs

- `pars/ledger/ledger.py`:
  - `class Ledger`:
    - `create(run_id, config)` — `ensure_run_tree` + `write_config` + 初始 state
    - `load(run_id) -> LedgerView`(只读快照)
    - `list_runs(limit=20, filter_phase=None) -> list[LedgerView]` — 按 ULID 时间倒序
    - `ensure_exists(run_id)` — 给 resume / retry 用的校验
    - 暴露 `config`, `state`, `metrics` 三个属性(懒加载)
  - `LedgerView` = dataclass(config, state, metrics_tail)
- `pars/cli/status.py`:
  - `pars sft status <run-id>` → 用 rich.Table 打印核心字段(phase / elapsed / usd / gpu-h / stuck_state / 最后 5 条 metrics)
  - `pars sft status` 不传 id → 打印 `list_runs(limit=10)` 一览表
- `tests/unit/test_ledger.py` — 覆盖 create / load / list / ensure
- `tests/unit/test_cli_status.py` — CliRunner 测 happy path + 不存在 run_id 的错误消息

## Implementation plan

1. `Ledger` 类是外观,**不重复**底层逻辑 — 调用 T006/T007/T008 的函数
2. `list_runs`:`Path("runs").glob("*")`,过滤有 `config.yaml` 的目录,按目录名(ULID,自带时序)排序
3. `pars sft status <id>`:
   - `rich.Table`,列 = ["字段","值"]
   - 计算 `elapsed_human = timedelta(seconds=state.wall_clock_elapsed_s)` 格式化
   - `needs_human_review=True` 时用红色 + 醒目提示
4. `pars sft status`(无 id):
   - `rich.Table`,列 = ["run_id","phase","elapsed","usd","stuck"],最多 10 行
5. 测试用 `tmp_runs_dir` fixture(T003 已提供)

## Verification

- [ ] `uv run pytest tests/unit/test_ledger.py tests/unit/test_cli_status.py -v` 全绿
- [ ] 手动:`uv run pars sft status 00000000000000000000000000` → 退出码 2 + 打印"run 不存在"
- [ ] 手动:用 `Ledger.create` 造一个假 run,再 `pars sft status <id>` → rich 表显示正确字段
- [ ] `pars sft status` 无参 → 列表打印 ≥ 1 行(若有 run)
- [ ] 性能:造 100 个假 run,`time pars sft status` < 2s

## Known gotchas

- `list_runs` 只读目录名排序 = 按 ULID 时间正确,无需 stat() 每目录(对 100+ run 重要)
- rich.Table 宽度在窄终端会溢出,用 `table.show_lines=True` + `no_wrap=False`
- Status 命令**必须不动任何状态**(纯读,不更新 last_update);写操作留给 orch
