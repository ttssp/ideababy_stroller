## Context

落地 Typer app 骨架 + 所有子命令占位(`pars sft {start,status,retry,report,compare,resume}`),使后续 CLI 命令 task(T006 / T023 / T024 / T025)可以并行填空。对齐 **spec §2.1** CLI 命令套。

关联:Outcome = 全部(所有用户交互入口)。

- `spec.md §2.1` CLI 命令套列表
- `tech-stack.md §1` Typer + rich
- 上游:T004 的 `pars/cli/__init__.py` 与 `pars/logging.py`

## Outputs

- `pars/cli/main.py` — Typer app,含:
  - 顶层 `app = typer.Typer(name="pars", help="RecallKit v0.1 · 单人后训练决策循环")`
  - `sft = typer.Typer(name="sft", help="SFT 子命令")`,挂在 `app`
  - `sft.start`, `sft.status`, `sft.retry`, `sft.report`, `sft.compare`, `sft.resume` 6 个占位命令(每个仅 `raise NotImplementedError("T006 负责实现")` 之类)
  - `app.callback()` 启动时调 `pars.logging.configure_root()`
- `pars/cli/_common.py` — 共享工具:`validate_run_id(ulid_str)`, `print_run_dir(path)`, `rich_table_for_config(d)`
- `tests/unit/test_cli_smoke.py` — 烟雾测试:`pars --help` / `pars sft --help` / 6 个子命令 `--help` 都 exit 0 + stdout 含关键字

## Implementation plan

1. `main.py`:
   - `import typer; from rich.console import Console`
   - 两个 Typer 实例(顶层 + sft 子组)
   - 6 个 `@sft.command()` 占位,每个只含 `typer.echo("T00X 未实现"); raise typer.Exit(1)`
   - `app.callback()` 里调 logging init
   - 文件末尾 `if __name__ == "__main__": app()`
2. `_common.py`:3 个小工具函数
3. 测试用 `typer.testing.CliRunner`:`runner.invoke(app, ["--help"])` 断言 exit_code==0 + 检查 stdout 含 `sft`

## Known gotchas

- Typer 的 command 占位必须有 docstring,否则 `--help` 不优雅;每个占位都写 "TODO: T00X 实现"
- 不在此 task 内实现任何真实逻辑;占位用 `raise typer.Exit(1)` 而不是 `raise NotImplementedError`,避免 CLI 栈回溯吓用户
