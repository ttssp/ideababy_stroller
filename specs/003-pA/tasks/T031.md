## Context

fleet mode 在单机 N 张 GPU 上并发跑 N 个 worker。**单 GPU 同时只能 1 个 worker**(否则 OOM 误杀),需要 lock。lock 必须能跨 process(fleet 命令 fork worker 后 fleet 退出,worker 仍持锁)。

对应 O9(GPU lock 不发生 OOM)、C23、C26、R12、R13。

```python
@dataclass
class LockData:
    pid: int
    start_time: float            # process start_time(/proc/PID/stat 或 psutil),用于 PID 复用检测
    worker_run_id: str
    acquired_at: datetime

class GpuLock:
    """单 GPU 文件锁;fcntl.flock + atomic file create + stale detection."""
    
    def __init__(self, gpu_uuid: str, locks_root: Path): ...
    
    def acquire(self, worker_run_id: str, *, timeout_s: int = 0) -> bool:
        """timeout_s=0 → 非阻塞,失败立即返回 False
           timeout_s>0 → 阻塞最多 timeout_s,期间每 5s 重试 + 检测 stale
           成功 → 写 LockData 到 runs/.gpu_locks/<gpu_uuid>.lock"""
    
    def release(self): ...
    
    def __enter__(self): self.acquire(...); return self
    def __exit__(self, *_): self.release()
    
    @property
    def is_held(self) -> bool: ...
    
    @staticmethod
    def list_active(locks_root: Path) -> list[tuple[str, LockData]]:
        """T029 fleet status 调这个;返回 (gpu_uuid, lock_data) list,自动剔除 stale"""

def is_stale(lock_data: LockData) -> bool:
    """检测 PID + start_time 是否仍是同一 process
       1. psutil.pid_exists(pid) is False → stale
       2. psutil.Process(pid).create_time() != start_time(±1s tolerance) → stale (PID 复用)
       否则 → alive"""

def force_unlock(gpu_uuid: str, locks_root: Path):
    """operator 手工解锁(R13 兜底);写 audit log."""
```

## Known gotchas

- macOS 上 `psutil.Process.create_time()` 精度可能不够,start_time 比较用 ±1s tolerance
- fcntl.flock 在 NFS 上可能失效,但 fleet 是单机场景,本地文件系统(ext4/apfs)都 OK
- `runs/.gpu_locks/` 目录如果不存在,acquire 应该 mkdir(parents=True, exist_ok=True)
- worker_run_id 写到 lock 文件后,T029 fleet status 可关联"哪个 GPU 在跑哪个 run"
- fallback UUID(无 nvidia-smi)会让 lock 退化到"hostname+idx",多机器 NFS 共享 runs/ 时可能误锁;**v0.2 不支持多机共享 runs/**(C22 跨机硬拒绝原则),所以这个不是问题
