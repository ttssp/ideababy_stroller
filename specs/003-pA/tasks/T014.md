## Implementation plan

1. 重写 `pre_tool_use.sh` 用 bash regex(`[[ ... =~ ... ]]`),每条 deny 带 comment 说明来源(如 `# deny rm -rf : architecture §6`)
2. hook 最后一行 `exit 0`(allow),任何 match 用 `echo "DENY: $reason" >&2; exit 2`
3. 测试:把 hook 当普通 bash 脚本跑,构造 JSON 输入(`echo '{...}' | bash pre_tool_use.sh`),断言 exit code
4. `test_worker_cannot_write_claude_dir` 用 mock claude 模拟尝试 `Edit(.claude/settings.json)` 动作,断言 hook 拦截 + 真实文件系统 EACCES 双保险

## Known gotchas

- bash regex 非贪婪 / 空格问题:用 `[[ "$cmd" =~ ^rm[[:space:]]+-[rRfF]+[[:space:]]+/ ]]` 而不是简单字符串匹配
- `curl http://localhost:...` 要 allow(T010 proxy 通信),`curl http://example.com` 要 deny;白名单用户 IP = 127.0.0.1 / localhost
- 测试 hook 时,Claude Code hook JSON schema 细节需真实对齐(可 mock 一个 JSON)
