## Implementation plan

1. 先写 `run_config.yaml`:
   - base: Qwen/Qwen3-4B
   - dataset: gsm8k (train[:100])
   - lora_rank=16, alpha=32, lr=2e-4, epochs=3
   - eval tasks = [gsm8k] n_shot=5 limit=100
   - budget: usd_cap=10, wall_clock=8h, gpu_hours=8
2. 写 `scenario.md`:解释 research_question(验证 Qwen3-4B 在 gsm8k-100 LoRA 能否提升 5%+ accuracy),预期 happy path + fallback
3. **真实跑一次**:操作员本机跑 `pars sft start --config demo/.../run_config.yaml` → 抓 `runs/<id>/report.md` 的数字填 `expected_metrics.md`(不 mock,不编造)
4. README 按 Outputs 结构写
5. `docs/demo-reproduction.md`:clone + uv sync + `pars sft start ...` 最小步骤
6. `docs/h200-rsync-playbook.md`(**仅"远端重跑"手工 how-to,不写跨机 resume**):
   - `rsync -av runs/<id>/ user@h200:/remote/runs/<id>/`
   - `rsync -av checkpoints/<id>/ user@h200:/remote/checkpoints/<id>/`(可选,含 HF cache 的 rsync)
   - H200 侧:`RECALLKIT_RUN_DIR=/remote/runs/<id> RECALLKIT_CKPT_DIR=/remote/checkpoints/<id> uv run python artifacts/lora_script.py`(**从零重新训练**)
   - rsync 回本机
   - **明示不支持章节**:"⚠️ `pars sft resume` 不支持跨机器;本 playbook 只教'重跑训练',不是'resume'。若需同机续跑,见 `pars sft resume <id>`。原因 = v0.1 不保证 optimizer state / RNG / CUDA 版本跨机器一致性(C22)。"
7. 如果实测 Qwen3-4B OOM,fallback 到 TinyLlama 1.1B + 文档说明(R1 缓解的兑现)
