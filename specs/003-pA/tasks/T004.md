## Context

把 `architecture.md §4.1` 的包布局落成空骨架,让下游 task 并行 import 各自模块不撞空目录。同时落地**统一 logging**(structlog or stdlib + JSON formatter),后续所有模块走统一 logger。

还建立 `pars/paths.py`:**路径可移植层**(D18 / R10 缓解)— 所有 `RECALLKIT_RUN_DIR` / `RECALLKIT_CKPT_DIR` / `HF_HOME` 的 env-var 解析与默认值都在这里。

## Inputs

- `architecture.md §4.1` 包布局
- `architecture.md §10` 路径可移植性规则
- 上游:T001

## Outputs

- `pars/{cli,orch,ledger,proxy,report,compare,safety,budget,stuck,workflow}/__init__.py` — 空包(每个含 `"""模块简介"""` docstring)
- `pars/logging.py` — `get_logger(name)` 返回配置好的 logger;支持 `PARS_LOG_LEVEL` env var;JSON lines 格式输出到 stderr(避免污染 stdout 解析)
- `pars/paths.py`:
  - `run_dir(run_id: str) -> Path` — 优先 `$RECALLKIT_RUN_DIR`,否则 `./runs/<id>/`
  - `ckpt_dir(run_id: str) -> Path` — 优先 `$RECALLKIT_CKPT_DIR`,否则 `./checkpoints/<id>/`
  - `worktree_dir(run_id: str) -> Path` — `./.worktrees/<id>/`(**无** env override,worktree 必须本地)
  - `hf_home() -> Path` — 优先 `$HF_HOME`,否则 `~/.cache/huggingface`
  - `repo_root() -> Path` — 通过 `git rev-parse --show-toplevel`
- `templates/.gitkeep` / `worker_claude_dir/.gitkeep` — 占位让 git 跟踪空目录
- `tests/{unit,integration,e2e}/__init__.py` — 空
- `tests/unit/test_paths.py` — 覆盖 env override 与默认值

## Implementation plan

1. 批量 `mkdir -p` + `touch __init__.py`(按 file_domain 列表)
2. `pars/logging.py`:用 stdlib `logging` + `json.dumps` formatter;export `get_logger(name)` 与 `configure_root()`;`configure_root()` 在 CLI 启动时调用一次
3. `pars/paths.py`:
   - 顶部 `from pathlib import Path; import os`
   - 函数实现如 Outputs 所述
   - **关键**:所有函数**只返回路径对象,不创建目录**(由调用方负责 mkdir;测试友好)
   - 另外暴露 `ensure_run_tree(run_id)` 辅助函数,创建所有子目录
4. `tests/unit/test_paths.py`:至少 6 个 test,覆盖 env var 有/无两种情形 × 4 个函数

## Known gotchas

- `paths.run_dir` 返回**绝对路径**(用 `Path(...).resolve()`),否则 worker cwd 切换后路径失效;测试要锁这个性质
- 日志默认只输出到 stderr,因为 stdout 会被 `claude -p --output-format stream-json` parser 解析(T008)
- 不引入 structlog(压依赖预算,用 stdlib 够)
