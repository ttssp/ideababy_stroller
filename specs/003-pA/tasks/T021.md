## Context

**O2 的实现**:markdown 决策报告**必有**训练曲线 PNG + 分数对比表 + 失败归因文字。schema 校验在 CI 跑(SLA §4 gate)。

`pars sft report <run-id>` 把 config + metrics + failure_attribution 拼出最终 `report.md`。

关联 Outcome:O2+O5(wall_clock + usd 元数据);关联 Risk:R3 (schema 校验给失败归因兜底)。

## Inputs

- `architecture.md §9` 报告 schema 全文
- `SLA.md §1.1` 报告 schema 合规 = 100%
- 上游:T008 metrics,T020 failure schema

## Outputs

- `templates/report.md.j2` — Jinja2 模板,对齐 architecture §9 schema(元数据 / 训练曲线 / 分数对比 / 失败归因 / 决策)
- `pars/report/charts.py`:
  - `render_training_curve(metrics: list[dict], output: Path) -> Path` — matplotlib PNG(train_loss + eval_loss vs epoch 双线)
  - `render_score_comparison(metrics: list[dict], output: Path) -> Path` — 可选,v0.1 用 markdown table 即可,PNG 不强制
- `pars/report/renderer.py`:
  - `render_report(run_id) -> Path`
    - 读 config / state / metrics / failure_attribution
    - 调 `render_training_curve` → `runs/<id>/artifacts/training_curve.png`
    - Jinja2 render `report.md.j2` → `runs/<id>/report.md`
    - 返回 report path
- `pars/report/schema_validator.py`:
  - `validate_report_schema(report_path: Path) -> tuple[bool, list[str]]`
  - 断言:
    - H2 sections 存在:`## 元数据` / `## 训练曲线` / `## 分数对比` / `## 失败归因` / `## 决策`
    - `![training curve]` 引用的 PNG 文件存在
    - `## 分数对比` 下有 markdown table(`|.*|.*|`)
    - `## 决策` 段含关键字 [继续|停止|改方向] 之一(non-greedy)
    - **failure_attribution.md 存在且 `validate_quality()` 返回 `(True, [])`** — **严格 gate**(R1 Medium #2:与 T020 contract 统一,不是警告);若 failure 归因质量不达标,**report 整体 schema fail**,O2 不通过
- `tests/unit/test_report_schema_validator.py` — ≥ 10 test
- `tests/integration/test_report_end_to_end.py`:
  - 用 fixture 造假 run(config + metrics + failure_attribution) → render_report → validate_report_schema == True
  - 人工去掉 `## 失败归因` section → validate False

## Implementation plan

1. `charts.py`:matplotlib 无头模式(`Agg` backend),避免 GUI 依赖
2. `templates/report.md.j2`:严格对齐 architecture §9 每个 H2 + 字段
3. `renderer.py`:顺序读数据 → 调 charts → 调 Jinja → 写 report.md
4. `schema_validator.py`:
   - 用 `mistune` 或简单正则分 H2 sections?不装新依赖 — 用简单正则 `^## (.*)$`,m=MULTILINE
   - 断言必要 H2 全存在;其他元素用正则检
5. 测试用合成 run,覆盖所有 H2 missing 场景

## Known gotchas

- matplotlib `Agg` backend 不需 DISPLAY;文件顶部 `matplotlib.use("Agg")`
- 中文字符在 matplotlib PNG 里需字体配置;v0.1 标题用英文避坑(内文中文无问题)
- Jinja2 里 `{% for m in metrics %}` 若 metric 数据类型漂移会炸;template render 前 dict 规范化
- 报告不允许 LLM 美化(PRD §8 UX 原则 2);本 task **不**用 LLM 调用,纯 template
