## Context

**R5 pip 投毒 / C17 硬约束 / D12** 的实现(R1 HIGH #3 pip/uv 契约统一 · 对齐 architecture.md §7):worker 内**仅允许**以下三种等价命令:

1. `pip install -r requirements-locked.txt --require-hashes`
2. `uv pip install -r requirements-locked.txt --require-hashes`
3. `uv sync --frozen`(等价 — `--frozen` 强制按 `uv.lock` 精确安装,含 hash 校验)

其他格式(`pip install <pkg>` / `pip install git+` / `pip install --index-url` / `python -m pip install` / `easy_install` / `pipx install` / `uv pip install <pkg>` 等)**一律 deny**。

**唯一真相源 = `architecture.md §7`**。spec §2.1 / C17 / D12 + 本 task + T012 hook 占位全部回指 §7。

T012 做了占位,本 task 做严谨的命令解析 + 可测的 policy 模块。

## Outputs

- `pars/safety/pip_policy.py`:
  - `is_pip_install_allowed(cmd: str) -> tuple[bool, str]` — 纯函数,返回 (allowed, reason)
  - **白名单(对齐 architecture §7,3 条合法 + 空格宽容 normalize)**:
    1. `shlex.split(cmd)` 按顺序 strictly equal to `["pip","install","-r","requirements-locked.txt","--require-hashes"]`
    2. 或 `["uv","pip","install","-r","requirements-locked.txt","--require-hashes"]`
    3. 或 `["uv","sync","--frozen"]`
  - **禁止任何额外参数**(如 `-v`、`--quiet`、`--no-cache-dir`)以避免参数注入绕过

  - 读 stdin JSON → 提取 `.input.command`
  - 调 `python -c "from pars.safety.pip_policy import is_pip_install_allowed; import sys; ok,r=is_pip_install_allowed(sys.argv[1]); sys.exit(0 if ok else 2)" "$cmd"`
  - 或纯 bash 实现(若不想让 hook 依赖 Python module 可导入)
- `tests/unit/test_pip_policy.py` — ≥ 12 test:

## Implementation plan

1. `is_pip_install_allowed` 核心:shlex.split → normalize → match allowlist
2. allowlist 用 tuple 比较(避免 regex 被绕过)
3. 考虑 `python -m pip install ...` / `easy_install` / `pipx install` 变体 — 全 deny(也不 allow)
4. hook 用纯 bash 或 Python one-liner,取决于 worker 是否能 import pars(worker 的 cwd 是 worktree,sys.path 需含项目根;用 Python 方便测试)

## Known gotchas

- shlex.split 在 Windows 行为不同,但不支持 Windows
- `pip install . --require-hashes` 安装本地包?deny(严格)
- `uv pip install` / `uv sync --frozen` 由 architecture §7 **显式**允许,本 task 不再自行决策,直接对齐 §7 三条白名单
- `uv pip install <pkg>` 不等价 allow(需 `-r requirements-locked.txt --require-hashes` 才算)

## Out of scope

- ❌ 不做运行时网络层拦截(不做 seccomp / iptables)
- ❌ 不做 hash 内容校验(pip `--require-hashes` 自己做;本 task 只确保命令被迫用此 flag)
