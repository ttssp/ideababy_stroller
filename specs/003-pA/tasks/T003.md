## Context

质量 gate 的基础设施(SLA §4 Error budget → coverage < 70% PR block / ruff 失败 PR block)。对齐 CLAUDE.md 个人偏好("Conventional Commits / 中文注释")。

关联:Outcome = 全部间接(无此基础 CI gate 无法执行);不直接缓解 risk。

## Inputs

- `tech-stack.md §1` 指定 ruff / pytest / pre-commit / pytest-cov
- 上游:T001 骨架

## Outputs

- `pyproject.toml` 追加 `[tool.ruff]`(line-length=100, target=py312, select=[E,F,I,UP,B,S,PL], ignore=[],src=["pars","tests"])
- `pyproject.toml` 追加 `[tool.pytest.ini_options]`(testpaths=["tests"], addopts="-ra --strict-markers --cov=pars --cov-report=term-missing --cov-fail-under=70", markers=["integration","e2e","slow"])
- `pyproject.toml` 追加 `[tool.coverage.run]` + `[tool.coverage.report]`
- `.pre-commit-config.yaml` — 挂 ruff / ruff-format / Conventional Commits(commitizen 或 commit-msg hook)
- `tests/__init__.py` 空
- `tests/conftest.py` — 共享 fixture(`tmp_runs_dir`, `mock_anthropic_key`, `fake_gpu_snapshot`)

## Implementation plan

1. `[tool.ruff]` 设 `line-length=100`, target-version="py312", 选 rule set `["E","F","I","UP","B","S","PL","RUF"]`,`[lint.per-file-ignores]` 放 tests 放宽 `S101`
2. `[tool.pytest.ini_options]`:testpaths, addopts 含 `--cov-fail-under=70`,markers 声明 `integration` / `e2e` / `slow`(这三种 marker 后续 task 用)
3. `[tool.coverage.run]`:`source=["pars"]`, `omit=["pars/**/__main__.py"]`
4. `.pre-commit-config.yaml` 3 个 hook:ruff check、ruff format、commit-msg(Conventional Commits,用 `conventional-pre-commit`)
5. `tests/conftest.py` 定义:
   - `tmp_runs_dir`(tmp_path 下创 `runs/` 结构)
   - `fake_anthropic_key`(monkeypatch `ANTHROPIC_API_KEY=sk-test`)
   - `fake_gpu_snapshot`(返回 dict 模拟 `nvidia-smi` 输出,给 T017 用)

## Verification

- [ ] `uv run ruff check .` 0 error(可能只 warnings,可忽略)
- [ ] `uv run ruff format --check .` 0 diff
- [ ] `uv run pytest --collect-only` 显示 0 test collected(此刻无 test,正常)
- [ ] `uv run pre-commit install` 成功
- [ ] `git commit -m "broken commit"` 被 commit-msg hook 阻止
- [ ] `git commit -m "feat: init"` 通过

## Known gotchas

- `S` 规则(flake8-bandit)在 tests 下误报多,用 `[lint.per-file-ignores]` 放宽
- pytest `--cov-fail-under=70` 在 T001/T002 阶段会让所有 pytest 失败;在 `addopts` 里加 `--cov-fail-under=70` 仅 CI 启用,本地用 `pytest --no-cov` 绕过,或开关放 env var
- Conventional Commits 用 `conventional-pre-commit` 仓库,version pin
