## Context

直接缓解 **R5 pip 供应链投毒**(Critical/High);实现 **spec C17 / D12** 硬约束 = worker 内 `pip install` 仅允许 `-r requirements-locked.txt --require-hashes`。

对齐 **tech-stack.md §1 + §6** 的"每依赖 SHA-256 hash""exact pin"。

关联:R5、R8(训练后端踩坑需版本 pin);所有后续需要依赖的 task 都上游于此。

## Inputs

- `tech-stack.md §8` 版本表参考
- 上游:T001 的 `pyproject.toml` 骨架
- 无需 env var

## Outputs

- `pyproject.toml` — runtime 组填 unsloth / transformers / peft / trl / accelerate / bitsandbytes / datasets / lm-eval / typer / rich / jinja2 / matplotlib / deepdiff / psutil / litellm / ulid-py / pyyaml;dev 组填 ruff / pytest / pytest-cov / pre-commit / pip-audit / pip-licenses
- `uv.lock` — exact versions + hashes(由 `uv lock` 生成)
- `requirements-locked.txt` — 由 `uv export --format requirements-txt --hashes` 派生,**每依赖必含 SHA-256 hash**
- `scripts/lock-deps.sh` — 单入口脚本:`uv lock && uv export --hashes > requirements-locked.txt`
- `docs/dependency-policy.md` — 说明升级流程(操作员手工 `uv add` 后重跑脚本,禁止 worker 直接改依赖)

## Implementation plan

1. 用 `uv add` 追加所有 runtime 依赖(全部 exact pin `==X.Y.Z`,版本参考 tech-stack.md §8)
2. 用 `uv add --dev` 追加 dev 依赖(dev 允许 `>=`)
3. 跑 `uv lock` 生成 `uv.lock`
4. 跑 `uv export --format requirements-txt --hashes --no-dev > requirements-locked.txt`
5. 写 `scripts/lock-deps.sh`(≤ 20 行,含 `set -euo pipefail`)
6. 写 `docs/dependency-policy.md`:
   - 升级流程:`uv add <pkg>==<ver>` → `bash scripts/lock-deps.sh` → review diff → commit
   - worker **禁止**改 `pyproject.toml` / `requirements-locked.txt`(hook 拦截在 T015)
   - 每月 `uv run pip-audit -r requirements-locked.txt` 扫描
7. 统计总依赖数,若 > 30 runtime 或 > 10 dev,回到操作员讨论削减

## Verification

- [ ] `uv sync --frozen` 成功(证明 lock 自洽)
- [ ] `grep -c "sha256:" requirements-locked.txt` ≥ 运行依赖传递闭包数(每依赖至少 1 个 hash)
- [ ] `head -20 requirements-locked.txt` 肉眼看到 `unsloth==`, `transformers==`, `lm-eval==` 等
- [ ] `uv run pip-audit -r requirements-locked.txt` 无 critical/high(若有,本 task 阻塞升级)
- [ ] `bash scripts/lock-deps.sh` 可重入,不产生 diff(已锁)
- [ ] `wc -l requirements-locked.txt`:文件非空

## Known gotchas

- Unsloth 依赖链含 CUDA-specific wheel;若 `uv lock` 在 macOS 开发机上报 unresolvable,需在 `pyproject.toml` 用 `[tool.uv]` 声明 `environments` 多平台(显式区分 linux + macOS)。**如果多平台环境声明也无法锁(罕见),本 task fail**(不再注释 Unsloth 降级 — 会弱化本机路径可信度,R1 Medium #4);操作员需回看 **OQ1** 决定改选 Axolotl,或升级/降级 Python/CUDA 版本使 Unsloth 可锁。参考 architecture §13(Unsloth 要求)+ tech-stack §2
- `lm-eval` 的 extras 需要选:推荐仅 `lm-eval>=0.4.5`(不带 extras,避免引入 openai client 之类多余包)
- bitsandbytes 仅 Linux CUDA 可装;macOS 本地开发时 `uv sync --no-optional-deps` 跳过
- hash 格式必须是 SHA-256,不能混 MD5
