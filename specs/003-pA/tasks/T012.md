## Context

**R6 / C16 / C21 硬约束** 的实现(R1 BLOCKER #3 fail-closed 改写 · D-OP2):防 worker 修改自己的 hooks 让 deny 失效(Sakana v1 真实事故)。

**决策升级(对齐 spec v0.2 C21 · D-OP2)**:
- **首选**:bindfs `-o ro`(macOS + MacFUSE)/ `mount --bind + remount ro`(Linux)— 真只读 mount
- **fallback**:sudo `chflags uchg`(macOS)/ sudo `chattr +i`(Linux)— immutable flag
- **纯 `chmod -R a-w` 不是可接受的生产路径**(Python `os.chmod` 面可绕过,worker 允许 `Bash(python:*)`)
- 二者都不可用 → **CLI refuse to start**(退出码 2 + 指向 OQ5)

**主验证**:在允许的 Python 面下尝试改回 `.claude/` 可写必须失败(EACCES/EPERM)。SHA-256 post-run 校验降级为第二道验证。

**安全 P0 task**,Opus + 手工 review。

关联 Risk:R6(Critical/High);关联 Outcome:前置给所有 worker 启动路径。

## Implementation plan

1. 写 `worker_claude_dir/settings.json`:
   ```json
   {
     "permissions": {
       "allow": ["Read(*)", "Write(.worktrees/**)", "Write(runs/**)", "Write(checkpoints/**)", "Bash(python:*)", "Bash(uv:*)", "Bash(pip install -r requirements-locked.txt --require-hashes)"],
       "deny": ["Bash(rm -rf:*)", "Bash(curl:*)", "Bash(wget:*)", "Bash(cat:*.env*)", "Read(*.env)", "Read(*.pem)", "Read(*.key)", "Bash(chmod:*)", "Bash(chflags:*)", "Bash(chattr:*)", "Bash(sudo:*)", "Write(.claude/**)", "Edit(.claude/**)"]
     },
     "hooks": {
       "preToolUse": ["./hooks/pre_tool_use.sh", "./hooks/pre_pip_install.sh"]
     }
   }
   ```
2. `pre_tool_use.sh`:bash 脚本,读 stdin JSON(Claude Code hook 协议),若 `.tool.name == "Bash"` 且 `.input.command` 匹配危险正则 → `exit 2`;否则 `exit 0`
3. `pre_pip_install.sh`:同上,仅检 `pip install`,allowlist = `^pip install -r requirements-locked\.txt --require-hashes$`
4. `readonly_mount.py`:
   - 用 `platform.system()` 分支
   - 尝试 bindfs(macOS) 或 mount --bind(Linux) → 失败则 cp -R + 不可变位
   - **关键**:`ensure_readonly_claude_dir` 返回 handle,`release` 可回滚;保证 fail 路径不留 uchg 文件阻止后续 rm
5. `integrity.py`:递归 walk,对每文件算 sha256,再把 `(rel_path, sha)` 列表排序 json.dumps → 总 sha

## Known gotchas

- bindfs 在 CI runner **几乎必然无权限**(需 FUSE kext 加载);CI 测试用 `RECALLKIT_ALLOW_UNSAFE_CHMOD=1` 跑 chmod 路径,但**标记 banner**警告"这不是生产路径"
- macOS `chflags uchg` 文件 `rm -rf` 会失败,release 时必须先 `chflags nouchg` 再 rm(否则留垃圾)
- Linux `chattr +i` 需要 root,**非 root 环境 C21 要求 refuse to start**,不再自动降级
- OQ5 装 MacFUSE 需用户重启一次(kext 加载);首次启动自动检测并给友好提示
- Claude Code 的 hook 协议 JSON 字段名以实际 Claude Code 版本为准;**确认**字段是 `tool.name` / `input.command`(参考 tech-stack §1 Claude Code CLI 2026-04-latest)
- SHA-256 tree 算法要**忽略** `.DS_Store` / `.gitkeep` / 符号链接目标(只算链接本身)
