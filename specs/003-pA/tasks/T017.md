  - `probe_disk_io(prev: dict | None) -> tuple[int, dict]` — 增量字节数 + 最新快照

- `pars/stuck/state_machine.py`:
  - `class StuckStateMachine`:
    - 当前态 + 状态开始时间
    - 方法 `transition(gpu, cpu, disk_delta_mbs, net_delta_mbs, tool_use_event_seen, elapsed_since_start) -> NewState`
    - **规则严格对齐 `architecture.md §8.2` 转移条件表**(不得自行微调阈值或窗口):
      - `idle → training`:`gpu > 20% 持续 30s`(6 个 5s 样本全 > 20%)
      - `idle → downloading`:`net > 1 MB/s 持续 30s`
      - `training → idle`:`gpu < 5% **AND** cpu < 10% **AND** disk_io < 1 MB/s 持续 60s`(12 个样本 AND 条件)
      - `downloading → idle`:`net < 100 KB/s **AND** disk_io < 1 MB/s 持续 60s`

    - **disk_io 显式阈值 1 MB/s 活跃**(§8.2 表),不得省略
    - 每次 transition 记录到 `state.json.stuck_state`(写 T007 state)

    - 跟踪 `stuck_restart_count`(从 state.json 读)
    - `should_restart() -> bool`:`count < 3`

    - `record_human_review_needed()` → **写 `runs/<id>/stuck_lock` 文件** + `state.json.needs_human_review=True`(§8.5 持久化)

## Implementation plan

1. 先写 `probes.py`:每函数独立,纯读取,易测
2. 再写 `state_machine.py`:pure function (probes + history) → new state;**不**引入 I/O,便于单测
3. `monitor.py` 是 I/O shell:调 probes + SM + worker API
4. 状态转换规则用 **explicit if-else 链 + comment 引用 architecture §8 的 mermaid 对应条**;不用 state pattern class(过度设计)
5. 60s 滚动窗口用 `collections.deque(maxlen=12)`(**5s 采样 × 12 样本 = 60s**,严格对齐 `architecture.md §8.1`;**禁止**使用 `maxlen=6` / 10s 采样 / 60s tick 等旧粒度)
6. 单测每条 transition 规则独立断言:构造 fake probe samples → 调 `transition()` → 检查 new state
7. circuit breaker 熔断测试:连续 3 次 `record_restart()` → 第 4 次 `should_restart()==False`

## Known gotchas

- nvidia-smi 在 mac 开发机没有 → probe 返回 None → SM 必须能处理 None(降级 = 忽略该信号,只看 CPU/IO)
- `psutil.Process(pid).children(recursive=True)` 在 pid 已死时抛 NoSuchProcess → probe 捕获 + 返回 0
- **采样周期固定 5s / 窗口固定 12 样本**(唯一真相源 = `architecture.md §8.1`);更短烧 CPU,更长使 30s / 60s 转移条件失准;严禁回退到"60s 轮询"旧方案
- `collections.deque` 非线程安全;但 monitor 是单线程循环,OK
- 测试时**不能真 sleep 15min**;用 mock clock(`time.monotonic` patch)或把阈值做成可配置(测试注入 15s)
