## Goal
实现 O10 失败告警链路 + 风险监控 dashboard, **R2: Phase 1 末段提前**:
1. **O10 失败告警** (架构 §5):
   - `apscheduler` 每天 09:00 跑 `FailureAlertMonitor.check()` → SQL `count(decisions where pre_commit_at >= now()-14d AND status='committed')` (R2: 仅 committed)
   - < 2 时:
     - 写入 `alerts` 表 (T002 schema)
     - 调 Telegram bot push (依赖 T017; T017 未 ship 前用 mock; T017 ship 后真发)
     - Web UI 全局 banner (red, dismissable; 模板 `_alert_banner.html` 注入 base.html)
   - **R2 H1: 提供 engage 链接** "降级 B-lite": 点击调 `T010.ConflictWorker.pause()` + (T015 ship 后) `T015.MonthlyScheduler.pause()`. v0.1 不在 UI 暴露 toggle, 仅在 alert banner 给 link, T022 接管 UI

2. **tab 打开计数** (OP-6 mitigation):
   - middleware 记录 `(tab, opened_at, dismissed)` 到 `tab_open_log` 表 (本 task 加 alembic migration 0006)
   - 月度 review (T015) 调用 `tab_metrics.ratio(matrix, decisions)` 检查偏离 (T015 ship 后)

3. **风险快照 dashboard** (`/settings/dashboard`, risks.md "本月风险快照" 格式 7 行):
   - OP-1 录入耗时 p95 (从 request_log; **R2: 度量 commit 阶段 wall-clock**, 不再是整个 POST /decisions)
   - OP-2 累计工时 (从 weekly_maintenance_log T014; T014 未 ship 时 placeholder)
   - OP-4 周维护 4 周平均 (T014; placeholder if missing)
   - TECH-2 LLM 月成本 (llm_usage)
   - TECH-3 PDF 解析失败率 (parse_failures / advisor_reports)
   - O10 连续低决策周数
   - BUS-1 上次备份距今

4. dry-run 脚本: `scripts/dry_run_alert.py` 临时插入 mock 数据触发告警, 验证 Telegram + banner 双通道 (spec §6 O10 ship 后 dry-run)

5. **R2 H1: pause_pipeline 集成**:
   - `monitor/pause_pipeline.py` 提供 façade: `pause_all_pipelines()` 调 T010.ConflictWorker.pause() + T015.MonthlyScheduler.pause() (后者在 T015 ship 前是 no-op stub)
   - `unpause_all_pipelines()` 反向

      """B-lite 一键降级."""
      conflict_worker.pause()  # T010 hook
      try:
          monthly_scheduler.pause()  # T015 hook (T015 未 ship 时 import 失败 → no-op)

      conflict_worker.resume()
      try:

      except ImportError:
          pass

- middleware 不能影响 commit < 30s (TECH-5: middleware 写 SQL 用 BackgroundTask 异步)
- decisions count 必须 `WHERE status='committed'` (R2: 不算 draft)
