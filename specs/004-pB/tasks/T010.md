## Goal
基于 T007 三占位 StrategyModule (analyze only, R2 B2/B3 修复后) 实现:
1. **`ConflictReportAssembler` 中立 domain service (R2 D22, B2 解决)**:
   - `assemble(signals: list[StrategySignal], env_snapshot: EnvSnapshot) → ConflictReport`
   - 不是 lane, 不读其他 lane 内部状态
   - LLM prompt 严格 instruction: "不要选 winner / 推荐 / 最终结论, 仅描述 X/Y/Z 三方观点 + 共同点 + 分歧点"
   - 输出 `ConflictReport` 含 `signals` (≥ 3, 即使有 confidence=0.0), `divergence_root_cause` (白话, "暂无分歧" 或具体根因), `has_divergence`, **`rendered_order_seed: int = hash(",".join(sorted(s.source_id for s in signals)) + day_str) % 6`** (UI 据此随机化三列顺序)
   - 永远无 `priority` / `winner` / `recommended` 字段 (R10 红线)

2. **`ConflictReportService.generate_for_decision_draft(draft_id)`** (T008 draft 阶段同步调用):
   - 从 registry.all() 拿 3 个 StrategyModule
   - `signals = await asyncio.gather(*[m.analyze(...) for m in modules])`
   - `report = await ConflictReportAssembler.assemble(signals, env_snapshot)`
   - 写入 `conflict_reports` 表 + 返回 report_id (供 T008 写入 decision_drafts.conflict_report_ref)
   - **新签名**: 服务于 draft 同步流, 不再走 pending_jobs 异步 (R2 B1 配合)

3. **`ConflictWorker`** (R2 修订: 仅服务于离线场景 + cache warmer):
   - 拉 cache_warmer_jobs (而非 conflict_report_jobs, 后者已废弃因 draft 同步)
   - 暴露 **`pause()` / `resume()` / `is_paused()` hook (R2 H1)**
   - paused 时 cache_warmer 不消费 job, 但 draft 同步路径不受影响 (因为不走 worker)

4. **`ConflictCacheWarmer` 服务 (R3 修订, B-R2-3 配套)**:
   - AdvisorParser 完成后 enqueue: 当前 30-50 关注股 × (3 lane analyze + 1 assemble) 预生成
   - **不预热 Rebuttal** (R3 修订: Rebuttal 输入依赖 draft.intended_action + draft_reason, 每次唯一, 不可缓存)
   - 写入 LLM Cache, 服务于 D21 ConflictReport 命中常态 (≥ 95% 目标)
   - 单独 worker 队列 (低优先级), 不阻塞主路径

5. **Web UI `/conflicts` + `/conflicts/{report_id}`**:
   - **三列等宽** (CSS grid 1fr 1fr 1fr)
   - **顺序按 ConflictReport.rendered_order_seed 排列** (R2 D22, R10 红线工程化)
   - **不留"总结位"** — 不固定 advisor 第 1 / agent_synthesis 第 3
   - `divergence_root_cause` 渲染白话根因
   - 默认空态: `has_divergence=False` 或 `signals` 全部 confidence=0.0 时, 显示 "暂无分歧" 而非空屏
   - R3 不变量: 每条 signal 的 `rationale_plain` 必显式渲染, 空字符串 → 渲染失败异常 (而非静默)

## Known gotchas
- assembler LLM prompt 必须明确"不要选 winner / 推荐 / 最终结论"; PR 任何加 priority field 必拒
- 三列顺序按 rendered_order_seed 是 R10 心理层面 honor (Codex R1 H_human_3): 即使设计上没字段, UI 固定 advisor 第 1 / agent_synthesis 第 3 也是默认优先级
- empty_state 不是错误状态, 文案要平静
- assembler 失败应 raise 让 T008 draft endpoint 知道 (而非写伪报告); **R3 B-R2-2: T008 catch 后必须 raise 503, 不允许 placeholder fallback**
- **R2**: draft 阶段同步流, T010 必须先于 T008 ship (T010 暴露 ConflictReportService.generate_for_decision_draft 给 T008 调)
- **R2 H1**: pause hook 必须在 T010 outputs 显式存在, T020 和 T022 都依赖它. 即使 v0.1 UI 不暴露 toggle, 单测必须覆盖
- **R3 B-R2-3**: ConflictCacheWarmer **不预热 Rebuttal** (Rebuttal 输入依赖 draft 内容, 每次唯一, 不可缓存); Rebuttal 走 T013 fast-path
- **R3 不变量 #13**: ConflictReport.signals 永远来自真实 LLM, assembler 失败时直接 raise, 不返回 placeholder no-divergence stub
