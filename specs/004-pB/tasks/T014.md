## Goal
周度 review (PRD §S4 review & learning):
- `apscheduler` cron 周日 21:00 (本地时区) 触发 `WeeklyReviewService.generate()`
- 内容: 上周决策档案统计 (按 action 分类 buy/sell/**hold/wait** 同等 D2/R11) +
  录入次数 + 录入平均耗时 + LLM 成本 + push 次数 + parse_failure 数 + 维护工时填报
  入口 (Q9 不直接覆盖, 但 review 是 weekly 维护工时 sign-off 的入口, OP-4 mitigation)
- 渲染 Web `/reviews/weekly` (静态周报) + 内置"本周维护工时" form (POST 写入 SQLite
  `weekly_maintenance_log` 表; 此表在本 task 通过新 alembic migration 添加, 触及
  alembic/versions/ 仅本 task 的新 migration 文件 002_*.py)
- "不动" 占比可视化 (D2 + R11): 周报必须显示 hold + wait 占比与原因摘要
- 用 LLMClient 生成简短摘要 (异步, 不阻塞 cron)

## Inputs
- T002 / T003 决策 + LLM usage + alert + advisor + holdings 数据
- T006 advisor reports (周度可能引用最新一份)
- T010 conflict reports (引用最近的)
- D2 / R11 不动一等输出
- C13 / OP-4 维护工时

## Outputs
- `services/weekly_review_service.py`: `generate(week_id)` → dict for template
- `services/scheduler.py`: apscheduler `AsyncIOScheduler`, 注册 weekly + monthly +
  daily alert; 启动时从 main.py 启动
- `ui/router_weekly_review.py`: GET `/reviews/weekly[?week=YYYY-WW]`, POST
  `/reviews/weekly/maintenance` 写工时
- `templates/reviews/weekly.html`: 5 个 section (决策概览 / 不动专区 / 错误与失败
  / LLM 成本 / 维护工时填报)
- `weekly_review_v1.md` prompt: 一段 ≤ 200 字摘要, 偏 calibration (不动是好事)
- alembic migration `0002_weekly_maintenance.py`: 新表 `weekly_maintenance_log`
  (week_id PK, hours float, signed_at)
- 集成测试: seed 5 个决策 → 跑 generate → 模板含 5 个数字 + hold/wait 比例
- scheduler 测试: mock clock, 验证周日 21:00 触发

## Implementation plan
1. 写新 migration 002, 加 weekly_maintenance_log 表
2. service: 聚合 SQL (count by action / sum llm_usage / count alerts) + LLM 摘要
3. scheduler: AsyncIOScheduler + cron expression `0 21 * * 0`; 主进程启动时加载
4. router 渲染模板, 默认本周; ?week= 参数支持回看
5. 维护工时 form: HTMX POST 立刻入库 + 立刻显示 4 周滚动平均 (与 OP-4 阈值 3h
   对比, > 3h 红色)
6. prompt: 强调"不动 / 等待"的 calibration 价值, 反诱导高频 (R6)
7. 测试覆盖 SQL 聚合 + cron 触发 + 模板渲染

## Known gotchas
- apscheduler 在 asyncio 中用 AsyncIOScheduler (不要 BackgroundScheduler)
- **R2 M3**: 时区用 zoneinfo, 默认 `Asia/Taipei` (与 worktree 环境对齐); 实际机制 fallback 到 `tzlocal()` 自动检测; 配置 .env `TZ=` 可覆盖
- LLM 摘要不阻塞 cron, fallback 到无摘要纯统计
