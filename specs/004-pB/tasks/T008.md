## Goal
本 task 是 PRD 硬门槛 C11/O5 (commit 阶段 < 30 秒) 的工程实现 — **R2 修订: 双阶段录入流**.

### 路由设计
- `GET /decisions/new`: 一屏 draft 表单 (3 字段: ticker / intended_action / draft_reason), 默认填上次值 (D13)
- `POST /decisions/draft`:
  1. INSERT decision_drafts (status='draft', refs=NULL)
  2. **同步并发**调用:
     - `ConflictReportAssembler.assemble(signals=[lane.analyze(...)] for 3 lanes, env_snapshot)` (T010 提供, asyncio.gather)
     - `DevilAdvocateService.generate(draft, env_snapshot)` (T013 提供)
  3. UPDATE decision_drafts SET conflict_report_ref=..., devils_rebuttal_ref=...
  4. 302 → `/decisions/draft/{draft_id}/preview`
  - **超时**: 10s 上限 (D21), spinner 显示 "正在生成三路冲突报告 (cache miss)"; 超时后 fallback 到 placeholder no-divergence ConflictReport, 但 UI 显式标 "首次 ticker, 报告生成中..."
- `GET /decisions/draft/{draft_id}/preview`: 渲染三列等宽 + 顺序 hash 随机化 (T010 提供 ConflictReport.rendered_order_seed) + 反方一句话 + commit 表单 (final_action / final_reason / **强制 yes/no would_have_acted_without_agent**)
- `POST /decisions/{draft_id}/commit`:
  1. 校验 draft 存在且 status='draft'
  2. 校验 final_reason ≤ 80 char + would_have_acted_without_agent ∈ {true, false} (M1: 不接受 NULL)
  3. BEGIN; INSERT decisions (status='committed', 复用 draft 的 refs); UPDATE decision_drafts SET status='committed', committed_at=now(); COMMIT
  4. 302 → `/decisions/{trade_id}`
  5. **绝对禁止 LLM 调用** (架构不变量 §9.1, R2 修订: commit 路径无 LLM)

### UX 细节 (R2)
- ticker 字段: HTMX autocomplete 关注股 30-50 (本地 cache, 不查 LLM)
- intended_action: 4 个 radio (BUY/SELL/HOLD/WAIT, label 显示 "1/2/3/4" 数字键盘快捷键)
- draft_reason: textarea max 80 char, 实时倒数显示
- draft submit: Enter 触发
- preview 页:
  - 三列等宽 (CSS grid 1fr 1fr 1fr)
  - 顺序按 ConflictReport.rendered_order_seed (T010 输出) 排列, 不固定 advisor 第 1 / agent_synthesis 第 3
  - 一句反方在三列下方
  - 下面是 commit form
- commit form 字段:
  - final_action (默认 = intended_action, human 可改, 数字键 1/2/3/4)
  - final_reason (默认 = draft_reason, human 可改)
  - **would_have_acted_without_agent: 两个 radio (yes / no), 都未选时 commit 按钮 disabled** (R2 M1)
- commit submit: Enter (commit 阶段无 LLM, wall-clock < 1s)

## Known gotchas
- HTMX form submit + Enter 处理: 用 `hx-trigger="submit, keyup[key=='Enter']"` 慎重避免 IME 上屏被吞 (中文输入)
- ticker autocomplete 数据来自内存 cache (T012 录入后存内存), 不查 SQLite 每次
- post_mortem field T018 才填; new / preview 路径不渲染
- D2 / R11: list 页 action 列必须含 hold + wait, 不能只显示 buy/sell
- **R2**: draft 阶段 LLM 调用 ≤ 10s 上限, 超时 fallback 必须有 (UI 友好 + 可继续 commit); 测试覆盖
- **R2**: commit 阶段绝对无 LLM, 任何后续 PR 加 LLM 调用必拒
- **R2 M1**: would_have_acted_without_agent 在 schema 是 `INTEGER NOT NULL` (无默认), commit endpoint 校验 ∈ {0, 1}, 不接受 NULL
