## Goal
建立项目骨架: `uv` 项目、Python 3.12.7、FastAPI/HTMX/Pico/SQLite/anthropic/telegram-bot 等
**全部 pinned** 依赖、ruff+mypy+pytest 配置、pre-commit 钩子拦截敏感文件 (`.env`, `db.sqlite`,
`inbox/`, `llm_cache/`, Q10 默认假设)、GitHub Actions CI、`scripts/start.sh` 启动脚本骨架,
`src/decision_ledger/main.py` 仅做 entry 占位 (FastAPI app 实例 + 启动事件 hook 占位, 子组件
后续 Phase 接入)。完成后 `uv sync && uv run pytest` 通过冒烟测试。

## Implementation plan
1. `uv init` 初始化项目, 写入 `pyproject.toml` 中 §1 全部依赖 (按 tech-stack.md 钉版本)
2. 配置 ruff (line-length 100, target py312, 启用 `S608`, `S101` 等 security rules)
3. 配置 mypy strict (`disallow_untyped_defs=true`, `warn_unused_ignores=true`)
4. 配置 pytest (`asyncio_mode=auto`, `testpaths=tests`, marker `slow`/`e2e`)
5. 写 `.gitignore` + `.env.example` + `.pre-commit-config.yaml`
6. 写最小 FastAPI app, bind `127.0.0.1`, 添加 `/healthz`
7. 写 config 加载 + fail-fast 校验
8. 写 GitHub Actions workflow
9. `uv run pre-commit install` 验证 hook 工作
10. 写 smoke test 与 config test

## Known gotchas
- macOS 系统 SQLite ≥ 3.46, 但 CI Linux runner 默认 ≤ 3.40 — pyproject 不要求 SQLite 上限
- pre-commit hook 速度敏感 (违反 C13), ruff/mypy 用 cache; 整体 < 10s
- `uv sync --frozen` 必须用于 CI, 本地开发可 `uv sync` 允许更新
