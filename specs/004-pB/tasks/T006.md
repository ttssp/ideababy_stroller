## Goal
实现半自动咨询师 pipeline (D9, R2 RESOLVED): `watchdog` 监听 `~/decision_ledger/inbox/` 新增 PDF →
入 `pending_jobs` 队列 → worker 调 `pdfplumber` 提取文本 → 通过 T005 LLMClient 发结构化
prompt (Anthropic tool use, 输出 schema = `{advisor_id, week_id, recommendations: [
{ticker, direction, confidence, rationale_plain}], raw_summary}`) → 通过
`AdvisorRepository.upsert_weekly` 入库; 失败入 `parse_failures` 表。提供 Web UI 列出
解析失败 PDF + 手工结构化录入界面 (TECH-3 fallback)。同时提供 `scripts/proxyman_fetch.sh`
shell 模板 (Q5 RESOLVED 后锁路径)。架构必须支持多 advisor (C20: `advisor_id` 字段非 hardcode)。

**R2 新增**:
- `ui/templates/advisor/paste.html` + GET/POST `/advisor/paste`: Web UI 提供"贴 PDF / 文本"按钮作为 fallback (不是主路径)
- parse 完成后**自动 enqueue ConflictCacheWarmer** (T010 服务) 给当前 30-50 关注股做 conflict_report 预热, 服务 R2 D21 cache 命中常态

   不可用 → enqueue retry (T005 已处理)

## Known gotchas
- pdfplumber 中文 + 复杂排版可能产出乱序文本 — 接受, 让 LLM 处理
- watched folder 路径来自 config (`DECISION_LEDGER_HOME`), 不 hardcode
- prompt injection 测试要真跑一次 LLM mock + schema 验证, 不能仅信"prompt 写了忽略"
- 大 PDF (> 50 页) 切片, 单次 LLM 输入 token < 100k
