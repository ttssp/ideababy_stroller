## Goal
所有 LLM 调用必须经过此抽象层 (架构不变量 §9.8)。封装 anthropic SDK, 提供:
- async 接口 (D19), 不阻塞 event loop, 内部用 `asyncio.to_thread` 隔离 sync block
- 文件缓存按 `sha256(advisor_week_id + ticker + prompt_template_version + model)`
  (D11), 命中跳过 API
- 用量记账: 每次调用写入 `llm_usage` 表 (含 prompt_tokens / output_tokens / cost_usd
  / cache_hit / model / prompt_template_version, T002 schema)
- retry: 5 次指数退避 (TECH-7); 主模型 Sonnet 4.6 → fallback Haiku 4.5
- 月成本超 $40 自动降级 Sonnet → Haiku (TECH-2 mitigation)
- 输出 redact: 日志中 API key `sk-ant-***` (SEC-3)
- structured output 支持 (Anthropic tool use, SEC-4 mitigation: 不接受 free-form, 防
  prompt injection)
- prompt 版本号管理: prompt 模板放 `docs/prompts/<name>_v<N>.md`, 调用时声明
  `prompt_template_version` 字段

  从 `docs/prompts/` 读取
- `docs/prompts/README.md`: prompt 版本号约定 + 列出已知模板名 (advisor_parser /

- 集成测: 用 respx mock httpx 给一次 5xx + 一次 200 序列, 验证 retry 成功

## Implementation plan
1. `client.py` 实现 `call`, 顺序: 1) 构造 cache_key 2) cache.get 3) miss → anthropic
   SDK + tool use 4) 校验 schema (pydantic) 5) usage.record 6) cache.put
6. `usage.py` 调用 `LLMUsageRepository.insert` + `monthly_total_cost`
7. `retry.py` 包装 1 次 cache miss 调用, 5 次指数退避; 5 次仍失败抛
   `LLMUnavailableError`, 上层决定 fallback 或 enqueue parse_failure
8. `cache.py` 写文件原子化 (写 tmp 再 rename)
9. log 配置 SecretStr redact
10. 单测覆盖 9.1 不变量 (录入路径无 LLM, 此层提供 async 接口供 worker 调)
11. 文档: `docs/prompts/README.md` 写版本号约定 (语义 = `<name>_v<N>.md`,
    每次变 prompt 即 bump N)

## Known gotchas
- anthropic SDK 同步调用本质 sync, 必须 `asyncio.to_thread` 隔离 (TECH-5)
- cache_key 必须含 `prompt_template_version`, 否则 prompt 改但 cache 不更新会污染
- structured output 用 Anthropic tool use, 不用 prompt 强制 JSON (TECH-4 / SEC-4)
- 月成本统计要在写 cache 之前调用 (cache hit 也要记 cache_hit=true 但 cost=0)
