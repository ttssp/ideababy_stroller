## Goal
Telegram bot (`python-telegram-bot` 21.9, long-polling, D15 仅承担 push + 简单查询):
- 周日 21:00 push 周度 review 摘要 (T014 service 提供数据); 触发器 = T014 scheduler
- event 触发 push: human 在 watchlist 标了 ticker 财报日 (Q9 + non-goals A1: 被动
  配置, 不主动整合 event 数据库 v0.1) → **按 ticker.market 决定 session window (R2 D24)**:
  - US: 09:30-16:00 ET
  - HK: 09:30-16:00 HKT
  - CN: 09:30-15:00 CST
  - 默认 US 若 ticker.market 未填
- 简单只读查询: `/start`, `/weekly`, `/ticker TSM`, `/help`
- **rate limiter**: 7 天滚动窗口 ≤ 1 周报 + 5 event (架构 §3.5 + R4 红线)
- 三路冲突报告**叙事版** (D3 / D15): 三句话, 每路一句, 含分歧根因; 比 Web 三列
  简短; **顺序按 ConflictReport.rendered_order_seed 随机 (R2 D22 一致)**
- 失败告警 (T020) 通过此 bot 推送 (本 task 仅提供 `bot.send_alert()` 接口)
- 单白名单 chat_id 验证 (compliance §3.2 + LEG-2): `.env` 中 `TELEGRAM_CHAT_ID`,
  非该 id 的消息直接拒绝

  ```python
  from zoneinfo import ZoneInfo

  MARKET_SESSIONS = {
      "US": (ZoneInfo("America/New_York"), "09:30", "16:00"),
      "HK": (ZoneInfo("Asia/Hong_Kong"), "09:30", "16:00"),
      "CN": (ZoneInfo("Asia/Shanghai"), "09:30", "15:00"),
  }

  def is_in_session(ticker_market: str, now: datetime | None = None) -> bool:
      market = ticker_market or "US"  # 默认 US fallback
      tz, start, end = MARKET_SESSIONS[market]
      ...
  ```

## Known gotchas
- python-telegram-bot v21 的 application context 与 v20 不同; 严格按 v21 文档
- long-polling 在网络断开后自动重连 ≤ 1 min (SLA §1.1)
- chat_id 白名单是 compliance 硬要求 (§3.2 + LEG-2), 任何 PR 不可去掉
- event push 在 v0.1 仅被动 trigger, 不接 CME / earnings 数据库
- **R2 D24**: ticker.market 字段必须在 watchlist 录入时填好 (T012 已支持 enum); 默认 US 仅作 fallback, 期望主路径填好
- **R2 D24**: per-market session window 用 `zoneinfo` (PEP 615), 不要用 `pytz` (deprecated)
- **R2 D22**: conflict_narrative 三句话顺序按 rendered_order_seed, 任何 PR 想固定顺序 → 违反 R10 必拒
