## Goal
封装 SQLite 访问层: 用 `aiosqlite` 异步驱动 + Repository pattern, 全部 SQL 用 `?`
parametrized placeholder (SEC-5 mitigation), 提供单一 writer pattern (架构 §9.6 不变量,
TECH-1 mitigation: 一个 `asyncio.Lock` 守护写入)。每个 domain 模型对应一个 Repository,
暴露 `insert / get / list / update / delete` 与必需的领域查询。所有写入限定短事务
(< 100ms, TECH-1)。LLMUsage / Alert / Note / Job repository 提供 audit trail 写入接口。

## Inputs
- T002 输出的 domain 模型与 alembic schema
- `architecture.md` §9 不变量 (单一 writer / 短事务 / 不直接读他模块状态)
- `architecture.md` §3.2 表设计与索引

## Outputs
- `repository/base.py`: `BaseRepository` 抽象, `AsyncConnectionPool` (单连接 + WAL),
  `WriterLock` (`asyncio.Lock`)
- 每个具体 repo 实现 CRUD + 领域查询:
  - `DecisionRepository`: `insert`, `update_post_mortem`, `update_conflict_ref`,
    `update_rebuttal_ref`, `count_since(days)`, `list_by_action`, `list_by_ticker`,
    `count_would_have_acted_without_agent`
  - `AdvisorRepository`: `upsert_weekly`, `latest_for_week`, `list_for_ticker`,
    `record_parse_failure`
  - `PortfolioRepository`: `replace_watchlist`, `get_watchlist`, `upsert_holding_snapshot`,
    `latest_snapshot`
  - `ConflictRepository`: `insert`, `get`, `list_recent`
  - `RebuttalRepository`: `insert`, `get_for_decision`
  - `NoteRepository`: `insert`, `find_by_content_hash` (去重), `search_fulltext`
  - `AlertRepository`: `insert`, `latest_active`, `dismiss`
  - `LLMUsageRepository`: `insert`, `monthly_total_cost`, `cache_hit_rate`
  - `JobQueueRepository`: `enqueue`, `claim_next`, `mark_done`, `mark_failed`
- 集成测试: 每个 repo 至少 1 条 happy + 1 条 negative (含并发写测试: 10 个 task 同时
  insert decision 不丢条, WAL + asyncio.Lock 保护)

## Implementation plan
1. `base.py` 提供 `connection()` async context manager, 启动时执行 WAL pragma
2. 每个 repo 类单职责, 仅依赖 `connection`
3. 全部 SQL 用 named/positional `?` placeholder, 不字符串拼接
4. 查询返回 pydantic 模型 (从 row dict 构造)
5. 写入路径加 `asyncio.Lock`, 读路径不加锁
6. 集成测试用 tmpdir + alembic upgrade 每个 testcase 重建库
7. 并发测试: `asyncio.gather` 10 个 insert, assert count==10

## Known gotchas
- aiosqlite 的 row_factory 必须显式设 `aiosqlite.Row`
- WAL 模式下 reader 不阻塞 writer, 但仍需 asyncio.Lock 保护多 task 写
- post_mortem_json 字段是 JSON blob, 用 `json.dumps/loads` 在 repo 边界翻译
- index 在 T002 已定义, repo 仅用不创建
