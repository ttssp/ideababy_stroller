## Goal
实现 O5 双条件验证 (R2 修订口径, spec.md §6 O5):
- (a) **自动化 E2E timing test** (Playwright Python binding): 模拟键盘输入完整双阶段流程
  - 阶段 1 (draft 创建): 测 < 10s (D21 上限, cache miss 场景)
  - **阶段 2 (commit) wall-clock < 30s** (C11 硬门槛, R2)
  - 总时长 (cache 命中常态): < 30s 全程
- (b) **手动压测脚本** `scripts/manual_press_test.py`: 引导 human 连续 5 次手工录入, 程序记录每次 commit 阶段 wall-clock + draft 阶段 LLM 等待, 全部 commit < 30s 才写入 `release_log.jsonl`; 任何一次 ≥ 30s 标 fail

测试在 ship 前 + 每次 release 必跑. failing → release block (PRD R1 硬门槛, OP-1 触发降级 B-lite 候选).

## Known gotchas
- Playwright headless 模式比真浏览器快, 阈值留余量 (CI 设 < 25s 比 30s 安全, 真用仍 < 30)
- 中文输入法对录入耗时影响显著 — manual 必须包含中文 reason 真实键入
- CI runner 性能波动, 阈值用 P95 不是单次
- **R2**: draft 阶段 LLM 等待**不算入 30s** (是 D21 单独度量), 不能与 commit 阶段混淆; 计时点严格 = preview 200 OK 起算
- **R2**: 测试需覆盖 cache 命中 & cache miss 两种 draft 场景, miss 场景 < 10s 不破 D21
