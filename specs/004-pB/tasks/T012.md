## Goal
关注股 (30-50) + 持仓快照录入 UI:
- `GET/POST /settings/watchlist`: 粘贴 CSV (`ticker,sector,note`) 或表单单行添加; 重置
  全列表
- `GET/POST /settings/holdings`: 粘贴 JSON (`[{"ticker": "TSM", "qty": 100,
  "cost_basis": 92.0}]`) 或表单; 写入 `holdings_snapshot` (T002 schema, 含 snapshot_at)
- 不接券商 API (架构 §1 边界)
- 默认无错; ticker 字段去重 + uppercase
- 关注股不限市场 (Q3 跨市场为非目标 v0.1, 但 ticker 字段不限市场)

## Inputs
- T002 `Watchlist`, `Holding`, `HoldingsSnapshot` 模型
- T003 `PortfolioRepository.replace_watchlist`, `upsert_holding_snapshot`
- T004 settings tab 占位
- Q9 onboarding 步骤 2-3 (粘贴 CSV / JSON, 5+3 分钟)

## Outputs
- `services/portfolio_service.py`: `replace_watchlist(rows: list[dict])`,
  `upsert_snapshot(holdings: list[dict])`, 校验 ticker 格式 / 数量上限 50
- `ui/router_settings.py`: 4 个路由 (watchlist GET/POST + holdings GET/POST)
- 模板: 表单 + textarea (CSV/JSON 粘贴) + 表格预览
- 单测: 重复 ticker 去重; 超 50 个 ticker 报错; 非法 JSON 报错
- 集成测: 粘贴 sample CSV 后 watchlist 表 30 行; 粘贴 JSON 后 holdings 1 条快照

## Implementation plan
1. service: parse CSV (Python stdlib), parse JSON, 校验 + dedupe + uppercase
2. router: 表单 + textarea, HTMX 预览 (paste → preview table)
3. 模板用 Pico CSS classless table
4. 单测覆盖 dedupe + 上限 + 非法格式
5. 集成测试用 TestClient + tmp DB

## Known gotchas
- CSV 粘贴要去 BOM 头 (excel 导出 BOM)
- JSON 数字精度 (cost_basis float, 容差到 2 位小数)
- 50 上限是软约束 (Q3 默认), 超过需 human 确认 (跳警告)

## Out of scope for this task
- 真券商 API (永远 R1 红线)
- 多市场 cross-signal 计算 (v1.0+)
- 市场行情自动同步 (永远不接 v0.1)
