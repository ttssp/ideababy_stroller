## Goal
搭建 server-rendered Web UI 外壳: FastAPI app 注册 8 个 tab 路由 (D14: 决策档案 / 冲突
报告 / 错位矩阵 / 笔记 wiki / 周 review / 月 review / 学习检查 / 设置), 每个 tab 暂时
渲染空态占位页 (后续 task 替换 inner)。base.html 含 Pico CSS + HTMX 引入, nav 默认首页
= **决策档案** (OP-6 mitigation), 错位矩阵 tab 故意排第三。footer 永远显示自用免责声明
(compliance.md §4.1)。FastAPI 强制 bind `127.0.0.1` (架构不变量 §9.7) + 反 0.0.0.0 单测。

## Inputs
- D14 tab 列表
- compliance.md §4.1 完整免责文案
- architecture.md §9.7 localhost binding 不变量
- tech-stack.md §1 (Jinja2 3.1.4, HTMX 2.0.4, Pico CSS 2.0.6)

  挂载 `/static`, 注册 shell router; bind 检查写在 main.py

  tab 链接, 任何未实现的 tab 暂时返回 404 → 模板将通过后续 task 接管时再生效
  (FastAPI 启动时各 router 已 include 进 app)
- `templates/base.html`: 含 `<head>`, Pico CSS, HTMX, app.css; `{% block content %}`;
  `_nav.html` + `_footer.html` include; **alert banner hook**: 顶部预留
  `<div id="alert-banner" hx-get="/_partials/alert-banner" hx-trigger="load, every 60s"></div>`,
  此 endpoint 在 T004 阶段返回空 (无 active alert), T020 接管后返回真 banner partial

- `templates/_footer.html`: compliance.md §4.1 4 行文案 + LLM 输出注释 + version
- 各 tab `_shell_placeholder.html`: 空态占位 ("暂无数据 / 此功能待启用 (T0XX)"),
  `router_shell` 在后续 task 真实路由 (T008/T010/T011/T012/T014/T015/T016/T019/T020)
  挂载前用其作 fallback; 后续 task 各自创建本 tab 真模板, 并接管路由
- 静态资源: Pico CSS 2.0.6 + HTMX 2.0.4 (downloaded, vendored, 不走 CDN, localhost-only)
- 集成测试: GET `/` 200 + 含 footer 文案 + nav 8 项顺序正确; 未实现 tab GET (e.g.

- E2E: `test_first_screen_latency.py` Playwright 启动 server, 测首屏 < 2s (SLA §1.2)

## Implementation plan
1. 写 `create_app` 工厂, 挂 `StaticFiles` + Jinja2 environment
2. 写 router_shell: 仅 GET `/` 渲染 index.html (项目说明 + 跳决策档案); 不接管其它
   tab 路由
3. 写 base.html, 含 Pico CSS + HTMX, 引入 _nav + _footer
4. 写 _nav.html: nav 列表硬编码顺序 (决策档案首页), 当前 active tab 高亮
5. 写 _footer.html: 含 §4.1 完整 4 行 + "LLM 输出可能错误" + 版本号
6. **不修改** T001 的 `main.py` — 而是让 `ui/app.py` 模块在 import 时通过 T001 的
   plugin registry (`register_router`, `register_startup`) 自动 wire; main.py 在
   `import decision_ledger.ui` 触发时即接管真 UI 路由; 强制 `127.0.0.1` 由 T001 已
   实现 (T004 仅添加反 0.0.0.0 单测)
7. 集成测试: TestClient 跑每个 tab 路由
8. E2E latency 测试: 启动 uvicorn, Playwright `goto("http://127.0.0.1:8000/")`,
   断言 `performance.timing.domContentLoadedEventEnd - navigationStart < 2000`
