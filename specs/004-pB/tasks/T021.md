## Goal (R3 简化版, 4h)
首次启动引导 + 文档 (O6 验证 ≤ 15 分钟). Q9 默认 7 步分解 (15 分钟):
1. 启动 (`./scripts/start.sh`) — 1 分钟
2. 录入 30-50 关注股 — 5 分钟 (复用 T012 路由)
3. 录入持仓快照 — 3 分钟 (复用 T012)
4. 拖第一份咨询师 PDF 到 inbox (T006), 等待解析 — 1 分钟
5. 完成首次决策档案录入 (任意 ticker, action=hold/wait, 测试理由) — 30 秒~1 分钟 (复用 T008 双阶段)
6. Telegram bot 注册 + chat_id 绑定 — 3 分钟 (T017 已配置)
7. **看完空态文字 (R3 简化: 单一静态文字 "首次季度学习提醒在 90 天后自动出现", 无 wiki / 学习引用)** — 1.5 分钟

**R3 简化实现**:
- `OnboardingService` 状态机: 7 步逻辑保留, 但**模板合并为 3 个**: `welcome.html` (步 1) / `steps_combined.html` (步 2-7 单页 multi-section) / `done.html`
- 首次启动 redirect 到 `/onboarding`, 否则跳过
- 内置计时: 每步 enter/leave 写 timestamp, 总耗时 ≤ 15 分钟才标 `o6_pass=true`
- E2E 测试 `test_onboarding.py`: Playwright 模拟 7 步 (在 steps_combined.html 内 section 跳转), 计时 < 900s
- `docs/runbook.md`: human 自用使用手册极简版 (架构 §8 + compliance §4.2 法律边界 + 备份命令)
- `README.md`: 项目说明极简版 (≤ 100 行: 安装 + 启动 + 链接 runbook)

**R3 砍掉**:
- 7 个独立 step 模板 (合并为 1 个 multi-section)
- in-app tooltip (改为 inline 文字)
- step 7 wiki 空态引用 (改为静态文字)

- E2E 测试 PDF 解析慢 (LLM 调用), step 4 用 fixture mock LLM 跳过真请求

- 15 分钟阈值 CI 难稳定, e2e 标 `@pytest.mark.slow` + 阈值留 buffer (≤ 600s 自动模式)
- step 5 用 T008 双阶段 (R2): draft → preview → commit, 测试 mock cache 命中场景 (avoid LLM cold start in onboarding)
- **R2 H3**: step 7 不能引入 T019 import (会 break onboarding 在 T019 未 ship 时); 仅静态文案 + 引用 T020 alert banner mechanism
