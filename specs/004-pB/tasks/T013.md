## Goal
实现 `DevilAdvocateService` 独立服务 (R2 D23, B2/B1 解决; R3 B-R2-3 fast-path):
- `DevilAdvocateService.generate(decision_draft: DecisionDraft, env_snapshot: EnvSnapshot) → Rebuttal`
- LLM prompt: 反驳 `decision_draft.intended_action`, 一句话 ≤ 80 字, 必含 "考虑反方" 语气, 不诱导高频 (R6)
- **R2 修订: 不再是 StrategyModule lane**, 不实现任何 StrategyModule.* 方法
- **R2 修订: draft 阶段同步调用** (由 T008 在 asyncio.gather 中并发调本 service), 生成完毕才返回 draft preview
- **R3 修订 (B-R2-3): fast-path LLM 配置** — 因 Rebuttal 输入唯一不可缓存, 必须用最快配置达 ≤ 3s 上限:
  - **model**: Sonnet 4.6 (与主路径相同, 不降到 Haiku 避免质量风险)
  - **max_tokens ≤ 100** (强制短输出 ≤ 80 字)
  - **极简 prompt** (≤ 200 token 上下文, 删除 few-shot examples)
  - **temperature 0.3** (减少采样开销 + 输出稳定)
  - **wall-clock 目标 ≤ 3s 95th percentile**
- **R3 修订: 不查 cache** — 因每次 input 唯一, cache 必 miss, 浪费 cache lookup 时间
- 不变量 §9.4: `Rebuttal.rebuttal_text` 永远非空 (R3 / OP-7)
- 作用: 强制 human 在 commit 前听一句反方意见 (Glossary), OP-3 mitigation (forcing function 之一)

## Known gotchas
- LLM 可能返回多句, prompt 强制一句; 模板渲染 truncate 80 字
- 不变量 §9.4: `Rebuttal.rebuttal_text` 非空 validator T002 已加, 此处再单测一次
- Devil's advocate 是 glossary 定义的占位, 不是 production 反驳; 后续 task 不要拓展为复杂多角度
- **R2 修订**: 本 service 必须先于 T008 ship, T008 在 draft 阶段同步调用
- **R2 修订**: 调用方是 T008 (DecisionRecorder.create_draft 内 asyncio.gather), 不是 worker. 任何 PR 把 generate 改回 worker 异步 → 违反 R2 B1, 必拒
- **R3 修订 (B-R2-3)**: Rebuttal **不可缓存** (输入唯一 every draft); fast-path 配置必须固定. 任何 PR 想加 cache_lookup=True 必拒 (浪费时间). 任何 PR 想 max_tokens > 100 必拒 (超 ≤ 80 字契约 + 拖慢).
- **R3 修订**: timeout 3s 上限 — 单点超 3s → service.generate 抛 TimeoutError, 由 T008 整体 5s timeout 兜底, 最终触发 503
