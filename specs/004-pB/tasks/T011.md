## Goal
实现错位矩阵可视化 (D16: HTML table + CSS 着色, 不引入 plotly/matplotlib, 5 秒看完
原则). Q8 默认假设:
- 行 = 30-50 关注股 (ticker)
- 列 = [咨询师方向, 占位模型方向, agent 综合方向, 当前持仓 %]
- 着色: 错位 (咨询师强推 vs 持仓 < 板块平均) = 黄色; 反向错位 (咨询师谨慎 vs 持仓
  > 板块平均) = 红色; 无错位 = 默认色
- 排序: 默认按错位强度降序

OP-6 mitigation: 错位矩阵 tab 在 nav 排第三 (T004 已配); UI 不上首页, 不诱导成"看板".

## Inputs
- T002 schema (ticker / holdings / advisor_reports / signals)
- T003 `PortfolioRepository`, `AdvisorRepository`, signals 缓存
- T007 三 StrategyModule (用 `analyze` 拿三方向)
- Q8 默认维度

## Outputs
- `services/matrix_service.py`: `MatrixService.build()` 返回二维表格 dict 列表;
  错位强度算法 = `abs(advisor_direction_score - holding_position_score)`,
  方向 score: long=+1 / short=-1 / wait/neutral/no_view=0
- `ui/router_matrix.py`: GET `/matrix` 渲染 table
- `templates/matrix/_table.html`: 严格 HTML table; 每个 cell `class="cell-{level}"`
  控制颜色 (yellow / red / neutral)
- `matrix.css`: 三色; 用 Pico CSS 兼容
- 单测: `test_matrix_service::test_misalignment_calc` (advisor=long + holding=0 →
  yellow); `test_reverse_misalignment` (advisor=short + holding=heavy → red);
  `test_aligned_neutral_color`
- 集成测: `/matrix` 返回 HTML 含 `<table>`, ≥ 30 行, 颜色 class 正确

## Implementation plan
1. matrix_service: 拉 watchlist + holdings_snapshot + 最近 advisor_reports +
   running 三 signal cache (复用 T010 写入的 conflict_reports 中 signals)
2. 计算错位强度, 排序
3. 模板: `_table.html` thead 4 列 + tbody 30-50 行; 单 cell 包 ticker direction
4. CSS 三色背景 + 字体粗细, accessible (色盲友好用 emoji 备份)
5. 单测每种错位类型一条 (≥ 5 case)
6. 集成测试 mock seed data, 断言 cell 颜色
