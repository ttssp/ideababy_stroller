## 3. Refined candidates

R1 的 4 个切点收敛为 **3 个 peer**(按"先验证可靠采集 → 再加形态 → 再加结构化"的真实风险顺序排):

### Refined A · "单形态打通 · 可靠优先"(原 Opus-A 瘦身 + Readwise 教训)
- **v0.1**:只采**图文分析**一种形态,但把"**不漏 + 自动不失效**"做到位(失败告警 + 网站变化能感知)。轻结构化(时间+标的标签+原文),够检索够粗喂 004。预警/回放明确 v0.2+。
- **赌注**:008 真瓶颈是"单源可靠采集"(搜索③印证)。先用最少形态把这件事打穿,其余是增量。
- **时间**:~4-6 周(窄),余量留给"可靠"这个真难点。**Confidence: M+**。
- **最适合**:如果你认同"先证明能可靠采到一种,再谈扩展"。

### Refined B · "图文+预警 · 守住有时效内容"(原两版 C 合并)
- **v0.1**:图文 + 盘中预警(漏了就没价值的两类),轻结构化台账(GPT 的 provenance 定义),跳回放。预警要及时通知。
- **赌注**:有时效的内容(预警)最不能漏,值得 v0.1 就纳入;回放晚收无妨。
- **风险**:预警与图文是否同源未验证(§5)。若不同源,B 的成本接近全形态。
- **时间**:~7-9 周。**Confidence: M**(GPT 估 6-8 周,我偏保守因预警同源未验)。
- **最适合**:如果"漏掉盘中预警"是你最痛的点。

### Refined C · "全形态原始档案"(两版 B,基本不变)
- **v0.1**:三形态全收,只原始留存 + 时间线,结构化/检索后置。
- **赌注**:最怕漏历史,先全装进来。
- **风险**:回放(形态最重 + 微信源头不确定)可能撑爆 2-3 月,拖累更高价值的图文/预警。
- **时间**:~9-12 周(吃满或偏紧)。**Confidence: L-M**。
- **最适合**:如果"任何形态被漏"对你不可接受,且愿担更长周期。

## 4. The single biggest tradeoff human must decide

**轴:v0.1 是"先把一种形态的可靠采集打穿"(窄而稳)还是"先把所有有价值内容装进来"(广而不确定)?**

这个轴卡在你两个并列最高优先级的内部张力上:
- "**不漏**"拉向 **C**(全装进来才不漏)
- "**省心/不失效**"拉向 **A**(先把一种采稳,才谈得上自动不失效;贪多每种都不稳=最不省心)

**搜索给的裁决倾向 A**:008 源头(微信小程序)没有成熟采集通道,真瓶颈是"单源能否可靠采",不是形态数。专业工具(Readwise)敢多形态首发是因为它们的源头有成熟通道——008 没这条件。

**但有个前置决定这个轴怎么选**:GPT 提的"**先观察顾问 1-2 周发布节奏**"。如果观察发现预警/回放其实很少、图文占 90%,那 A 几乎等于全部,选 A 毫无损失;如果预警频繁且关键,B 的价值才凸显。**建议:进 L4 前先做这个观察 + 采集落点探针,再定 A/B/C。**

## 5. What I'm less sure about now than in R1

- **预警与图文是否同源**:R1 我假设 C(图文+预警)比全形态省;现在不确定——若预警走推送、图文走小程序页面,B 的"省"是假的。**L4 必须实测**。
- **采集落点 vs 合规的交叉**:搜索②让我意识到落点选择不只是可靠性问题,还是**合规问题**——某些"贴源"手段可能正好踩访问控制红线。落点决策现在比 R1 时更重、更敏感。
- **深结构化(原 Opus-A)我现在更不看好**:搜索③④ + GPT 的越界警告让我觉得"抽观点要点"既有技术成本又有越界风险,L3R2 我把它从独立候选降级了。
