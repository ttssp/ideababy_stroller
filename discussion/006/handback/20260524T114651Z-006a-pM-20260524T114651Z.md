## §7 Known gotchas

1. **`set -e` + `$(...)` + python3 `sys.exit()` 时序 bug**:本修反映 bash 已知 nuance — `$(...)` 命令替换内的 python3 退非 0 时,`$?` 在 `set -e` 触发 trap 之前会被 shell 错误捕到 0。临时关 `-e` 在 capture 区间是最小改面 fix(修法 A)。同 pattern 还可能在 lib/handback-validator/check-*.sh 内潜伏(A4 audit 跟进)。

2. **修法 A 用 `set +e / set -e` 而非 `|| true` 或 `2>&1` 转发**:试过 `2>&1`(已存在),python3 错误已合并到 stdout 进了 `VALIDATE_RESULT`,但因 `VALIDATE_RC=0` 后续 echo 跳过。所以问题不是 stderr 丢,是 rc 时序。修法 A 不动 capture 内容、只动 rc 捕获时序,最精准。

3. **macOS write(2) ≤PIPE_BUF 字节原子 不需 flock**:T003 verification 2026-05-24 5 并发实证 jsonl 无撕裂(6/6 行全合法 JSON)· spec FU-T003-stderr-fix.md §"Known gotchas" #3 已记 · 不顺手加 flock 引 macOS brew 依赖。

4. **SSOT drift 标记 tags=[drift]**:本 hand-back 触 §6.3 schema drift tag · IDS 端 `/handback-review 006` 端必须处理 A1(否则 bootstrap-kit 下次 ship 新 PRD 时仍带原 bug)。若 IDS 拒绝反向同步,XenoDev 下次 bootstrap 重落会冲掉本修 — 需 spec 加 known-limit。
