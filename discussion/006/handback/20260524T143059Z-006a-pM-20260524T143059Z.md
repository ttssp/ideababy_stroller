## §7 Known gotchas

1. **`ln src dst` 在 dst 存在时 fail 是 POSIX 保证**(non-overwrite 默认 · 跟 GNU ln 选项 `-n` 同效果但更可移植)· 本 fallback 第二步 ln tmp→TGT 也走这个语义 · 双重 EEXIST 保护。

2. **mock 策略不需 sudo**:cross-device test 用 PATH 注入 stub ln(non-LD_PRELOAD)· 第一次 DRAFT→TGT 退 1 模 EXDEV · 标记文件防多次触发 · 后续 ln(同目录 tmp→TGT)走 real ln · 跨 macOS / Linux 都跑得通。spec §"Known gotchas" #1 提建议 hdiutil RAM disk 但 sudo 不适 dev 机 / CI 路径 · stub 路径更轻量。

3. **stub cp corruption 策略**:real cp 后 `echo "CORRUPTED" >> dst` 让 cp 出来的文件比 DRAFT 大 + sha 不一致 · 实证触发 fragment 的 sha 漂移 hard-fail 路径。

4. **`date +%s$RANDOM`** 生成 tmp 后缀:spec §"Known gotchas" #2 提 BSD date 不支持 `+%N` · 改用 `$(date +%s)$RANDOM` 兜底。本实装直接走这个 · 不试 `+%N`。

5. **rename(2) 不绑**:bash 没原生 rename · 用 cp + sha + ln tmp→TGT 模拟 POSIX rename 原子语义。`ln` 本身原子 · 即使 SIGKILL 也不会出半文件。

6. **PUBLISH_MODE 给 SLA 追踪**:operator 看 stdout 知道走的是 ln 还是 cp+ln · cp+ln 比 ln 慢一截(real cp + sha + 第二次 ln vs 单 ln)· 真生产采集后 spec 加 SLA doc。当前 dev 机 macOS /tmp 跟 /Users 同卷,always PUBLISH_MODE=ln。

7. **gen-handback.sh `--tag` 多 tag bug**:本 hand-back gen 时 `--tag drift,spec-gap-fix` 被当一个 string · YAML 出 `- drift,spec-gap-fix` 单元素 · 已手动改成数组。A3 留 IDS 决议 + lib FU 修。
