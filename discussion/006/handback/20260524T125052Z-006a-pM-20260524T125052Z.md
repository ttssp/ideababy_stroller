## §7 Known gotchas

1. **sed replacement escape 顺序**:`\` 必须先 escape,否则后续 escape `&` / `|` 引入的 `\` 会被再次 escape 成 `\\\\&`。当前实装严格按 `\` → `&` → `|` 顺序。改顺序就破。

2. **`&` 在 sed replacement 的特殊语义**:sed `s|pattern|replacement|` 里 `&` = 整个 match 的内容。例如 `s|{{RATIONALE}}|A & B|g` 实际输出 `A {{RATIONALE}} B`(`&` 展开成匹配的 `{{RATIONALE}}` 字符串)。所以 user input 的 `&` 必须 escape 成 `\&`。

3. **`--scope=working-tree` vs `--scope=branch`**:codex review `--scope=working-tree` 只看 uncommitted 改动 · `--scope=branch --base main` 看 worktree branch vs main 全 diff。round 1 commit 完再跑 round 2 必须用 branch scope 否则看不到改动报"未检测到代码变更" → 误 fail closed。本 ship round 2 真路径学习留 §7 gotcha 防再踩。

4. **template `{{VAR}}` 占位符 vs fixture schema 字面**:template 必须跟 `lib/handback-validator/test-fixtures/valid/<ts>-008a-pA-<ts>.md` 字段顺序 / YAML 缩进(2 空格 ASCII)/ 字段名 完全等价,否则下游 6 约束 fail。本 ship 实证 SHA256 等价(用相同入参 template vs main inline 仅 ts 差)。

5. **post-gen SSOT 断言不可去**:gen-handback.sh line 268+ 重读 $OUT 断言三字段 == HANDOFF 抽取值(防 user input 注入 frontmatter 提前覆盖)。本 ship 改 inline → template **不动这个断言**(它防的是注入,跟 template 实现无关)。
