## §7 Known gotchas

1. **`--scope working-tree` 不展开 untracked 目录**:codex companion 看 working-tree 时,untracked 目录(eg `.claude/worktrees/T005/`)被报告为 "contents excluded",不读子文件内容。原 spec 设计依赖这个就死了。本 ship 学到的教训:**codex review 的 attack target 必须 git tracked**(staged 或 committed)。

2. **macOS BSD 无 `timeout` 命令**:GNU coreutils 的 timeout 在 stock macOS 不存在。本 ship 用 perl alarm 实装跨平台兜底(`perl -e 'alarm 180; exec @ARGV'`)· 退 124 不一定,实际看 perl 自身行为。

3. **`find | head -1` 取 shell glob 序非 semver**:`1.10.0` 在 glob 排在 `1.2.0` 后(字母序),但 semver 上 `1.10.0 > 1.2.0`。改 `sort -V | tail -1` 但 sort -V 是 GNU 扩展。本 ship 用 sort -V(本机实证 macOS 2.3-Apple 真支持)· 但留 A1 IDS 决议加 gsort/python3 fallback。

4. **codex companion 即使 review 失败也渲 header**:`# Codex Review` / `# Codex Adversarial Review` header 在 review 真出 + review 失败 (rc=1) 两种 case 都会渲。所以 link test 不能只验 header 存在,必须严判 rc=0。本 ship round 2 真修了这个 fake-green。

5. **`--scope working-tree` vs `--scope=branch --base main`**:worktree 内刚 commit 后 working tree clean,跑 review --scope working-tree codex 看不到改动报"未检测到代码变更" → 误 fail closed。本 session(T004 + T005 都遇到)用 `--scope=branch --base main` 看 worktree branch vs main 全 diff 是真路径。

6. **link test 跑 1 次消耗 codex CLI quota** · 60-90s · 不适合 CI(本 task spec §"Known gotchas" 已记 "operator 手跑")。本 ship 加 `last-link-test.log` 进 .gitignore 防本地跑产物入仓。
