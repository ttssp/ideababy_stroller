## §7 Known gotchas

1. **wrapper 不开 `set -e`**:开了会让 T001 runner 退非 0 时 wrapper 也死,RC 透传 break。spec §"Known gotchas" 已记;round 1 codex P1 (mktemp 失败) 也是这个根因(刻意不开 -e 但要为 mktemp 失败加显式 fallback)。

2. **`mktemp` 在 sandbox /tmp 不可写时必须有 fallback**:round 1 codex P1 真路径学到 — wrapper 内 STDERR_LOG=$(mktemp ...) 失败时 STDERR_LOG="" → 后续 2>"" ambiguous redirect runner 根本不跑。修法:`STDERR_LOG=$(mktemp ... 2>/dev/null) || STDERR_LOG=""` + 用 if 分支(STDERR_LOG 空时不重定向)· test runner 同 pattern 但 exit 2 precondition fail。

3. **`2>/tmp/file` 在 /tmp 不可写时 redirect 失败 → 命令根本没跑**:round 2 codex P2-2 真路径学到 — writer.sh stderr 改走 fd 2 直出(`$(...) 2>&1` capture + WRITER_RC),失败时 stderr 已经出过,WARN 一行带 EVAL_LOG_DIR hint。

4. **default EVAL_LOG_DIR 应用 `$REPO_ROOT/.eval` 不用 cwd**:round 3 codex P2-1 真路径学到 — writer.sh 内默认 `LOG_DIR=${EVAL_LOG_DIR:-.eval}` 用 cwd 解析,wrapper 从子目录跑时 event 写错位置。修法:wrapper 调 writer 前 `export EVAL_LOG_DIR="${EVAL_LOG_DIR:-$REPO_ROOT/.eval}"` · operator 显式设的仍优先(test 隔离不破)。

5. **chmod 0 在 root/特权 CI 不阻 mkdir**:round 1 codex P3 真路径学到 — test 用 file-path-as-dir(touch file + EVAL_LOG_DIR=file/sub)100% 跨平台触发 writer.sh mkdir -p 失败 + 双断言(RC=0 + stderr 含 WARN)· 比 chmod 0 dir 稳。

6. **JSON details 字段含中文会被 `json.dumps` 默认 ensure_ascii=True 转 unicode escape**:eg 中文 stderr 进 event 会变 `行数` · 仍合法 JSON · `json.loads` 能读 · 不影响 schema 校验 · 可读性稍差。若想保中文可选 `ensure_ascii=False` · 非 ship blocker · 留 follow-up。

7. **gen-handback.sh `--tag drift,feature` 多 tag 写入 frontmatter 是 `- drift,feature`(1 个 enum-invalid string)非 array**:FU-producer-2 hand-back §3 A3 已标。本 hand-back 手动改 tags 节加 drift。建议 T007 ship 触动 gen-handback.sh 多 tag bug 修。
