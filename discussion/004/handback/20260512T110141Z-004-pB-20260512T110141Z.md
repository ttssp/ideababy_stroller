## §1 Rationale

v0.1 product bug · F1b production routes wire 后显形:
`projects/004-pB/src/decision_ledger/ui/templates/reviews/weekly.html` L108
`{% include "...partial.html" %}` 不显式传 maintenance · partial 内 6 处引
`maintenance.*`(current_hours / over_threshold / threshold / rolling_avg)·

partial 内 maintenance.* 解为 review.maintenance.* · `{% endwith %}` 后绑定不
泄漏父作用域。

## §7 Known gotchas

1. **Jinja2 include kwarg 语法限制**:`{% include "x" with context maintenance=... %}` 不支持
   keyword args(Jinja2 设计选择)· 标准做法用 `{% with %}` 块包 include · 未来类似场景注意

2. **partial 命名风格不一致(v0.1 残留)**:`_weekly_maintenance_partial.html` 用顶层
   `maintenance.*` · `weekly.html` L101 用 `review.maintenance.*` 嵌套 · 这是 v0.1 设计
   不一致 · 本 ship 用 `{% with %}` 桥接 · A2 backlog 统一(refactor)

3. **POST 路径不走 {% with %} 块**:router L137-143 POST /reviews/weekly/maintenance 直接
   渲染 partial 传顶层 `{"maintenance": maintenance}` · partial 顶层引用直接命中 · GET 走 with
   桥接 · POST 走顶层变量 · 两路径语义一致(codex 用本地 Jinja2 内存样例 verify 过)

4. **{% endwith %} 不污染父作用域**:weekly.html L101 用 review.maintenance.over_threshold ·
   `{% with %}` 块内绑定退出 endwith 后消失 · 不影响父模板后续代码

5. **KNOWN_500_PATHS 全清空 + 类型保留**:`frozenset[str] = frozenset()` append-only
   pattern · 未来 F1b smoke 发现新 500 bug 直接加进 frozenset · 不破 import 链

6. **件 3.1 阶段 2c 波 1 闭环**:W1(7eb8626)+ W2(09f6cc1)= 两 v0.1 product bug 全
   ship · 用户实际可用 /notes 笔记列表 + /reviews/weekly 周报全页 · 波 2(T011)可起
