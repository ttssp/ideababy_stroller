## 0. 我接收到的新 framing 简述 (以确保两个模型对齐)

Human 读完 stage-L2-explore-004.md 后发现之前的产品化 framing 不对, 做了三个重大校正:

1. **服务对象 = 他自己一个人, 不商业化, 不卖钱**。工具是私用。

这把 idea 从"投资纪律层 (薄层, 不卖 alpha)" 彻底重定位成**"个人投资副驾驶 + 金融教练 + 可进化的私有策略引擎"的三合一**。下面我用全新视角重新 unpack。

## 1. 这个 idea, 在新 framing 下重新展开

### 谁是用户, 这次看得更清楚了

一个非常具体的人: **一个 ML PhD 背景的中年人, 技术 sense 强但金融素养初级, 已经付费订阅了一位哥大背景的华语投顾, 有三市场 (美港 A) 小规模持仓 (30-50 只), 能承受 20% 回撤, 想为全家负责, 但不想花一年时间去 CFA 系统学习**。

之前两轮把他假设成了"认知能力强 + 外部输入多 + 缺执行闭环"的人——这个画像对"投资纪律层"成立, 但对这个人**不成立**。真实的他更接近: **认知能力强 (元能力), 但某个具体领域 (金融) 的 domain knowledge 弱, 且他清楚这一点, 不想假装**。

这是一个在技术产品里很少被精确服务的用户类型: **元能力超强但某领域新手**。他不是金融小白 (他能理解"估值"、"仓位"这类概念, 能看懂财报结构), 但他也不是 practitioner (他无法凭直觉说"这图形是头肩顶还是双底")。他学东西快, 但他要**在实操中学**, 不愿意先读 500 页教材再开始。

### 没有这个副驾驶的日常

晚上 10 点, 刷完咨询师周报, 看到"建议右侧布局半导体"。他想操作但**不知道怎么具体执行**:
- "右侧布局"具体是什么意思? 等突破哪条均线? 还是看成交量确认?
- 30 只关注股里哪几只是半导体? TSM / NVDA / AMD / ASML / 中芯 / SMIC... 这些都叫半导体, 但他们 trade 逻辑一样吗?
- 加仓多少合适? 5%? 10%? 他不知道怎么换算。
- 咨询师说的"右侧"是技术面的还是估值面的? 他分不清。

他**有想法 + 有信息输入 + 有交易能力**, 但**中间那一层 "把观点翻译成具体操作 + 知道为什么" 缺了**。结果要么他凭感觉买了, 然后被动跟随咨询师, 没学到东西; 要么他不操作, 错过机会; 要么他拖到周末查资料, 拖过了右侧点位。

### 有了这个副驾驶的日常

晚上 10 点, 他打开 agent。Agent 已经**主动做完了几件事**(不是等他问):

1. **监控并解析本周咨询师观点**: "本周周报核心 3 条 — (a) 右侧布局半导体, (b) 谨慎港股消费, (c) 美元走弱利好黄金。"
2. **结合你的 30 只关注股**: "你的关注列表里的半导体: TSM / NVDA / AMD / SMIC, 按咨询师逻辑, TSM 和 AMD 当前技术面接近'右侧确认'(20 日均线上方 + 成交量放大), NVDA 已经 overshoot, SMIC 仍在左侧. **优先考虑 TSM 或 AMD**。"
3. **给出具体仓位建议**: "若你采纳, 建议 TSM 从当前 4.2% 加到 6% (单持仓阈值 8%), 对应大约 $X. 这一笔若入, 你的半导体总暴露到 18%, 仍在你自定的 25% 板块上限内。"
4. **白话教你为什么**: 弹出一小段侧栏: "'右侧布局'指股价已脱离底部并得到确认, 通常看两个信号 — 20 日均线上方站稳 + 成交量放大. TSM 本周两个都满足了. '左侧'则相反, 是股价还在底部区域、没确认反转时就入场——更便宜但风险更高. 咨询师本周建议右侧, 因为他认为半导体趋势已确立, 不追求最低点."
5. **基于你的私有策略交叉检验**: "你自建的动量模型对 TSM 今日信号: +0.7 (中等看多); AMD: +0.3 (弱看多). 模型和咨询师在 TSM 上一致, 在 AMD 上你的模型弱于咨询师——可能你模型没纳入咨询师用的某个行业叙事。"
6. **一个简洁的决策界面**: 三选一 — [加仓 TSM 到 6%] / [先 watch, 明天再决] / [不做]. 每选一项, 都记下理由和当前环境快照。

他花 5 分钟决定, 按了"加仓 TSM 到 6%". Agent 记下: "2026-04-24 按咨询师右侧信号 + 私有动量模型 +0.7 共识, 加仓 TSM 从 4.2% → 6%." **他同时学到了**: 20 日均线 = 右侧信号之一, 成交量放大 = 确认。这两条知识进了他的"个人金融笔记", agent 下次讲相关话题时不会再重复解释。

### 30 秒 aha: 第一次打开

他第一次打开, agent 不问他 API, 不让他连券商。先要三件东西: (a) 你 30-50 只关注股, (b) 你订阅咨询师最近 4 期周报的文字/图, (c) 你目前的持仓大致分布 (按板块即可). 5 分钟录入。

然后 agent 给他看一张**他从未看过的视图**:

> 你的持仓 vs 咨询师观点 vs 市场板块: **错位矩阵**。
> 咨询师本月重点看好 AI 基础设施, 你持仓在 AI 基础设施上只有 3% 暴露, 远低于他的强度建议。
> 咨询师谨慎消费, 你持仓有 18% 消费股。
> 上述"错位"里, 有 60% 是你之前没意识到的 (即使你自己读了咨询师每一篇).

他的 aha 不是"AI 好厉害"; 是**"原来我一直在读, 但我并没有真把咨询师的观点落到自己组合里"**。这个 aha 是一个**认知刺痛**, 不是一个兴奋 — 他突然意识到自己此前半年的订阅费, 价值没被兑现。这种刺痛会让他留下来。

### 6 个月后熟练期

半年后, 他的使用节奏稳定成:
- **早晨 5 分钟**: agent 推送"今日 3 件事需要关注 + 1 件事可能该动" (基于咨询师 + 私有模型 + 市场)
- **每周日 30 分钟**: review 本周所有决策, 标记"按建议 / 按自己判断 / 不动" 三类, agent 计算各自回报
- **每月 1 次深调**: 他调整自己的私有模型参数, 基于这个月的结果
- **遇到 event**: 财报、FOMC、非农 — agent 提前 12 小时推一条"这事对你哪些持仓有影响 + 咨询师说过什么 + 我建议你怎么准备"

他开始**看见自己的进化轨迹**:
- 3 个月前他有 40% 的 trade 是"没想清楚就下单", 现在降到 15%
- 他的私有模型 v1 和咨询师一致率 60%, 他自己的决策和咨询师一致率从 50% 升到 70% (他学会了)
- 他已经掌握 8-10 条技术指标的白话含义, 能独立读懂大部分咨询师的周报语言
- 他的年化回报**第一次跑赢 naive 持有标普**约 3%, 不是靠 alpha, 是靠少犯情绪错误 + 更准的仓位管理

**6 个月后他真正得到的, 不是'投资变好了'**, 是**'他不再是投资新手了'**, 而且**agent 还在陪他继续进化**。

### 如果只有他一个人用 (因为不商业化)

这个问题问得好。答案: **这反而让产品比商业版本更自由**。
- 不用考虑合规 → 可以直接给方向性建议、仓位建议, 不用每页写 disclaimer
- 不用考虑 scale → 可以用最 expensive 的 LLM 推理、最大的上下文、慢响应但更深分析
- 不用考虑多用户兼容 → 所有数据结构为他一人优化, 私有模型为他一人训练
- 不用考虑新手 onboarding → 可以假设他有 CS 背景, UX 可以 geek 一点 (Jupyter-style 而非 app-store style)
- 不用考虑 "万一用户诉我" → agent 可以大胆尝试建议, 错了就错了, 数据反馈回去改进下次

他自己**是开发者也是用户**, 这是一个罕见的结构, 类似 **Obsidian 重度用户 + 插件生态** 或 **Emacs 用户 + 自己写 elisp** 或 **quant researcher 的个人研究项目**。产品形态上, 它更像 **"一个 Jupyter notebook 级别的个人投资决策环境"**, 不像 "一个 app"。

## 2. 在新 framing 下, 真正的 novelty 在哪

### 旧 L2R2 对 novelty 的判定 (部分仍有效)

L2R2 说"真 novelty 不在 concept, 在一个非常窄的执行切片"。这个**仍然成立但 slice 不同了**。

### 新 framing 下的真 novelty

**不是"投资纪律层", 而是"个人可进化的金融元工具"**。具体来说:

1. **"一个 ML PhD 给金融初级的自己建一个投资助手"这个结构本身很少见**。市面上的 AI 投资产品要么面向小白 (robo-advisor) 要么面向专业 (Bloomberg Terminal), 很少面向"元能力强但 domain 初级"的人。这类人通常的选择是"雇一个顾问或者干脆不投资", 而他选择了第三条路: **自建**。
2. **"学中做 + 做中学"的嵌入式金融教育**。绝大多数金融教育是离线的 (课程、书、Coursera), agent 这个场景下的教育是**贴着每个具体决策发生的** — "我现在要决定, 这里有个我不懂的词, 告诉我什么意思, 同时告诉我这个具体决策怎么办"。这个节奏下的学习效率远高于离线学习。
3. **"私有模型 as first-class citizen"**。商业 AI 投资产品的模型是厂商的; 这里模型是**用户自己**的, agent 只负责把模型的输出 + 外部信息 + 用户决策交互起来。**Agent 是一个编织器, 不是一个黑箱**。

### 但要诚实: novelty 也有显著薄弱点

- "ML PhD 用 AI 做自用量化工具"这件事**在 quant 圈并不新**。无数 quant 个人项目 (GitHub 上有成千上万), 有些还用 LLM 做周报监控。
- "把顾问观点翻译成个人化操作"这件事**在财富管理行业 hybrid advice 模式里也在发生** (Vanguard Hybrid, Schwab Intelligent, Betterment Premium 等 — 这是 L2R2 GPT 验证过的)。
- 纯 novelty 压缩下来是: **"ML PhD + 华语投顾订阅者 + 金融初级学习者 + 美港 A 三市场 + 不商业化"这个 exact intersection**。这个交集里的人, 可能全中国也就几万个。**这不是 novelty, 这是 hyper-niche self-use**。

**结论**: 做"私人项目"的意义**不需要 novelty 来合法化**。这是 idea 的 framing 本来该有的诚实 — 它不需要是新的, 它只需要是**对我这一个人有用的**。

## 3. Utility — 3 个具体使用场景 (新 framing 下)

### 场景 A: 周一早晨, 咨询师说"右侧布局半导体", 他第一次不是凭感觉操作
上文 §1 已展开。关键情感 outcome: **第一次, 他不再觉得自己在"跟 Tips", 他觉得自己在"根据 validated reasoning 做决策"**。他对妻子说晚饭时: "这个工具让我感觉自己不是瞎买。"

### 场景 B: 某个周三下午 2 点, FOMC 会议快开, 他在会议间隙打开 agent
Agent 弹出: "2 小时后 FOMC 决议. 你持仓中对利率最敏感的 5 只 (依赖信用的消费、小盘、地产 ETF) 合计占 22%. 过去 4 次 FOMC 后 24 小时内, 这类标的平均回撤 -1.8%, 你上次在类似场景是 2026-01 FOMC, 你选择了不动, 事后看选择是对的 (+0.5% 在 3 天内回补). **本次建议: 不动, 保留观察**. 如果决议鹰派, 明日开盘 -2% 是你的止损或加仓 trigger, 你上次设的是 stop-out, 要不要改成 opportunistic buy?" **他看完, 意识到自己过去三次都在 FOMC 前慌张调仓结果都吃亏**。他这次选了"不动 + 设置 -2% 加仓 trigger"。事后 FOMC 没那么鸽也没那么鹰, 没触发, 他赢了 0.3% (本不属于他的)。**最关键的情感 outcome**: 他意识到 **agent 在把他的历史行为变成他的教材**, 这比任何网课都更 personal。

### 场景 C: 6 个月周日晚上回顾, 他看见自己的进化曲线
打开 "半年 review":
- 本期做了 38 个决策, 27 个按 agent 建议, 7 个按自己判断 (和 agent 分歧), 4 个完全自己决定 (agent 无意见)
- 按 agent 建议的 27 个里, 21 个正向回报 (贡献 +4.2%)
- 按自己判断的 7 个里, 3 个正向 (-0.8% 净)
- 完全自己决定的 4 个, 2 个正向 (+0.3% 净)
- 他的私有模型 v3 vs v1, 在这 6 个月的样本上, 胜率从 54% 升到 61%
- 最大的学习: "我单独做决策比跟 agent 合作差 4%, 但差别在**没耐心等确认**, 不在方向看错. 下半年我设一个规则: 如果我想偏离 agent 建议, 先写 3 句话理由到系统里, 24 小时后再决定."
- 他发现自己掌握了 15-20 条金融概念, 已经能读懂咨询师大部分周报的原始术语

**情感 outcome**: 不是"我变富了", 是**"我变成了一个真的会投资的人, 而且还在变好"**. 这是这个工具给他的最核心价值。

## 4. 自然延伸 (v0.1 之后)

### v0.2 (1-3 个月内)
- **个人金融笔记系统**: 每次 agent 解释的新概念, 存进他自己的 wiki. 他可以随时查询/复习. 这解决"我不想学但要懂"。
- **自动化咨询师监控**: agent 自动 parse 他的微信订阅, 每周抓取并结构化 (模型 → 方向 → 具体标的 → 置信度). 他不用再手动贴。
- **简单的私有模型 v1**: 他用 ML 技能做第一版, 可能是 XGBoost + 技术指标特征, 每周出分值。Agent 把它的输出纳入决策建议。

### v0.5 (3-9 个月)
- **Trade 前 pre-mortem + 后 post-mortem** (这是旧 L2 的核心 loop, 现在作为子系统保留, 不再是全部): 配合建议使用, 让每笔决策都留下轨迹。
- **多市场协同**: 美股、港股、A股的 cross-market signal (如美股半导体 ↔ A股半导体 ↔ 港股芯片, 或美元 ↔ 黄金 ↔ 中概)。
- **事件日历**: 主动推送财报/FOMC/非农等对他持仓有影响的事件。

### v1.0 (9-18 个月)
- **私有模型 v2+**: 他基于 v0.1-v0.5 的样本持续迭代模型, agent 提供 backtest + 实盘对比环境 (像一个 personal quant research kit)
- **配偶可见度模块** (可选): 一份简化版, 展示"家庭财务健康 + 最近决策理由", 让妻子能在任何时候看到, 但不用学金融术语. 这对应 L2R1 提过的场景 C (家庭对话)
- **持续学习的模型 tutor**: 基于他 6 个月的决策样本, agent 主动指出 "你似乎在 [某种情境] 下倾向于 [某种错误]". 这从"教金融概念"升级到"教他自己"。

### v1.5+ (长期, 不急)
- **半自动化执行** (仍需他确认): 事件触发时, agent 可以预先起草好一个 order, 他一键确认. 缩短决策到执行的时间。
- **Paper trading 测试新策略**: 在真钱之外, agent 维护一个模拟账户, 让他可以把新想法先跑 1-3 个月再决定是否投入真仓。

## 5. 新 framing 下的自然限制

### 仍然有效的限制
- **不做期权、加密、高杠杆、日内** — 用户自己不懂、不感兴趣、且违背"稳健 + 长期"目标
- **不自动下单** — user 保留最终决策权 (心理舒适度 + 学习路径需要)
- **不 global, 聚焦三市场 + 华语输入** — 这是 user 的实际情况

### 新 framing 下需要新红线
- **不许 agent 变成 "信号黑箱 / 神谕"**: 每次建议必须带 "为什么 + 依据 + 白话解释". 如果某天他不再读解释直接按, 产品就在悄悄失败 (他没在学). 需要一个机制定期提醒/测验他, 确保他的金融素养在成长。
- **不许私有模型变成过拟合的玩具**: ML PhD 最容易犯的错是把自己的测试集当成 gospel. Agent 要主动挑战他的模型 — "你这个模型过去 3 个月在 [某种情景] 表现反常, 要不要 retrain?" 这类 meta-feedback 必须存在。
- **不许路径依赖咨询师一个人**: 那位哥大顾问是重要 input, 但不该是 gospel. Agent 要追踪 "你按咨询师建议的结果 vs 按自己或按模型的结果", 让他看清哪一路是真 edge、哪一路只是 anchor.
- **不许变成"看盘焦虑机"**: 主动提醒要节制 — 最多每日 1 次晨报 + 每周 1 次周报 + event 前 1 次, 不能演变成高频通知让他全天挂在上面. 目的是让他更少焦虑而非更多。
- **不许淹没在工具实现中**: 他的主业不是金融, 也不是金融工具开发. 如果他 80% 时间在调 agent 而不是在做 CS 主业, 这个 idea 对他的整体生活就是负的. 产品要自我约束 — 让他每周花在系统维护上的时间 <= 3 小时 (实际使用时间不限)。

### 消失的限制 (商业产品的枷锁, 自用不受)
- 不再限制"必须合规 / 不能给具体仓位建议 / 不能承诺 alpha"
- 不再限制"必须 onboard 普通用户 / 必须 UX 友好到 grandma 能用"
- 不再限制"必须跟 benchmark 比 / 必须有清晰市场定位"
- 不再限制"必须抗付费流失"

## 6. 新 framing 下的三个 honest 问题 (seed L2R3 search — 给 Codex)

1. **"自用 AI 工具长期自己用下去" 这件事, 在类似受众 (高 domain 知识 + 某领域新手 + 自建工具) 里, 成功案例和失败案例分别长什么样?** 比如: 程序员自己做的生产力工具 (bullet journal app)、quant researcher 的个人 trading system、Emacs/Obsidian 重度用户的自定义 setup — 有多少做了 6-18 个月之后消失了? 原因是什么? (用户是唯一维护者, 没人推着他用, 有什么 pattern 让这种工具"能活久"?)

2. **金融初学者用 AI 辅助做实战投资 ("做中学") 的真实学习效率和风险 — 有没有类似实验的文献/案例?** 搜: "AI tutor investing 学习曲线"、"retail investor learning curve"、"using AI while investing beginner outcomes"。关键是: 这种"嵌入式学习"真的比"先学完再投"更快地建立金融认知, 还是会让人**停留在永远依赖 agent 的状态**, 金融素养并未真正增长?

3. **付费订阅投顾 + 自建 AI 放大器, 存不存在"用久了反而不需要咨询师"的动态?** 这个问题对 user 的长期经济成本有影响 (如果 3 年后他退订咨询师, 他就省了 $500/年, 但他的 edge 可能也在下降). 搜: 付费投顾客户的 churn 原因分布里, "我学会了自己做" 占比多少。

## 总结 (给 Codex L2R3 和后续 synthesis 看)

新 framing 下, idea 从**"投资纪律层 (商业化, 不承诺 alpha, 卖克制)"** 彻底转变为 **"个人可进化的投资副驾驶 + 金融教练 (自用, 承诺稳定赚钱, 愿给方向性建议和仓位建议)"**。

核心价值主张不是"让你不背叛自己的判断" (那是商业产品话术), 是 **"让一个元能力强但金融初级的人, 通过实战 + agent 辅助, 逐渐变成会投资的人, 同时稳定赚钱"**。

关键成功要素 (新 framing 下):
- **教育嵌入决策**: 每次建议都白话解释 + 知识积累
- **私有模型 as first-class**: 用户自建, agent 只编织
- **愿意给具体建议**: 仓位、方向、时机都给, 不躲合规
- **抗"永久依赖"**: 要定期测验用户金融素养是否真的在涨
- **抗"工具吞噬主业"**: 维护时间上限约束

核心风险 (新 framing 下):
- 私有模型训不出 / 过拟合 / 用户信任不起来
- 咨询师信息自动解析精度不够
- "做中学" 变成 "做中不学" (永久依赖 agent, 金融素养未增长)
- 工具 break 后没人修 (单人维护风险)
- 6-12 个月后 user 怠惰停用 (自用产品最大失败模式)

**Spark** (用户会兴奋的那个瞬间):
不是"我赚钱了", 是 **"我读懂了以前读不懂的那条新闻, 并且独立判断它对我有什么影响 — 这是半年前的我做不到的"**。
