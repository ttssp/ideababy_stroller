## 读完 proposal 我听到的"真问题"

> 人到中年, 以前只想着安心上班赚工资, 忽略了投资理财。经济下行、人口下降、秩序混乱, 需要为整个家庭的生活质量负责。

表层诉求: 做一个"投资顾问+助手 agent"辅助个人决策。
底层诉求: **一个从未真正建立过理财肌肉的中年 CS PhD, 面对家庭财务的不确定性, 想要一个"陪伴+放大自己判断力"的东西**。

— proposal 隐含了三个假设: (a) 解法是 agent; (b) 场景是交易决策; (c) 服务对象是"我自己这个操盘人"。下面的 Part A/B/C 会分别松动这三个假设。

---

## Part A · 相邻方向 (Adjacent)

### A1. "家庭 CFO" 而非"个人操盘手"
**一句话**: 不是辅助你选股, 而是替全家管"总资产-负债-现金流-保险-税-教育金"的局, 炒股只是其中一个 sleeve。
**这是什么感受**: 早晨手机弹出: "本月房贷+保费+孩子学费现金流缺口 ¥8k, 建议从货币基金账户转入; 港股持仓 YTD +7%, 已超过你的 15% 再平衡阈值, 是否减仓?" 它管的不是"涨跌", 是"这个家能不能睡好觉"。用户是你, 但决策边界是全家, 不是一个交易员。

### A2. "投资教练"而非"投资顾问"
**一句话**: 不告诉你买什么, 而是在你想买的时候问你三个问题, 强迫你写下假设, 事后复盘"你当时凭什么觉得这是好机会"。
**这是什么感受**: 你想买英伟达, 它弹出: "相比你 3 周前说的组合纪律 (单持仓 ≤8%), 这会让 NVDA 到 12%, 确认吗? 你当前给出的多头理由是 '数据中心需求', 但你 2025-09 给出的空头风险是 '推理成本下降导致算力需求放缓' — 请回应这个矛盾。" 它不做预测, 它让你的判断"有记忆、能复盘"。对 CS PhD 特别适配: 你不需要被告诉, 你需要被挑战。

### A3. "信息源的去个人化" (把你订阅的那个哥大顾问, 变成 n 个顾问的平均态)
**一句话**: 把你付费订阅的那一个顾问, 扩展成"把同类级别的 10 个顾问/机构的公开观点聚合成一致度矩阵", 你不再依赖单点观点。
**这是什么感受**: 每周一早晨一页卡: "本周美股主题共识度排序: AI 资本开支 (9/10 机构看多) > 油价 (6/10 分歧) > 中国消费复苏 (2/10 看多, 分歧大)。你订阅的 [顾问 X] 独家观点: 看多小型医疗器械股, 没有其他顾问跟。" — 它把"你过度依赖一个人"这个隐藏风险暴露出来, 并告诉你哪些是共识、哪些是边际观点、哪些是孤立押注。

### A4. "被动投资者 + 卫星仓择时" 的工具化
**一句话**: 假设 70% 仓位锁死在 VTI/VOO 这类宽基指数 (彻底放弃主动), 只用一个工具伺候剩下 30% 的"卫星仓"的节奏、再平衡、税损收割。
**这是什么感受**: 每月发你一页: "核心仓 (70% 宽基) 无动作。卫星仓: AAPL 位置在止盈带上沿, 考虑减 1/3; TSM 账面浮亏 8%, 离 wash-sale 窗口还有 3 天, 可做税损收割; 你的美股账户现金 5%, 是否部分 DCA 进月末?" — 这承认"你 PhD 背景也无法系统性赢市场", 但把"你能赢的那 30%"的执行纪律外包给工具。

### A5. "同频社区中的私人镜像" (Audience swap: 个人 → 同阶段家庭圈)
**一句话**: 面向"35-50 岁、家庭现金流紧、有点专业背景但不是金融出身"的小圈子, 每人有一个自己的 agent, agent 之间匿名交换"决策理由骨架"而非具体持仓, 形成同频镜像。
**这是什么感受**: 你做了一个"减仓港股消费"的决定并写下理由 → agent 匿名化后推送: "本圈 34 人里, 7 人本周做了类似决策, 主因 4 人说'消费数据弱'、2 人说'外资流出'、1 人说'仅仓位再平衡'。你的理由归入最大集群。" 这把"孤独决策"变"群体校准", 但不变"跟单"。

## Part B · 极端方向 (Extended)

### B1. 最大雄心版: "家庭金融操作系统 (Family Financial OS)"
**一句话**: 不是一个 agent, 是一整套替中产家庭打理"钱+保险+税+信贷+遗产"的本地优先系统, 市场投资只是其中一层。
**这是什么感受**: 像 Notion 之于知识管理那样, 它是你家庭的"财务中台"。孩子将来要上大学, 它从 4 岁起就在帮你规划教育金路径; 父母将来要用, 它会问你"如果你父亲需要 50 万医疗现金, 你会从哪卖?"。投资只是其中的一个 module, 但整个系统的 framing 是"家庭生命周期财务"。野心: 让一个家庭可以不请财务顾问, 也做出机构级别的资产负债规划。

### B2. 最小聚焦版: "周日早晨 10 分钟"
**一句话**: 只做一件事 — 每周日早上 9 点给你一页 PDF, 总结本周你 30-50 只关注股的关键信号 + 你订阅那位顾问的核心观点 + 下周 3 个需要盯的事件。除此之外什么都不做。
**这是什么感受**: 没有 dashboard, 没有 chat, 没有 agent 对话。只有一个固定的每周仪式。像 Stratechery 之于科技分析、像 Morning Brew 之于新闻 — 一份专为你一个人定制的周刊。最小, 但仪式感最强, 最容易坚持最久。

### B3. 跨域移植: "把 AI code review 的范式搬到投资决策"
**一句话**: 就像 code review 强迫你在 merge 前暴露假设、列清单、被同行挑战 — 做一个"trade review": 你下单前必须走一遍 agent 的 linter (仓位/分散度/止损/相关性) + agent 扮演 devil's advocate 挑战你的多空理由, 通过了才放行。
**这是什么感受**: 你在券商 app 敲下买单的瞬间, agent 弹窗: "本次下单占你组合 6%, 触发'单持仓高集中'告警; 你的多头论点与你 2 周前写下的 [XYZ] 类股持有逻辑 80% 重叠, 已存在实质集中; 你是否要把这两只股视为一个 position 重新考虑大小?" — 投资决策第一次有了"PR 流程"。对 CS PhD 的心智模型特别顺。

## Part C · 重构方向 (Reframed)

### C1. "真问题不是投资, 是焦虑"
**一句话**: 中年家庭需要的不是多赚 5%, 而是"知道自己大概是安全的"的那个确定感 — 换个形态, 这是一个"家庭财务安全感仪表盘", 帮你看清"即使市场跌 30%, 我家的生活能撑多久"。
**这是什么感受**: 打开 app 只看一个数字: "你家的财务韧性 = 18 个月"。点进去分解: 现金覆盖 6 个月, 可变现非核心资产 4 个月, 保险兜底 8 个月。每个月它会问: "如果明天失业, 你和家人的生活质量会下降多少? 最低可接受值是多少?" — 它卖的不是 alpha, 是**晚上能睡着**。这可能是比"投资助手"更精确命中痛点的形态。

### C2. "解法不应该是 agent, 应该是协议/仪式"
**一句话**: 不做 agent, 做一套"中年家庭投资生活的仪式协议": 每周 15 分钟复盘模板 + 每月 1 次组合体检清单 + 每季度一次家庭财务会议脚本。Agent 只是辅助执行, 但**产品形态是"理财版本的 GTD/晨间日记"**。
**这是什么感受**: 它像"跑步 app 让你变成跑者"那样, 让你"变成会投资的人"。重点不是 AI 智能有多强, 而是这套仪式能不能嵌进你的生活节奏。这里的假设松动: 你需要的不是"更聪明的 AI", 你需要的是"一套让你持续做正确的事的结构"。

### C3. "服务对象不是操盘人, 是配偶"
**一句话**: 反向思考: 你在投资, 但真正的"用户"可能是你配偶 — ta 不懂投资, 不知道你在干什么, 担心你赔钱伤家庭。做一个"配偶友好的投资透明化工具", 让不懂的那一方也能理解、信任、参与(而非盲目放权)。
**这是什么感受**: 每月一页"给家属的报告": 大白话、无专业术语、用家庭月收入倍数而非 % 表达、用"这相当于我们三个月的家用"这种人话。配偶通过一个"简化视角"登录, 不看具体持仓, 只看"我们家财务健康度", 但可以对重大决策"投票"。这是一个彻底松动"服务对象"假设的重构 — 可能比 100 个自动化功能都更能改变一个家庭的金融关系。

## Part D · 我的 Top 3 (带 spark 说明)

### D1. "家庭 CFO" (来自 A1)
**如果它存在会是什么感受**: 它不是你打开来看涨跌的 app, 它是每天晚上 10 点推送一条:"今天你家净资产 -0.3%, 现金流 OK, 下周学费缴纳准备好, 下月港股增持信号暂缓。" 你睡觉前看一眼, 像看一眼孩子有没有睡好那种放心。
**Spark**: 市面上太多"让你更聪明交易"的工具, 几乎没有"帮你打理整个家庭财务生命"的工具 — 而后者才是中年人真正焦虑的维度。把投资从"游戏"重新嵌回"家庭责任"的框里, 是最大的语义回归。
**人自己大概不会想到**: 你的 proposal 写的是"投资顾问+助手", 这是**操盘手的 framing**; 但你自己在"我为什么想做这个"里写的是"为整个家庭的生活质量负责" — **你自己已经给出了更深的 framing, 只是被"顾问"这个产品心智模型遮蔽了**。很多 CS 背景的人默认"做一个 agent 解决 X", 但这里真正该做的是 "rethink framing 再决定要不要做 agent"。

### D2. "投资教练" / "决策 PR 流程" (A2 × B3 合并)
**如果它存在会是什么感受**: 你下单前必过一道关 — 像你写代码必过 CI 那样。它不预测市场, 它只让你"留下轨迹、暴露矛盾、被挑战"。3 个月后你翻回看: "2026-01 我为什么相信 TSM 会到 $250? 当时给了三个理由, 2 个已被证伪, 1 个仍成立 — 下次我该更谨慎对待这类 thesis。" 你不会变成巴菲特, 但你会变成一个**有纪律、有记忆、有复盘的**中等好的投资者, 而且你的进步会是可见的。
**Spark**: 大多数投资产品都在往"给你答案"的方向走(AI选股、量化信号、跟单); 但对一个 ML PhD, **你要的从来不是答案, 是结构化的挑战**。把 code review 的文化搬到投资 — 这是第一次把"程序员的元能力"用到自己的钱上, 且产品定位独特, 没什么人在做。
**人自己大概不会想到**: 你的领域偏见会让你倾向"用 ML 做预测"; 但 ML PhD 应该最懂**个人做不出超过 benchmark 的预测模型** — 你自己在 proposal 里也承认了"个人做不了量化"。所以真正该建的不是预测工具, 是**决策过程的基础设施**。这个跳跃很反直觉, 一般人不会主动做出来。

### D3. "家庭财务韧性仪表盘" (C1)
**如果它存在会是什么感受**: 主屏幕只有一个数字 + 一个时长条。"如果今天起全家没新收入, 你家可以维持 18 个月的当前生活质量。" 下面一行字: "上月 17, 这个月 18, 稳中有升。" 你每天看一眼。市场跌 20% 那天它可能掉到 14, 但它会告诉你: "仍在安全带内, 不要动作。" — 它把"焦虑"这个底层问题直接对上了 UI, 而不是绕道通过"帮你多赚钱来缓解焦虑"。
**Spark**: 几乎所有 fintech 都在卖"更高的回报", 极少有人卖"更清晰的安全感"。这是一个**情绪层产品**, 但底子是硬核的家庭资产负债建模。有机会做成"中年人早晨必看的那个数字", 就像跑步人早晨看的体重或步数。
**人自己大概不会想到**: CS PhD 的默认心智是"优化收益率", 但你 proposal 里真正动情的那句是"为整个家庭的生活质量负责" — **生活质量 ≠ 收益率**。生活质量的对偶指标是"抗冲击能力" (resilience), 不是 return。这个重构需要跨出"投资"的框, 站在"家庭生存"的框里看, 对一个正沉浸在"想做投资智能体"兴奋里的人, 是最容易被绕过的视角。
