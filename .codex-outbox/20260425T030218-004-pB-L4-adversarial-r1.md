## Why this verdict

这不是“再润几个 task 就能开 build”的状态。当前 spec 同时存在三个 contract 级问题：把 pre-commit 做成了 post-commit、把 `agent_synthesis` 既当泳道又当裁判、以及给了一个无法按字面实现的 `StrategyModule` IDL。  
如果现在直接进入 build，worker 只能在代码里偷偷改 contract，或者交付一个 paper-green 但行为错位的 v0.1。  
这些问题都正好打在本项目最不能错的三条线上：`<30s`、R9、R10。  
先做 round 2 收紧 spec，成本会远低于 build 后返工。

## If BLOCK or CONCERNS: 推荐 round 2 的聚焦点

- 先把 core loop 重写成真正的 **pre-commit**：三路报告与 rebuttal 何时生成、何时显示、何时正式落档，三步必须拆清。
- 把 `conflict_resolve` 从 `agent_synthesis` 里拆出来，重新定义“第三路 signal”与“中立聚合器”的边界。
- 修正 `StrategyModule` contract，使 `agent_synthesis` 的输入与 R9 解耦要求一致；同时回切 task DAG，优先把 O10 提前、把 `T021` 从 `T019` 解耦，并明确 D9 是否继续走 watched folder。
