## Challenge these specifically(按优先级)

1. **Stuck 状态机边界条件**(architecture §8, T017, T022)
   - 5min 冷启动豁免期内若真卡住(下载挂死、驱动 hang)怎么办?
   - GPU util 5% 阈值在 NVLink / 多卡 / MPS 场景下会不会误判?
   - 网络 IO MB/s 阈值遇到 HF 断点续传(bursty pattern)如何?
   - 60s 滚动窗口重叠边界是 inclusive 还是 exclusive?
   - 连续 3 次 restart 熔断后,状态持久化在哪里?操作员手工清除的路径?
   - spec.md O7 要求"stuck 误杀 = 0"(硬)— 状态机的 false negative (真卡但不触发 SIGINT)是否也被视为失败?

2. **API key 剩余攻击面**(architecture §5, T010/T011/T018)
   - worker 子进程用 `subprocess.Popen(env=clean_env)` 启动,但 Python subprocess 能否在 macOS 上真的保证**不继承** `ANTHROPIC_API_KEY`?
   - macOS `ps -E` 与 Linux `/proc/<pid>/environ` 能读到父进程 env 的差异,spec 都 cover 了吗?
   - LiteLLM proxy 自身的 auth token 在哪里?若 worker 可以调 proxy,worker 仍能烧 USD — 这不是 exfil 但是 DoS 成本
   - HF_TOKEN 允许进 worker env(D10) — HF token 能访问 gated 模型/私有 repo,scope 未必"小";若 worker 被 prompt injection,会不会把模型权重 exfil 到 HF mirror?
   - budget 硬上限 SIGINT 的延迟窗口(几秒级)内,worker 能烧掉多少 USD?

3. **配置只读分离的 macOS 无 root fallback**(architecture §6, T012)
   - `bindfs --ro` 需 MacFUSE 第三方依赖;`mount --bind` 仅 Linux;`chflags uchg` 需 sudo
   - 最常见 fallback 是 `chmod -R a-w .claude/` — worker 若拿到 `Bash(chmod:*)` 权限就能 `chmod u+w` 绕回
   - 仅靠 hooks deny `chmod` / `chattr` / `chflags` 够不够?attacker 能用 `python -c "import os; os.chmod(...)"` 绕过 Bash deny 吗?
   - worker 跑 `pip install` 能不能偷偷在 site-packages 写 hook 替换?

4. **pip 供应链完整防线**(architecture §?, T002/T015/R5)
   - `pip install --require-hashes` 是否启用?仅 pin 版本不够,还能 typosquatting + 同名覆盖
   - worker 跑 `pip install` 的权限怎么给?若禁则训练脚本不能动态安装依赖;若允许则 R5 还在
   - `requirements-locked.txt` 首次生成用 `pip-compile --generate-hashes`?spec 里是否明写?
   - CVE 监控 / snyk / pip-audit 在 CI 里跑吗?OSS 个人项目可能不跑 CI

5. **数据流与 checkpoint portability**(C9, T004/T019/T027)
