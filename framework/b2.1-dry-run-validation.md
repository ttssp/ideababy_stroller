### Step 4 · 6 约束自检(producer 端)

> ⚠ **Prerequisite**(B2.2 Block A friction #2 fix):本 step 必须在 **Step 3 mock hand-back 包产出后**跑,且要传 `--mode=producer`。
> **不要用 `discussion/<id>/<fork>/L4/HANDOFF.md`**(那是 forward hand-off 包,plan-start v3.0 产 / XenoDev 消费;无 `handback_id` 字段)。
> validator 检测到误用会显式 stderr 提示("⚠ 提示:本文件含 workspace.handback_target 但无 handback_id;可能是 forward HANDOFF.md")。

```bash
SOURCE=/Users/admin/codes/ideababy_stroller
KIT=$SOURCE/framework/xenodev-bootstrap-kit/handback-validator

# producer 模式:writer 端写入前自检(文件还在 /tmp,跳过 check-5 路径段校验)
bash $KIT/validate-handback.sh "$MOCK_BUILD/$HANDBACK_FILE" "$SOURCE" --mode=producer
echo "rc: $?"
```

**期望输出**:
- stderr 输出 `→ Validating ... (mode: producer)` + `⏭  check-5 skipped (mode=producer; ...)` + `✓ producer-mode checks PASS for <handback_id>(check-5 已跳)`
- exit 0

若 exit 1 → 看 stderr 末尾报哪条约束失败:
- `约束 3 (repo identity)` → frontmatter `source_repo_identity:` 字段填错(无 git remote 时 `expected_remote_url` 留空,`repo_marker` 必含 "Idea Incubator")
- `约束 6 (id charset)` → `prd_fork_id` 不匹配 `^[0-9]{3}[a-z]?(-p[A-Z])?$` regex(eg `006-mock` 不匹配,`006a-pM` 匹配 — 见 Step 3 命名规范)
- `frontmatter 必要字段缺失` → 检查 mock 包是不是真的有 `discussion_id` / `prd_fork_id` / `handback_id` / `workspace.handback_target` 4 字段
